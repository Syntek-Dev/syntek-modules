## User Story

As a developer, I want a Next.js security package so that every Syntek web application has
consistent CSP nonce injection, configurable security headers, XSS sanitisation utilities, and CSRF
helpers — all composable from `next.config.ts` and React components without boilerplate.

## Architecture: Layered Security

```text
Next.js Middleware Layer
  ├── withSyntekCsp — per-request nonce generation
  │   └── Merges nonce into CSP header (preserves existing policy)
  │   └── Stores nonce in x-nonce header for RSC layer
  │
Server Component Layer (RSC)
  ├── getCspNonce() — read nonce from incoming request
  │   └── Pass to <script nonce={nonce}> / <style nonce={nonce}>
  │
next.config.ts Layer
  ├── syntekSecurityHeaders() — build HSTS, X-Frame-Options, COEP, COOP, CORP
  │   └── Every option independently configurable; empty options omitted
  │   └── Used in async headers() callback
  │
Client Component / Utilities Layer
  ├── sanitiseHtml() — DOMPurify wrapper for XSS prevention
  ├── stripTags() — plaintext extraction (no external deps)
  ├── escapeHtml() — HTML entity escaping
  ├── getCsrfToken() — read Django csrftoken cookie
  ├── withCsrf() — add X-CSRFToken to fetch headers
  └── useCsrfToken() — reactive hook for CSRF on SPA navigation
```

**Entry Points in `package.json` exports map:**

```json
{
  "./middleware": "./src/middleware.ts",
  "./server": "./src/server.ts",
  "./headers": "./src/headers.ts",
  "./sanitise": "./src/sanitise.ts",
  "./csrf": "./src/csrf.ts"
}
```

## Middleware: CSP Nonce Injection

```typescript
import { withSyntekCsp } from "@syntek/ui-security/middleware";

// next.config.ts
export default withSyntekCsp(nextConfig);

// Behaviour:
// - Generates crypto.randomUUID() nonce per request
// - Injects nonce into Content-Security-Policy header
// - Stores nonce in x-nonce custom header for RSC to read
// - Merges nonce into existing CSP policy — does NOT replace
// - Safe to call multiple times in middleware chain
```

## Server: RSC-Safe Nonce Access

```typescript
import { getCspNonce } from '@syntek/ui-security/server';

// layout.tsx (RSC)
export default function RootLayout({ children }) {
  const nonce = getCspNonce(); // reads x-nonce header

  return (
    <html>
      <head>
        <script nonce={nonce}>
          // inline script — nonce required if CSP forbids unsafe-inline
        </script>
      </head>
      <body>{children}</body>
    </html>
  );
}

// getCspNonce returns null if no nonce is set (not in middleware chain)
// Safe to call in RSC context — reads from incoming request headers
```

## Headers: Security Headers Configuration

```typescript
import { syntekSecurityHeaders } from '@syntek/ui-security/headers';

// next.config.ts
async headers() {
  return [
    {
      source: '/(.*)',
      headers: syntekSecurityHeaders({
        csp: "default-src 'self'; script-src 'self' 'nonce-{nonce}'; style-src 'self' 'nonce-{nonce}'",
        hstsMaxAge: 31536000,           // number (seconds)
        hstsIncludeSubdomains: true,    // boolean
        hstsPreload: false,              // boolean
        frameOptions: 'DENY',            // 'DENY' | 'SAMEORIGIN'
        referrerPolicy: 'strict-origin-when-cross-origin',
        permissionsPolicy: 'geolocation=(), microphone=()',  // empty = omitted
        crossOriginEmbedderPolicy: 'require-corp',           // empty = omitted
        crossOriginOpenerPolicy: 'same-origin',              // empty = omitted
        crossOriginResourcePolicy: 'same-origin',            // empty = omitted
        xContentTypeOptions: true,  // when true, always sets to 'nosniff'
      }),
    },
  ];
}

// Returns { key: string; value: string }[] array for use in headers() callback
// Unset options are omitted — no insecure defaults silently applied
// xContentTypeOptions: true always sets nosniff (hardcoded correct value)
// Empty string options are excluded from output
```

## Sanitisation: XSS Prevention

```typescript
import { sanitiseHtml, stripTags, escapeHtml } from "@syntek/ui-security/sanitise";

// sanitiseHtml — DOMPurify wrapper (browser only, requires DOMPurify peer dep)
const clean = sanitiseHtml("<p>Safe</p><script>evil()</script>", {
  allowedTags: ["p", "a", "strong", "em"],
  allowedAttributes: { a: ["href", "title"] },
  forceHttps: true, // upgrades http:// to https:// in hrefs
});
// Returns '<p>Safe</p>' (script stripped)

// stripTags — plaintext extraction (no DOMPurify required)
const text = stripTags("<p>Hello <strong>world</strong></p>");
// Returns 'Hello world'

// escapeHtml — HTML entity escaping (no DOMPurify required)
const escaped = escapeHtml('<script>alert("XSS")</script>');
// Returns '&lt;script&gt;alert(&quot;XSS&quot;)&lt;/script&gt;'
```

**Sanitisation options:**

```typescript
interface SanitiseOptions {
  allowedTags?: string[];
  allowedAttributes?: Record<string, string[]>;
  forceHttps?: boolean;
}
```

## CSRF: Django Integration

```typescript
import { getCsrfToken, withCsrf, useCsrfToken } from '@syntek/ui-security/csrf';

// getCsrfToken — sync, reads Django's csrftoken cookie
const token = getCsrfToken();
// Returns string | null
// Requires Django CSRF_COOKIE_HTTPONLY: False (already set by syntek-security module)

// withCsrf — adds X-CSRFToken to fetch headers
const response = await fetch('/api/endpoint/', {
  method: 'POST',
  headers: withCsrf({
    'Content-Type': 'application/json',
  }),
  body: JSON.stringify({ ... }),
});
// Adds X-CSRFToken header if token is available
// Safe if no token present — returns original headers unchanged

// useCsrfToken — React hook, reactive on cookie change
function PaymentForm() {
  const token = useCsrfToken(); // null on initial load if no cookie

  return (
    <form
      onSubmit={async (e) => {
        e.preventDefault();
        const response = await fetch('/api/payment/', {
          method: 'POST',
          headers: withCsrf({ 'Content-Type': 'application/json' }),
          body: JSON.stringify({ ... }),
        });
      }}
    >
      {/* No token display needed — withCsrf adds header automatically */}
    </form>
  );
}

// No network request — reads from cookie set by Django
// Updates reactively if cookie is refreshed (e.g., SPA navigation with new session)
```

## Acceptance Criteria

```gherkin
# --- withSyntekCsp Middleware ---

Given the middleware is installed in next.config.ts
When two requests are made simultaneously
Then each receives a unique crypto.randomUUID() nonce

Given a request comes through withSyntekCsp
When the response headers contain an existing Content-Security-Policy
Then the nonce is merged into the header value — existing policy is preserved

Given a request comes through withSyntekCsp
When the response is complete
Then the x-nonce header contains the generated nonce

# --- getCspNonce ---

Given getCspNonce is called in an RSC
When the request has x-nonce header set (from middleware)
Then it returns the nonce as a string

Given getCspNonce is called in an RSC
When the request has no x-nonce header
Then it returns null

Given getCspNonce is called in a client component
Then it returns null (cannot read request headers on client)

# --- syntekSecurityHeaders ---

Given syntekSecurityHeaders is called with { csp: "default-src 'self'" }
When the headers array is returned
Then it includes a header with key 'Content-Security-Policy' and the provided value

Given syntekSecurityHeaders is called with { hstsMaxAge: 31536000, hstsIncludeSubdomains: true, hstsPreload: false }
When the headers array is returned
Then it includes Strict-Transport-Security header with max-age=31536000; includeSubDomains (no preload)

Given syntekSecurityHeaders is called with { frameOptions: 'DENY' }
When the headers array is returned
Then it includes X-Frame-Options: DENY

Given syntekSecurityHeaders is called with { frameOptions: 'SAMEORIGIN' }
When the headers array is returned
Then it includes X-Frame-Options: SAMEORIGIN

Given syntekSecurityHeaders is called with { xContentTypeOptions: true }
When the headers array is returned
Then it includes X-Content-Type-Options: nosniff (hardcoded)

Given syntekSecurityHeaders is called with { permissionsPolicy: '' }
When the headers array is returned
Then the Permissions-Policy header is omitted

Given syntekSecurityHeaders is called with { crossOriginEmbedderPolicy: 'require-corp' }
When the headers array is returned
Then it includes Cross-Origin-Embedder-Policy: require-corp

Given syntekSecurityHeaders is called with { crossOriginOpenerPolicy: 'same-origin' }
When the headers array is returned
Then it includes Cross-Origin-Opener-Policy: same-origin

Given syntekSecurityHeaders is called with { crossOriginResourcePolicy: 'same-origin' }
When the headers array is returned
Then it includes Cross-Origin-Resource-Policy: same-origin

Given syntekSecurityHeaders is called with referrerPolicy: 'strict-origin-when-cross-origin'
When the headers array is returned
Then it includes Referrer-Policy header with that value

Given syntekSecurityHeaders is called with no csp option
When the headers array is returned
Then the Content-Security-Policy header is omitted

# --- sanitiseHtml ---

Given sanitiseHtml is called with '<p>Safe</p><script>alert("xss")</script>'
When it is sanitised
Then the script tag is stripped and '<p>Safe</p>' is returned

Given sanitiseHtml is called with '<img src=x onerror="alert(1)">'
When it is sanitised
Then the onerror attribute is stripped

Given sanitiseHtml is called with '<a href="http://example.com">Link</a>'
And forceHttps: true is set
When it is sanitised
Then the href is upgraded to 'https://example.com'

Given sanitiseHtml is called with '<p>Text</p><img src="image.jpg">'
And allowedTags: ['p'] is set
When it is sanitised
Then only the <p> tag is preserved and <img> is stripped

# --- stripTags ---

Given stripTags is called with '<p>Hello <strong>world</strong></p>'
When it is executed
Then it returns 'Hello world'

Given stripTags is called with '<script>alert(1)</script>Visible'
When it is executed
Then it returns 'alert(1)Visible' (all tags stripped, text preserved)

# --- escapeHtml ---

Given escapeHtml is called with '<script>alert("xss")</script>'
When it is executed
Then it returns '&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;'

Given escapeHtml is called with "It's & great"
When it is executed
Then it returns 'It&#x27;s &amp; great'

# --- getCsrfToken ---

Given the csrftoken cookie is present with value 'abc123'
When getCsrfToken() is called
Then it returns 'abc123'

Given the csrftoken cookie is absent
When getCsrfToken() is called
Then it returns null

# --- withCsrf ---

Given withCsrf is called with { 'Content-Type': 'application/json' }
And getCsrfToken() returns 'abc123'
When the headers are returned
Then they include 'X-CSRFToken': 'abc123' and the original Content-Type

Given withCsrf is called with { 'Content-Type': 'application/json' }
And getCsrfToken() returns null
When the headers are returned
Then X-CSRFToken is not added and the original headers are unchanged

# --- useCsrfToken ---

Given useCsrfToken hook is mounted
And the csrftoken cookie is present
When the component renders
Then the hook returns the token value

Given useCsrfToken hook is mounted
And the csrftoken cookie is absent
When the component renders
Then the hook returns null

Given useCsrfToken hook is mounted
And the csrftoken cookie is initially absent
When the cookie is set via SPA navigation
Then the hook updates and returns the new token

Given useCsrfToken hook is mounted in a client component
When the component uses the token in fetch headers
Then withCsrf adds the token automatically to request headers
```
