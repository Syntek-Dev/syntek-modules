# US105 — `@syntek/mobile-gdpr`: Mobile Compliance UI Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a mobile compliance UI package so that I can embed a regime-aware consent
sheet, privacy settings screen, data export request, and account deletion flow into React Native
applications — wired to `syntek-gdpr` and `syntek-consent` via GraphQL.

## Key Coverage

- `useGDPR()` hook for React Native — same interface as web, reads regime config from GraphQL
- `ConsentSheet` — bottom sheet component for initial consent (regime-aware: GDPR category toggles
  or CCPA opt-out)
- `PrivacySettingsScreen` — full screen for managing consent, requesting data export, requesting
  deletion
- `DataExportRequestFlow` — submit SAR, polling status, share/save export file on completion
- `AccountDeletionFlow` — multi-step confirmation, typed confirmation, timeline display
- All labels overridable via props

## Acceptance Criteria

```gherkin
Scenario 1: GDPR regime consent sheet
Given regime = 'gdpr'
When the app launches for the first time
Then ConsentSheet is presented as a bottom sheet before any non-essential operations run

Scenario 2: CCPA regime consent sheet
Given regime = 'ccpa'
When ConsentSheet renders
Then a "Do Not Sell My Personal Information" toggle is shown

Scenario 3: Data export request on mobile
Given a user submits a data export request via DataExportRequestFlow
When the SAR mutation resolves
Then a polling indicator is shown and the user can share/save the export file when ready

Scenario 4: Account deletion on mobile
Given a user completes AccountDeletionFlow
When they confirm with the required text
Then the erasure mutation is called and a confirmation screen is shown with the timeline

Scenario 5: Label override on mobile
Given custom labels are provided
When any component renders
Then all user-facing text uses the provided overrides
```
