# US050 — `@syntek/data-table`: Sortable Paginated Table

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 6 — Web Frontend Packages

---

## User Story

As a developer, I want a sortable, filterable, paginated data table component so that I can display large datasets in Syntek applications with consistent UX and no per-project reimplementation.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a table is rendered with sortable columns
When a column header is clicked
Then the sort order toggles (asc → desc → none) and the data re-fetches with the new sort variables

Given a filter is applied to the table
When the filter value changes
Then the query re-fetches with the filter variable and the URL query string is updated to reflect the filter state

Given the table has more rows than the page size
When the user navigates pages
Then only the current page data is fetched and the pagination controls reflect the current position

Given a row action button is clicked
When the action is confirmed
Then the action callback fires with the row data and the table refreshes to reflect any data changes
```
