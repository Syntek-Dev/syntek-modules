# US062 — `syntek-subscriptions`: Multi-Provider Subscription Lifecycle

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend

## User Story

As a developer, I want a provider-agnostic subscriptions module so that I can manage recurring
billing cycles, failed payment retries, upgrades, downgrades, and cancellations across all
configured payment providers.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a user subscribes to a monthly plan with any configured provider
When the subscription is created
Then a Subscription record is created with status active and billing dates set

Given a subscription payment fails
When the retry schedule runs
Then payment is retried at configured intervals and subscription transitions to past_due

Given all retries are exhausted
When the final retry fails
Then subscription transitions to cancelled, access is revoked, and a cancellation notification is sent

Given a user upgrades their plan mid-cycle
When the upgrade mutation is called
Then a pro-rata charge is calculated and submitted to the provider, the plan is updated, and billing date is preserved

Given a subscription uses a different provider than Stripe
When lifecycle events arrive via that provider's webhook
Then the same Subscription model is updated correctly
```
