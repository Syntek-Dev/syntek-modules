# US062 — `syntek-subscriptions`: Recurring Subscription Lifecycle Module

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 5 — Backend Compliance & Media

---

## User Story

As a developer, I want a subscriptions module so that I can manage recurring billing cycles, handle failed payment retries, and support upgrades, downgrades, and cancellations independently of the core payments module.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a user subscribes to a monthly plan
When the subscription is created
Then a Subscription record is created with status `active`, the billing cycle dates are set, and a payment is taken via syntek-payments

Given a subscription payment fails
When the retry schedule runs
Then the payment is retried according to the configured retry intervals and the subscription status transitions to `past_due` after the first failure

Given all retries are exhausted
When the final retry fails
Then the subscription status transitions to `cancelled`, the user's access is revoked, and a cancellation notification is dispatched

Given a user upgrades their plan mid-cycle
When the upgrade mutation is called
Then a pro-rata charge is calculated, the subscription plan is updated immediately, and the next billing date is preserved
```
