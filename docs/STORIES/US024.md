## User Story

As a developer, I want an events module so that I can create ticketed events with real-time capacity management, waitlists, and QR code check-in.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given an event has 0 remaining tickets
When a purchase attempt is made
Then the purchase is rejected and the user is offered the waitlist

Given two purchase requests arrive simultaneously for the last ticket
When both are processed concurrently
Then exactly one succeeds and the other is rejected — no overselling (DB-level locking)

Given a ticket QR code is scanned for check-in
When the same QR code is scanned a second time
Then the second scan is rejected as a duplicate check-in

Given a ticket is released from the waitlist
When the next waitlist entrant is notified via `syntek-notifications`
Then they have a configurable purchase window before the next waitlist position is offered

Given a seat map is configured for an event
When a user selects a seat
Then the seat is reserved and unavailable to other users for the duration of the checkout session
```
