# US117 — `@syntek/mobile-maps`: Mobile Maps, Routing & Tracking Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a mobile maps, routing, and tracking package so that I can embed maps,
display optimised routes, broadcast a worker's live position, and view another tracker's position in
React Native — with native navigation app handoff for turn-by-turn directions.

## Architecture

`@syntek/mobile-maps` wraps `react-native-maps` with platform-conditional providers (Google Maps on
Android, Apple Maps on iOS by default, configurable). Live tracking updates arrive via the
`positionUpdated` GraphQL subscription (Strawberry WebSocket). GPS broadcasting uses Expo Location
for the tracker device, with optional background task support via Expo TaskManager. Location
permissions are handled natively per platform before any access.

## Key Coverage

- `useMap()` hook — reads provider config from GraphQL
- `<MapView>` component — wraps `react-native-maps` (provider: Google Maps on Android, Apple Maps on
  iOS by default; explicitly overridable)
- `<RouteMapView>` — displays optimised route polyline and numbered stop markers
- `<TrackingMapView>` — live tracker position via `positionUpdated` subscription; animated marker
- `usePositionBroadcast()` hook — for the tracker device: reads GPS position at
  `POSITION_UPDATE_INTERVAL_SECONDS`, calls `updatePosition` mutation
- `<NavigationLinks>` — renders "Open in Google Maps / Apple Maps / Waze" buttons; opens native app
  via `Linking.openURL`
- `<LocationPicker>` — map-based location selection with address reverse geocode
- Location permission handling (iOS and Android)
- Background location (optional, for trackers who lock the screen)

## Acceptance Criteria

```gherkin
Scenario 1: Android map provider
Given the app is on Android
When <MapView> renders
Then Google Maps tiles are shown using the react-native-maps Google Maps provider

Scenario 2: iOS map provider
Given the app is on iOS
When <MapView> renders
Then Apple Maps tiles are shown by default (configurable to Google Maps)

Scenario 3: Route display
Given a Route is active
When <RouteMapView> renders
Then the polyline and numbered stop markers are overlaid on the native map

Scenario 4: Position broadcast
Given a user is the tracker
When usePositionBroadcast() is active
Then the device's GPS position is sent to updatePosition every POSITION_UPDATE_INTERVAL_SECONDS seconds

Scenario 5: Live tracking view
Given a user is viewing a tracking session
When <TrackingMapView> renders
Then the tracker's marker updates in real time via the WebSocket subscription

Scenario 6: Navigation app handoff
Given navigationApps = ['google_maps', 'waze']
When <NavigationLinks> renders and a button is tapped
Then the selected navigation app opens with all route waypoints pre-loaded via deep link

Scenario 7: Location permission prompt
Given location permission has not been granted
When usePositionBroadcast() initialises
Then the appropriate iOS/Android permission prompt is shown before any location access
```
