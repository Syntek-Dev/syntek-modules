# US107 — `@syntek/ui-media`: React Media Upload & Gallery UI

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

---

## User Story

As a developer, I want a React media UI package so that I can embed image/video upload with preview
and crop, file upload with progress, a media gallery with lightbox, and a media library browser —
all connected to `syntek-media-core`, `syntek-media-upload`, and `syntek-files` via GraphQL.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given an ImageUpload component is rendered
When a user drops or selects an image file
Then a preview is shown immediately with optional crop and aspect ratio controls; no upload begins
until the user confirms

Given the user confirms an image upload
When the chunked upload pipeline begins
Then a progress bar shows bytes_received / bytes_total and a cancel button is available at all
times until the upload completes

Given an infected file is uploaded (mocked scan failure)
When the upload session is rejected with reason virus_detected
Then a clear error message is shown inline and the preview is cleared

Given a video file is selected
When the upload completes
Then a thumbnail frame extracted by Cloudinary is displayed and the VideoUpload component
transitions to a "processing" state with a spinner until transcoding_status becomes "ready"

Given MediaGallery is rendered
When the user selects one or more assets
Then the onSelect callback is called with the selected MediaAsset array

Given MediaLibrary is opened
When the user searches by tag or MIME type
Then the gallery filters without a page reload and URL sync preserves the filter state
```

## Components

| Component           | Description                                                                                    |
| ------------------- | ---------------------------------------------------------------------------------------------- |
| `ImageUpload`       | Drag-and-drop or click to select; preview with crop (react-image-crop); confirms before upload |
| `VideoUpload`       | File picker for video; chunked upload via US106; progress bar; transcoding status polling      |
| `FileUpload`        | Document file picker (PDF, DOCX, XLSX, etc.); uploads to syntek-files; progress bar            |
| `MediaGallery`      | Responsive grid of `MediaAsset` records; lightbox on click; single or multi-select mode        |
| `MediaLibrary`      | Full-screen modal gallery browser — search by tag/type, pagination, select and confirm         |
| `UploadDropZone`    | Headless drop target — wraps any content; calls `onDrop` with accepted/rejected files          |
| `MediaThumbnail`    | Single asset display — shows thumbnail slot URL, lazy-loaded, with ARIA alt text               |
| `UploadProgressBar` | Visual progress bar for active upload sessions; shows cancel button                            |

### Hooks

| Hook                              | Description                                                                   |
| --------------------------------- | ----------------------------------------------------------------------------- |
| `useMediaUpload(options)`         | Manages the full chunked upload lifecycle — initiate, chunk, complete, cancel |
| `useUploadProgress(sessionToken)` | Polls `uploadProgress` query; returns `{ status, percent, cancel }`           |
| `useMediaAssets(filters)`         | Wraps `mediaAssets` query with pagination and filter state                    |
| `useMediaAsset(id)`               | Single asset with transformation URLs                                         |
| `useFileUpload(options)`          | Manages file upload to syntek-files via GraphQL                               |

### `useMediaUpload` Options

```ts
interface UseMediaUploadOptions {
  commitTarget: "media" | "files";
  onProgress?: (percent: number) => void;
  onComplete?: (asset: MediaAsset | File) => void;
  onError?: (error: UploadError) => void;
  chunkSizeMb?: number; // defaults to CHUNK_SIZE_MB from server config
}
```

### `ImageUpload` Props

| Prop          | Type                           | Description                                        |
| ------------- | ------------------------------ | -------------------------------------------------- |
| `accept`      | `string[]`                     | MIME type allowlist override                       |
| `aspectRatio` | `number \| undefined`          | Lock crop to aspect ratio (e.g. `16/9`, `1`)       |
| `maxSizeMb`   | `number`                       | Client-side size check before upload               |
| `onComplete`  | `(asset: MediaAsset) => void`  | Called after successful upload and DB record       |
| `onError`     | `(error: UploadError) => void` | Called on MIME rejection, size exceeded, virus     |
| `multiple`    | `boolean`                      | Allow multi-file selection (uploaded sequentially) |
| `disabled`    | `boolean`                      | Read-only state                                    |

### Deployment Contexts

| Context              | How to use                                                      |
| -------------------- | --------------------------------------------------------------- |
| Profile avatar       | `<ImageUpload aspectRatio={1} onComplete={...} />`              |
| Hero image           | `<ImageUpload aspectRatio={16/9} onComplete={...} />`           |
| Video content upload | `<VideoUpload onComplete={...} />`                              |
| Document attachment  | `<FileUpload accept={['application/pdf']} onComplete={...} />`  |
| Media picker in form | `<MediaLibrary onSelect={...} />` (opens modal, returns assets) |
| Read-only gallery    | `<MediaGallery assets={assets} selectable={false} />`           |
