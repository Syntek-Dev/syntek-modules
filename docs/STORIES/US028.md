## User Story

As a developer, I want a reporting module so that I can define role-gated reports driven by aggregate DB queries and deliver them on schedule as PDF, Excel, or CSV.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a report definition specifies a `manager` role requirement
When a user without that role requests the report
Then a 403 is returned

Given a report is scheduled for Monday at 08:00
When Celery Beat triggers the report task
Then the report is generated and delivered to the configured destination (email, S3, or in-app notification)

Given a chart data query is requested
When the query runs for the second time within the cache TTL
Then results are served from Redis without hitting the database

Given an Excel export is requested
When the file is generated
Then it contains formatted headers, correct column data types, and freeze panes on row 1

Given a report definition uses a multi-step aggregation pipeline
When the pipeline runs
Then intermediate results are composable and the final output matches manually verified test data
```
