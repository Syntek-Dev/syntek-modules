# US063 — `syntek-analytics`: Privacy-First Analytics Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

---

## User Story

As a developer, I want a privacy-first analytics module so that I can track page views and custom
events via Plausible or Fathom without processing personal data, and gate all tracking behind GDPR
consent.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a user has not granted analytics consent
When any page loads
Then no analytics script is injected and no tracking requests are made

Given a user grants analytics consent
When a page view occurs
Then the page view is recorded in the configured analytics provider without PII in the payload

Given `track_event(name, properties)` is called from application code
When the event is dispatched
Then it is forwarded to the active analytics provider and a Django signal is emitted for internal subscribers

Given the analytics backend is switched from Plausible to Fathom
When the settings are updated and Django restarts
Then all subsequent events are routed to Fathom with no code changes in calling modules
```
