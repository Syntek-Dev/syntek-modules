# US063 — `syntek-analytics`: Privacy-First Analytics Module

**Last Updated**: 10/03/2026 **Version**: 0.2.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

## User Story

As a developer, I want a privacy-first analytics module so that I can track page views and custom
events via any supported analytics provider, gate all tracking behind GDPR consent, and expose
aggregate stats via GraphQL — with no PII in any tracking payload.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a user has not granted analytics consent
When any page loads
Then no analytics script is injected and no tracking requests are made

Given a user grants analytics consent
When a page view occurs
Then the page view is recorded in the configured analytics provider without PII in the payload

Given track_event(name, properties) is called from application code
When the event is dispatched
Then it is forwarded to the active analytics provider and a Django signal is emitted for internal
subscribers

Given the analytics backend is switched from Plausible to Fathom
When the settings are updated and Django restarts
Then all subsequent events are routed to Fathom with no code changes in calling modules

Given the analytics backend is set to GA4
When a page view is tracked
Then the event is forwarded to the GA4 Measurement Protocol API using the configured stream ID and
API secret, with no client-side PII

Given the analytics backend is set to Umami
When a custom event is tracked
Then it is posted to the configured Umami instance API with the site ID and no PII in the payload
```

## Configuration

```python
SYNTEK_ANALYTICS = {
    # --- Provider ---
    'BACKEND': 'plausible',           # 'plausible' | 'fathom' | 'ga4' | 'umami' | 'none'

    # --- Plausible ---
    'PLAUSIBLE_DOMAIN': '',           # e.g. 'yoursite.com'
    'PLAUSIBLE_API_URL': 'https://plausible.io',   # Self-hosted override
    'PLAUSIBLE_API_KEY': '',          # API key for server-side stats queries

    # --- Fathom ---
    'FATHOM_SITE_ID': '',
    'FATHOM_API_TOKEN': '',           # API token for server-side stats queries

    # --- GA4 (server-side Measurement Protocol only — no client JS) ---
    'GA4_MEASUREMENT_ID': '',         # e.g. 'G-XXXXXXXXXX'
    'GA4_API_SECRET': '',             # Measurement Protocol API secret
    'GA4_CLIENT_ID_HEADER': 'X-GA-Client-Id',   # Request header for anonymous client ID

    # --- Umami ---
    'UMAMI_SITE_ID': '',
    'UMAMI_API_URL': '',              # Self-hosted Umami instance URL
    'UMAMI_API_KEY': '',              # API key for server-side stats queries

    # --- Consent ---
    'REQUIRE_CONSENT': True,          # Gate all tracking behind syntek-gdpr consent check
    'CONSENT_CATEGORY': 'analytics', # GDPR consent category name (matches syntek-gdpr config)

    # --- Tenancy ---
    'TENANT_SCOPED': True,            # Each tenant can have its own analytics config override

    # --- Middleware ---
    'AUTO_PAGE_VIEWS': True,          # Middleware tracks server-side page views automatically
    'EXCLUDE_PATHS': ['/health/', '/metrics/'],   # Paths excluded from auto page view tracking
    'EXCLUDE_BOTS': True,             # Skip tracking for known bot user agents
}
```

### Provider Adapters

| Adapter            | Backend key | Page views | Custom events | Stats API                 |
| ------------------ | ----------- | ---------- | ------------- | ------------------------- |
| `PlausibleBackend` | `plausible` | ✓          | ✓             | ✓                         |
| `FathomBackend`    | `fathom`    | ✓          | ✓             | ✓                         |
| `GA4Backend`       | `ga4`       | ✓          | ✓             | ✗ (GA4 Data API separate) |
| `UmamiBackend`     | `umami`     | ✓          | ✓             | ✓                         |
| `NullBackend`      | `none`      | —          | —             | —                         |

All adapters implement the `AnalyticsBackend` protocol:

```python
class AnalyticsBackend(Protocol):
    def track_page_view(self, path: str, referrer: str, user_agent: str) -> None: ...
    def track_event(self, name: str, properties: dict) -> None: ...
    def stats(self, date_range: DateRange) -> AnalyticsStats: ...
```

## Public API

```python
from syntek_analytics import track_event, track_page_view

# Track a custom event (no PII allowed in properties)
track_event("donation_completed", {"amount_band": "10-50", "currency": "GBP"})

# Track a page view manually (auto-tracked by middleware if AUTO_PAGE_VIEWS = True)
track_page_view(request)
```

Both functions:

- Check consent gate before calling the adapter (no-op if consent not given and
  `REQUIRE_CONSENT = True`)
- Emit a `analytics_event_tracked` or `analytics_page_view_tracked` Django signal after dispatch

## GraphQL API

### Queries

| Query                                        | Returns                                                                  |
| -------------------------------------------- | ------------------------------------------------------------------------ |
| `analyticsStats(dateRange)`                  | `AnalyticsStats` — page views, unique visitors, top pages, top referrers |
| `analyticsTopEvents(dateRange, limit)`       | `[AnalyticsEvent]` — most frequent custom events                         |
| `analyticsPageViews(dateRange, granularity)` | `[AnalyticsDataPoint]` — time-series page view data                      |

All queries are permission-gated (requires `syntek_analytics.view_stats` permission).
