## User Story

As a developer, I want a route optimisation module so that I can compute an optimised visiting order
for multiple stops — for use cases such as courier deliveries, dog walker pickups, or field engineer
callouts — with live traffic data, ETA calculation, and navigation app deep-link generation, using a
configurable routing provider.

## Architecture

`syntek-routing` computes routes and stores itineraries server-side. It does not provide
turn-by-turn navigation — for in-vehicle navigation, the module generates deep-link URLs that open
Google Maps, Apple Maps, or Waze with all waypoints pre-loaded. The consuming project configures
which navigation apps to offer.

**Provider recommendations:**

- **OSRM** — open-source, self-hostable on the project's own Hetzner server, no per-call cost.
  Recommended for high-volume deployments (courier companies, logistics). Does not include live
  traffic by default but can be combined with a traffic tile overlay.
- **Google Routes Optimisation API** — includes live traffic, best ETAs, but has per-call billing.
  Recommended for lower-volume deployments where accuracy matters.
- **Mapbox Directions/Optimisation API** — good traffic data, lower cost than Google, generous free
  tier.

For live traffic display on the map (similar to Waze), use the routing provider's traffic-aware
directions endpoint. Waze does not offer a public data API — Waze, Google Maps, and Apple Maps are
accessed as navigation apps via deep link only.

## Configuration

```python
SYNTEK_ROUTING = {
    # Routing provider
    # 'osrm' (self-hosted, recommended) | 'google' | 'mapbox'
    'PROVIDER': 'osrm',

    # OSRM — self-hosted endpoint
    'OSRM_BASE_URL': env('OSRM_BASE_URL', default='http://router.project-osrm.org'),
    'OSRM_PROFILE': 'driving',   # 'driving' | 'cycling' | 'walking'

    # Google Routes / Directions
    'GOOGLE_MAPS_API_KEY': env('GOOGLE_MAPS_API_KEY', default=''),
    'GOOGLE_TRAFFIC_MODEL': 'best_guess',   # 'best_guess' | 'pessimistic' | 'optimistic'

    # Mapbox
    'MAPBOX_ACCESS_TOKEN': env('MAPBOX_ACCESS_TOKEN', default=''),

    # Route optimisation — max stops for automatic TSP optimisation
    'MAX_OPTIMISATION_STOPS': 25,

    # ETA
    'INCLUDE_TRAFFIC_IN_ETA': True,   # Requires Google or Mapbox provider
    'ETA_DEPARTURE_TIME': 'now',      # 'now' | ISO datetime string

    # Navigation app deep links — which apps to offer for handoff
    # Order determines display order in the UI
    'NAVIGATION_APPS': ['google_maps', 'apple_maps', 'waze'],
    # Options: 'google_maps' | 'apple_maps' | 'waze' | 'bing_maps'

    # Tracking integration — fires bus event when route is started/completed
    'TRACKING_ENABLED': True,
}
```

## Acceptance Criteria

```gherkin
Scenario 1: Stop reordering with optimisation
Given a list of 8 stop locations
When optimiseRoute(stops) is called
Then the stops are reordered into the shortest-distance visiting sequence using the configured provider's optimisation endpoint (or TSP algorithm for OSRM) and the optimised Route is returned with ordered waypoints

Scenario 2: Traffic-aware ETA with Google
Given PROVIDER = 'google' and INCLUDE_TRAFFIC_IN_ETA = True
When a route is computed
Then each leg's estimated_duration_seconds accounts for current live traffic conditions

Scenario 3: Provider switch to OSRM
Given PROVIDER = 'osrm'
When the same route is computed
Then the OSRM self-hosted endpoint is called and a valid route polyline is returned — provider switch requires only settings change

Scenario 4: Navigation deep links
Given a Route with 3 stops is computed
When navigationDeepLinks(routeId) is queried
Then one URL is returned for each provider in NAVIGATION_APPS, each containing all waypoints in order

Scenario 5: Navigation app filtering
Given NAVIGATION_APPS = ['google_maps', 'waze']
When navigationDeepLinks is queried
Then exactly two links are returned — no Apple Maps link

Scenario 6: Route start tracking
Given a route is started by the worker
When startRoute(routeId) mutation is called
Then a TrackingSession is created via syntek-tracking (if TRACKING_ENABLED = True) and route.started is fired on the bus

Scenario 7: Stop completion
Given a stop is marked complete
When completeStop(routeId, stopIndex) is called
Then the stop status updates, the next stop's ETA is recalculated, and route.stop_completed is fired on the bus
```
