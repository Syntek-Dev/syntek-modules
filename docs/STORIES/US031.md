# US031 — `syntek-documents`: MinIO Document Storage Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

---

## User Story

As a developer, I want a documents module so that PDFs and office files are stored in MinIO with
presigned URL access — completely separate from media (images/video) which lives in Cloudinary.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a PDF is uploaded via the documents endpoint
When the upload handler runs
Then the file is stored in MinIO and a `Document` record is created in PostgreSQL

Given a user with permission requests access to a document
When the presigned URL is generated
Then it expires after the configured TTL (default 15 minutes)

Given a user without permission requests access to a document
When the presigned URL request is made
Then a 403 is returned and no URL is generated

Given an image file (JPEG, PNG, etc.) is uploaded to the documents endpoint
When the MIME type is validated
Then the upload is rejected with 422 — images must go through `syntek-media`

Given a document is deleted
When the deletion handler runs
Then the file is removed from MinIO AND the `Document` record is removed from PostgreSQL atomically
```
