## User Story

As a developer, I want a reporting UI package so that I can display interactive charts, build
configurable reports, export data in multiple formats, and schedule recurring report delivery — all
connected to `syntek-reporting-core` and `syntek-reporting-export` via GraphQL.

## Acceptance Criteria

```gherkin
Given a LineChart is rendered with time-series data
When the data updates via useReportData
Then the chart re-renders with the new data, maintains correct axis labels and scale, and the ARIA
data table fallback reflects the updated rows

Given a user opens the ReportBuilder
When they select metrics, grouping, and a date range parameter
Then a debounced preview query fires and the chart updates to reflect the configuration

Given a user triggers a PDF export from the report UI
When the exportReport mutation completes
Then a download link is presented with a visible expiry countdown

Given a user without the required role views a report page
When the report component mounts
Then report data is not fetched, a permission-denied message is shown, and no GraphQL query fires

Given a chart is exported as PNG
When the export completes
Then a valid PNG file is downloaded with the chart rendered at 2× resolution

Given a ScheduledReportForm is submitted
When the mutation resolves
Then the new schedule appears in the scheduled reports list without a page reload
```

## Configuration

`@syntek/ui-reporting` has no `.env` configuration. All report definitions, parameters, and
permissions come from the GraphQL API (`syntek-reporting-core`). Export mutations target
`syntek-reporting-export`. No backend settings are consumed directly.

## Components

### Chart Components (Recharts wrappers)

| Component      | Description                                             |
| -------------- | ------------------------------------------------------- |
| `LineChart`    | Single or multi-series time-series line chart           |
| `BarChart`     | Vertical or horizontal bar chart; supports stacked mode |
| `AreaChart`    | Filled area chart; supports stacked mode                |
| `PieChart`     | Pie chart with optional legend                          |
| `DonutChart`   | Donut variant with centre label slot                    |
| `ScatterChart` | X/Y scatter plot with optional bubble size dimension    |

All chart components:

- Accept `data`, `xKey`, `yKey` (or `series` for multi-series), `colors`, `height`, `loading`,
  `error` props
- Render an ARIA `role="img"` wrapper with `aria-label`
- Include a hidden `<table>` data fallback for screen readers (togglable via `showDataTable` prop)
- Are responsive by default via Recharts `ResponsiveContainer`
- Support PNG and SVG export via the `onExport(format)` callback prop

### Hooks

| Hook                              | Description                                                                      |
| --------------------------------- | -------------------------------------------------------------------------------- |
| `useReportData(reportId, params)` | Fetches `reportResult` from GraphQL; returns `{ rows, loading, error, refetch }` |
| `useChart(rows, config)`          | Transforms raw report rows into Recharts-compatible series data                  |
| `useReportExport(runId)`          | Wraps `exportReport` mutation; returns `{ exportFile, trigger, loading }`        |
| `useScheduledReports(tenantId)`   | Fetches `scheduledReports`; returns CRUD helpers                                 |
| `useReportPermission(reportId)`   | Returns `{ allowed, reason }` — gates rendering before any query                 |

### Composite Components

| Component              | Description                                                                              |
| ---------------------- | ---------------------------------------------------------------------------------------- |
| `ReportBuilder`        | Full report configuration UI — metric selector, group-by, date range, chart type toggle  |
| `ReportViewer`         | Renders a named report with its default parameters; chart + data table toggle            |
| `ExportButton`         | Format picker (PDF/Excel/CSV) + download link with TTL countdown                         |
| `ScheduledReportForm`  | Cron expression builder, recipient list, format selector; create/update/delete           |
| `ReportPermissionGate` | Wraps any content; hides and shows denied message if `useReportPermission` returns false |
| `ChartExportButton`    | Triggers PNG or SVG export from a chart instance ref                                     |

### Deployment Contexts

| Context             | How to use                                                             |
| ------------------- | ---------------------------------------------------------------------- |
| Dashboard widget    | `<ReportViewer reportId="..." />` — self-contained                     |
| Report builder page | `<ReportBuilder reportId="..." onSave={...} />`                        |
| Scheduled delivery  | `<ScheduledReportForm reportId="..." />`                               |
| Embedded chart only | `const { rows } = useReportData(...)` + `<LineChart data={rows} .../>` |
| Export-only button  | `<ExportButton runId={...} />`                                         |
