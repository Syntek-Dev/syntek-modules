# US012 — `syntek-security`: Security Middleware Module

**Last Updated**: 10/03/2026 **Version**: 0.3.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

## User Story

As a developer, I want a security middleware module so that all Syntek applications have consistent,
configurable rate limiting (per-user, per-tenant, and per-IP), security headers, CORS, and IP
filtering out of the box — with every limit, header, and policy controllable from `settings.py`
without touching module source.

**Must Have**

## Configuration Contract

All behaviour is controlled via `SYNTEK_SECURITY` in `settings.py`. Validated at startup in
`AppConfig.ready()`.

```python
SYNTEK_SECURITY = {
    # --- Rate limiting ---
    # Rate limiting uses syntek-cache (Redis/Valkey) as the backend store.
    'RATE_LIMIT_ENABLED': True,
    'RATE_LIMIT_DEFAULT': '100/hour',       # str — applied to all endpoints without a specific limit
                                             #        Format: '<count>/<period>' where period is second|minute|hour|day
    'RATE_LIMIT_PER_USER': '1000/hour',     # str — per authenticated user (overrides DEFAULT for authenticated requests)
    'RATE_LIMIT_PER_TENANT': '50000/hour',  # str — aggregate across all users within a tenant
    'RATE_LIMIT_BURST': '20/minute',        # str — burst allowance applied on top of the sustained rate
    'RATE_LIMIT_HEADERS': True,             # bool — emit X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
    'RATE_LIMIT_EXEMPT_IPS': [],            # list[str] — IPs exempt from all rate limits (CIDR notation supported)
    'RATE_LIMIT_EXEMPT_USERS': [],          # list[str] — usernames exempt (e.g. health-check service accounts)

    # --- CORS ---
    'ALLOWED_ORIGINS': [],                  # list[str] — explicit origins; no wildcard for authenticated APIs
    'ALLOWED_METHODS': ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
    'ALLOWED_HEADERS': ['Authorization', 'Content-Type', 'X-CSRFToken'],
    'EXPOSE_HEADERS': ['X-RateLimit-Limit', 'X-RateLimit-Remaining', 'X-RateLimit-Reset'],
    'ALLOW_CREDENTIALS': True,              # bool

    # --- Security headers ---
    'CSP': "default-src 'self'",            # str — full Content-Security-Policy directive value
    'HSTS_MAX_AGE': 31536000,               # int (seconds) — 1 year default
    'HSTS_INCLUDE_SUBDOMAINS': True,        # bool
    'HSTS_PRELOAD': False,                  # bool
    'FRAME_OPTIONS': 'DENY',               # 'DENY' | 'SAMEORIGIN'
    'REFERRER_POLICY': 'strict-origin-when-cross-origin',  # str — Referrer-Policy header value
    'PERMISSIONS_POLICY': '',               # str — Permissions-Policy header value (empty = omitted)

    # --- Cross-Origin isolation headers (defence in depth) ---
    'COEP': '',                             # str — Cross-Origin-Embedder-Policy (empty = omitted)
                                             # Recommended: 'require-corp' for cross-origin isolation
    'COOP': '',                             # str — Cross-Origin-Opener-Policy (empty = omitted)
                                             # Recommended: 'same-origin' to prevent opener leakage
    'CORP': '',                             # str — Cross-Origin-Resource-Policy (empty = omitted)
                                             # Recommended: 'same-origin' for API-only origins

    # --- IP filtering ---
    'IP_BLOCKLIST': [],                     # list[str] — blocked IPs or CIDR ranges; checked before all middleware
    'IP_ALLOWLIST': [],                     # list[str] — if non-empty, only these IPs are allowed (whitelist mode)

    # --- Proxy trust (for Nginx + Cloudflare architecture per US076) ---
    'TRUST_PROXY': True,                    # bool — configure SECURE_PROXY_SSL_HEADER and friends
    'TRUSTED_PROXIES': ['127.0.0.1'],       # list[str] — IPs trusted as reverse proxies
}
```

**Key rules:**

- Rate limiting requires `syntek-cache` (US077) to be installed — validated at startup.
- `RATE_LIMIT_PER_TENANT` is tracked as an aggregate across all users of a tenant; individual user
  limits apply separately.
- `IP_ALLOWLIST` and `IP_BLOCKLIST` are mutually exclusive: if `IP_ALLOWLIST` is non-empty, only
  listed IPs are permitted; blocklist is ignored.
- Rate limit format: `<count>/<period>` where period is `second`, `minute`, `hour`, or `day`.
- `RATE_LIMIT_BURST` is applied as a short-window allowance and does not reset the sustained limit
  counter.

## Acceptance Criteria

```gherkin
# --- Settings validation ---

Given `RATE_LIMIT_ENABLED` is True and syntek-cache is not in INSTALLED_APPS
When `AppConfig.ready()` runs
Then `ImproperlyConfigured` is raised — rate limiting requires syntek-cache

Given `IP_ALLOWLIST` and `IP_BLOCKLIST` are both non-empty
When `AppConfig.ready()` runs
Then `ImproperlyConfigured` is raised — allowlist and blocklist are mutually exclusive

Given a rate limit string in an invalid format (e.g. '100/fortnight')
When `AppConfig.ready()` runs
Then `ImproperlyConfigured` is raised naming the invalid setting key and value

# --- Per-IP rate limiting ---

Given `RATE_LIMIT_DEFAULT` is '100/hour' and an unauthenticated client exceeds 100 requests
When the 101st request is made within the hour
Then a 429 is returned with a `Retry-After` header

# --- Per-user rate limiting ---

Given `RATE_LIMIT_PER_USER` is '1000/hour' and an authenticated user exceeds 1000 requests
When the 1001st request is made within the hour
Then a 429 is returned with a `Retry-After` header

Given `RATE_LIMIT_EXEMPT_USERS` includes 'health-check'
When the 'health-check' user sends 10000 requests
Then no 429 is returned — the user is exempt from all rate limits

# --- Per-tenant rate limiting ---

Given `RATE_LIMIT_PER_TENANT` is '50000/hour' and tenant 'acme' has 1000 users each sending 51 requests
When the tenant aggregate exceeds 50000 requests in the hour
Then subsequent requests from any user of tenant 'acme' return 429

# --- Burst allowance ---

Given `RATE_LIMIT_BURST` is '20/minute' and a user sends 20 requests in 30 seconds
When the 21st request arrives within the same minute
Then a 429 is returned — burst allowance is exhausted

# --- Rate limit headers ---

Given `RATE_LIMIT_HEADERS` is True
When any response is returned
Then X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers are present

Given `RATE_LIMIT_HEADERS` is False
When any response is returned
Then no X-RateLimit-* headers are present

# --- CORS ---

Given an origin not in `ALLOWED_ORIGINS` sends a CORS preflight
When the CORS middleware processes it
Then a 403 is returned with no CORS headers present

# --- Security headers ---

Given any response is sent by the application
When the security headers middleware runs
Then Content-Security-Policy, X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security,
and Referrer-Policy are all present with the configured values

Given `FRAME_OPTIONS` is 'SAMEORIGIN'
When a response is returned
Then X-Frame-Options is 'SAMEORIGIN'

# --- Cross-Origin isolation headers ---

Given `COEP` is 'require-corp'
When a response is returned
Then Cross-Origin-Embedder-Policy header is 'require-corp'

Given `COOP` is 'same-origin'
When a response is returned
Then Cross-Origin-Opener-Policy header is 'same-origin'

Given `CORP` is 'same-origin'
When a response is returned
Then Cross-Origin-Resource-Policy header is 'same-origin'

Given `COEP` is '' (empty string)
When a response is returned
Then Cross-Origin-Embedder-Policy header is absent from the response

Given `COOP` is '' (empty string)
When a response is returned
Then Cross-Origin-Opener-Policy header is absent from the response

Given `CORP` is '' (empty string)
When a response is returned
Then Cross-Origin-Resource-Policy header is absent from the response

# --- IP filtering ---

Given an IP address in `IP_BLOCKLIST`
When any request is made from that IP
Then a 403 is returned immediately before any view logic runs

Given `IP_ALLOWLIST` is non-empty and a request arrives from an IP not in the allowlist
When the IP middleware runs
Then a 403 is returned immediately

Given an IP in `RATE_LIMIT_EXEMPT_IPS`
When that IP sends 10000 requests
Then no 429 is returned — the IP is exempt from rate limiting but not from blocklist checks

# --- Proxy trust ---

Given `TRUST_PROXY` is True
When the Django application starts
Then SECURE_PROXY_SSL_HEADER, USE_X_FORWARDED_HOST, and SECURE_SSL_REDIRECT are configured
correctly so the application trusts the Nginx reverse proxy's TLS termination
```
