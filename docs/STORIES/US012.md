# US012 — `syntek-security`: Security Middleware Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

---

## User Story

As a developer, I want a security middleware module so that all Syntek applications have consistent,
configurable rate limiting, security headers, CORS, and IP filtering out of the box.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given `DEFAULT_RATE: 100/hour` is configured
When a client exceeds 100 requests in one hour
Then subsequent requests return 429 with a `Retry-After` header

Given an origin not in `ALLOWED_ORIGINS` sends a CORS preflight
When the CORS middleware processes it
Then a 403 is returned with no CORS headers present

Given any response is sent by the application
When the security headers middleware runs
Then `Content-Security-Policy`, `X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`, and `Referrer-Policy` are all present

Given an IP address in `IP_BLOCKLIST`
When any request is made from that IP
Then a 403 is returned immediately before any view logic runs

Given a state-changing request arrives without a CSRF token in a server-rendered context
When the CSRF middleware processes it
Then a 403 is returned
```
