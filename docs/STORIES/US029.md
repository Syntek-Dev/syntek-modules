# US029 — `syntek-gdpr`: GDPR Compliance Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

## User Story

As a developer building GDPR-compliant applications, I want a GDPR module so that subject access
requests, right-to-erasure, consent management, and data retention are handled automatically across
all installed modules.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a Subject Access Request is submitted for a data subject
When the SAR export task runs
Then all personal data for that subject across every installed `syntek-*` module is collected and exported as a structured package

Given a Right to Erasure request is submitted
When the erasure task runs
Then PII fields are anonymised across all installed modules without deleting audit chain records

Given a user's consent for a specific processing purpose is withdrawn
When any module attempts to process data under that purpose
Then the operation is rejected with a `ConsentWithdrawnError`

Given a retention policy of 12 months is set for a data category
When the retention task runs
Then records older than 12 months have PII anonymised and a retention log entry is written

Given a new `syntek-*` module is installed
When it registers its GDPR data classes
Then the SAR and erasure tasks automatically include it with no configuration change
```
