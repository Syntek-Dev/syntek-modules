## User Story

As a security engineer, I want a Rust crate providing field-level encryption and password hashing so
that sensitive data is never stored as plaintext in any database.

## Architecture Note

Encryption and decryption are performed exclusively in the **GraphQL middleware layer** (US008). The
frontend sends plaintext; the middleware encrypts before writing to the ORM and decrypts before
serialising responses. Django's ORM and PostgreSQL only ever see ciphertext. This crate provides the
cryptographic primitives that the middleware calls â€” it has no opinion about transport or
persistence.

## Acceptance Criteria

```gherkin
Given a plaintext value and a key
When `encrypt_field` is called
Then it returns a base64-encoded `nonce || ciphertext || tag` string

Given a valid ciphertext and matching key
When `decrypt_field` is called
Then it returns the original plaintext

Given a tampered ciphertext
When `decrypt_field` is called
Then it returns a `DecryptionError` and no plaintext is exposed

Given a plaintext password
When `hash_password` is called
Then it returns a valid Argon2id PHC string with m=65536, t=3, p=4

Given a raw password and its Argon2id hash
When `verify_password` is called
Then it returns `true` for the correct password and `false` for any other

Given any encryption or hashing operation completes
When the function returns
Then all sensitive in-memory values are zeroised via the `zeroize` crate

Given `encrypt_field` is called with model_name="User" and field_name="email"
When the same ciphertext is passed to `decrypt_field` with model_name="Order" and field_name="email"
Then it returns a `DecryptionError` (AAD mismatch prevents cross-model substitution)

Given `encrypt_field` is called 10,000 times
When the nonces are compared
Then no two nonces are identical

Given a list of (field_name, plaintext) pairs and a key
When `encrypt_fields_batch` is called with a model_name
Then it returns a Vec of base64-encoded ciphertexts in the same order, each with its own
unique nonce and its own AAD (model_name + field_name)

Given a list of (field_name, ciphertext) pairs and a key
When `decrypt_fields_batch` is called with a model_name
Then it returns the original plaintext values in the same order

Given `encrypt_fields_batch` is called and any single field fails to encrypt
When the error is returned
Then no partial results are returned â€” the entire batch fails atomically with a `BatchError`
listing which field failed

Given `decrypt_fields_batch` is called and any single field fails to decrypt
When the error is returned
Then no partial results are returned â€” the entire batch fails atomically with a `BatchError`
listing which field failed
```
