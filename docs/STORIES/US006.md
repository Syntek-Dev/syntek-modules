## User Story

As a security engineer, I want a Rust crate providing field-level encryption and password hashing so that sensitive data is never stored as plaintext in any database.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a plaintext value and a key
When `encrypt_field` is called
Then it returns a base64-encoded `nonce || ciphertext || tag` string

Given a valid ciphertext and matching key
When `decrypt_field` is called
Then it returns the original plaintext

Given a tampered ciphertext
When `decrypt_field` is called
Then it returns a `DecryptionError` and no plaintext is exposed

Given a plaintext password
When `hash_password` is called
Then it returns a valid Argon2id PHC string with m=65536, t=3, p=4

Given a raw password and its Argon2id hash
When `verify_password` is called
Then it returns `true` for the correct password and `false` for any other

Given any encryption or hashing operation completes
When the function returns
Then all sensitive in-memory values are zeroised via the `zeroize` crate

Given `encrypt_field` is called with model_name="User" and field_name="email"
When the same ciphertext is passed to `decrypt_field` with model_name="Order" and field_name="email"
Then it returns a `DecryptionError` (AAD mismatch prevents cross-model substitution)

Given `encrypt_field` is called 10,000 times
When the nonces are compared
Then no two nonces are identical
```
