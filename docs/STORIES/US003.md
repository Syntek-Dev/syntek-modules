# US003 — Design Token System

**Last Updated**: 08/03/2026 **Version**: 0.1.1 **Status**: Completed **Epic**: Epic 1 — Foundation
& Workspace

---

## User Story

As a frontend developer, I want a canonical set of design tokens so that all UI packages use
consistent, overridable values for colour, typography, spacing, and shadow without hardcoding.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the tokens package is installed
When I reference `var(--color-primary)` in any component
Then the value resolves to the default token value

Given a consuming project overrides `--color-primary` at :root
When the component renders
Then it uses the overridden value without any code change to the component

Given a new component is added to any UI package
When I run the linter
Then any hardcoded colour, spacing, or font-size value causes a lint failure

Given the tokens package is built
When I import from `@syntek/tokens`
Then TypeScript constants for all tokens are available with correct types

Given a token value is referenced in a NativeWind class
When the mobile component renders
Then the correct design token value is applied on both iOS and Android

Given the font tokens are defined in `tokens.css`
When a consuming project loads the page
Then the correct font family, weight, and fallback stack are applied from `--font-sans`, `--font-serif`, and `--font-mono` without any hardcoded font names in component code
```
