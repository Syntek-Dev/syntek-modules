# US003 — Design Token System

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 1 — Foundation & Workspace

---

## User Story

As a frontend developer, I want a canonical set of design tokens so that all UI packages use consistent, overridable values for colour, typography, spacing, and shadow without hardcoding.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the tokens package is installed
When I reference `var(--color-primary)` in any component
Then the value resolves to the default token value

Given a consuming project overrides `--color-primary` at :root
When the component renders
Then it uses the overridden value without any code change to the component

Given a new component is added to any UI package
When I run the linter
Then any hardcoded colour, spacing, or font-size value causes a lint failure

Given the tokens package is built
When I import from `@syntek/tokens`
Then TypeScript constants for all tokens are available with correct types

Given a token value is referenced in a NativeWind class
When the mobile component renders
Then the correct design token value is applied on both iOS and Android

Given the font tokens are defined in `tokens.css`
When a consuming project loads the page
Then the correct font family, weight, and fallback stack are applied from `--font-sans`, `--font-serif`, and `--font-mono` without any hardcoded font names in component code
```

## Tasks

- [ ] **Red phase:** write failing tests — token variable resolution, TS constant exports, lint enforcement, NativeWind integration, font token resolution
- [ ] Create `shared/tokens/` package (`@syntek/tokens`)
- [ ] Define `tokens.css`:
  - Colour: full palette + semantic aliases (primary, secondary, destructive, muted, surface, background, foreground, border)
  - Spacing: 4px base scale (4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96, 128)
  - Typography scale: font-size (xs through 5xl), line-height, letter-spacing, font-weight tokens
  - Font families: `--font-sans`, `--font-serif`, `--font-mono` — default to system stack; override with `next/font` values in consuming Next.js projects
  - Breakpoints: documented as CSS custom property references (sm 640px, md 768px, lg 1024px, xl 1280px, 2xl 1536px) and as TypeScript constants for use in JS
  - Border radius, shadow, z-index, transition duration, and easing tokens
- [ ] Define `tokens.ts`: TypeScript constants mirroring all CSS custom properties including breakpoint values
- [ ] Document `next/font` integration pattern — how consuming Next.js projects load custom fonts (e.g. Google Fonts via `next/font/google`) and map them to `--font-sans` / `--font-serif` at `:root`
- [ ] Configure ESLint rule to reject hardcoded colour, spacing, font, and breakpoint values in all UI packages
- [ ] Configure NativeWind preset consuming token values
- [ ] Document token override pattern for `syntek-platform` integration
