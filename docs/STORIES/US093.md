## User Story

As a developer, I want a bulk export module so that I can export any configured model to CSV, Excel,
or PDF asynchronously with custom field selection, export templates, scheduled recurring exports,
and optional email delivery ‚Äî all configurable without touching module internals.

## Acceptance Criteria

```gherkin
Given an export is started for a model with 100,000 rows
When the Celery export task runs
Then it streams the output in chunks without loading all rows into memory at once, and progress is
available via exportJobStatus

Given an export job completes
When the user requests the download URL
Then they receive a pre-signed URL valid for SYNTEK_BULK_EXPORT['DOWNLOAD_URL_TTL'] seconds

Given an export template is saved for a model
When a new export is started with that template name
Then the template's field list, ordering, and filters are applied automatically

Given a scheduled export is configured for a model
When the schedule fires
Then the export runs automatically and the output file is delivered to the configured destination
(email, storage path, or webhook URL)

Given an export is scheduled with email delivery
When the job completes
Then the output file is attached to an email and sent via syntek-notifications-email to the
configured recipient list

Given a project configures SYNTEK_BULK_EXPORT with a custom queryset hook for a model
When the export runs
Then only rows returned by the hook are included ‚Äî the consuming project controls the data scope

Given an export is requested with a field filter applied
When the job runs
Then only the filtered subset of rows and the configured subset of columns appear in the output

Given a PDF export is requested for a model with no PDF_TEMPLATE_HOOKS configured
When the job runs
Then a default paginated table layout is rendered using WeasyPrint with the model's field labels
as column headers

Given a PDF export is requested for a model with a PDF_TEMPLATE_HOOKS entry configured
When the job runs
Then the consuming project's Jinja2 HTML template is rendered with the queryset data and converted
to PDF via WeasyPrint ‚Äî the output reflects the custom layout, not the default table

Given a PDF export job completes
When the user requests the download URL
Then they receive a pre-signed URL for a .pdf file valid for SYNTEK_BULK_EXPORT['DOWNLOAD_URL_TTL']
seconds
```

## Configuration

All behaviour is controlled via `SYNTEK_BULK_EXPORT` in Django `settings.py`. Nothing is hardcoded.
Consuming projects configure which models are exportable, default field lists, filters, and delivery
destinations.

```python
SYNTEK_BULK_EXPORT = {
    # Models available for export ‚Äî key = slug; value = dotted model path
    'EXPORTABLE_MODELS': {
        'contacts': 'myapp.models.Contact',
        'orders': 'myapp.models.Order',
    },

    # Chunk size for streaming export (rows per iteration)
    'CHUNK_SIZE': 1000,

    # Supported output formats
    'SUPPORTED_FORMATS': ['csv', 'excel', 'pdf'],

    # PDF rendering settings
    'PDF_PAGE_SIZE': 'A4',          # 'A4' | 'Letter' | 'A3'
    'PDF_ORIENTATION': 'portrait',  # 'portrait' | 'landscape'

    # Custom Jinja2 HTML template per model ‚Äî dotted path to a .html file or callable
    # callable signature: (queryset, fields, context) -> str (rendered HTML)
    # If absent for a model, the default paginated table template is used
    'PDF_TEMPLATE_HOOKS': {
        'orders': 'myapp/exports/orders_pdf.html',
    },

    # Pre-signed download URL TTL in seconds
    'DOWNLOAD_URL_TTL': 3600,

    # Storage backend for completed export files ‚Äî 'local', 's3', 'minio'
    'STORAGE_BACKEND': 'minio',

    # Custom queryset hooks per model ‚Äî dotted path to callable(user, tenant) -> QuerySet
    'QUERYSET_HOOKS': {
        'contacts': 'myapp.exports.contacts_queryset',
    },

    # Custom field label overrides per model ‚Äî maps model field name to export column header
    'FIELD_LABELS': {
        'contacts': {
            'email': 'Email Address',
            'created_at': 'Date Added',
        },
    },

    # Scheduled exports ‚Äî list of export job definitions
    'SCHEDULED_EXPORTS': [
        {
            'name': 'weekly-contacts',
            'model_slug': 'contacts',
            'schedule': 'every_monday_09:00',  # cron or named schedule
            'format': 'csv',
            'delivery': 'email',
            'recipients': ['admin@example.com'],
        },
    ],
}
```

## Data Model

### `ExportJob`

| Field           | Type      | Notes                                             |
| --------------- | --------- | ------------------------------------------------- |
| `id`            | UUID      | Primary key                                       |
| `model_slug`    | CharField | Maps to `EXPORTABLE_MODELS` key                   |
| `status`        | CharField | `pending` / `processing` / `completed` / `failed` |
| `format`        | CharField | `csv` / `excel` / `pdf`                           |
| `total_rows`    | IntField  | Populated after queryset count                    |
| `exported`      | IntField  | Rows written so far                               |
| `filters`       | JSONField | Query filters applied to this job                 |
| `fields`        | JSONField | Ordered list of field names included in output    |
| `template_name` | CharField | Null unless a named template was used             |
| `file_path`     | CharField | Storage path of completed output file             |
| `requested_by`  | FK (null) | Auth user ‚Äî null for scheduled exports            |
| `requested_at`  | datetime  |                                                   |
| `completed_at`  | datetime  | Null until complete                               |
| `is_scheduled`  | bool      | True when triggered by a scheduled config entry   |
| `tenant`        | FK (null) |                                                   |

### `ExportTemplate`

Saved named export configuration ‚Äî field list, ordering, filters ‚Äî for repeat use.

| Field        | Type      | Notes                           |
| ------------ | --------- | ------------------------------- |
| `name`       | CharField | Unique per (model_slug, tenant) |
| `model_slug` | CharField |                                 |
| `fields`     | JSONField | Ordered list of field names     |
| `filters`    | JSONField | Default query filters           |
| `ordering`   | JSONField | Default ordering fields         |
| `created_by` | FK        | Auth user                       |
| `tenant`     | FK (null) |                                 |
