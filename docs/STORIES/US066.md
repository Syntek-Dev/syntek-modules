## User Story

As a developer, I want a fully configurable inventory module so that I can track stock levels across
multiple locations, record every movement with an immutable audit trail, support barcode and QR code
scanning, dispatch low-stock alerts, and manage reservations ‚Äî all through a clean GraphQL API.

## Acceptance Criteria

```gherkin
Given a stock adjustment is saved
When the movement is recorded
Then a StockMovement entry is created with reason code, reference, operator, and resulting balance;
  stock never goes below zero unless ALLOW_NEGATIVE_STOCK is True; all movements are immutable ‚Äî
  corrections are made via counter-movement entries only

Given stock for an item at a location reaches or falls below the configured low-stock threshold
When the threshold check runs after each movement
Then a LowStockAlert record is created and a notification is dispatched via syntek-notifications
  (US019); duplicate alerts are suppressed until stock rises above the threshold and falls again

Given a stock transfer is initiated between two locations
When the transfer is processed
Then the source location's quantity decreases and the destination's quantity increases atomically
  within a single DB transaction ‚Äî no partial state is possible; if either write fails the
  transaction rolls back completely

Given a stock reservation is created on order placement
When the order is cancelled
Then the reserved quantity is released back to available stock atomically; the reservation record
  transitions to released status

Given an InventoryItem is created with BARCODE_ENABLED = True
When generateBarcode(itemId, format) mutation is called
Then a barcode image is generated in the requested format (EAN-13, Code128, or QR) and stored
  via syntek-files (US031) or locally; the item's barcode_value field is set

Given a barcode scan is submitted via scanBarcode mutation
When the scan is processed
Then the matching InventoryItem is returned; if no match exists a BarcodeNotFound error is raised;
  the scan event is recorded in InventoryScanLog
```

## Configuration

```python
SYNTEK_INVENTORY = {
    # General
    'ALLOW_NEGATIVE_STOCK': False,        # permit stock levels below zero
    'DEFAULT_UNIT': 'unit',               # display label for quantity unit
    'TRACK_COST_PRICE': True,             # record cost price on StockMovement
    'CURRENCY': 'GBP',

    # Locations
    'MULTI_LOCATION_ENABLED': True,
    'DEFAULT_LOCATION_NAME': 'Default',

    # Barcode & QR
    'BARCODE_ENABLED': True,
    'SUPPORTED_BARCODE_FORMATS': ['ean13', 'code128', 'qr'],
    'BARCODE_STORAGE_BACKEND': 'files',   # files (US031) | local | none
    'BARCODE_LOCAL_PATH': '',             # only used when BARCODE_STORAGE_BACKEND = local

    # Categories & SKU
    'CATEGORIES_ENABLED': True,
    'SKU_AUTO_GENERATE': False,
    'SKU_PREFIX': '',                     # prefix for auto-generated SKUs

    # Low-stock alerts
    'LOW_STOCK_ALERTS_ENABLED': True,
    'DEFAULT_LOW_STOCK_THRESHOLD': 5,     # items; overridable per InventoryItem
    'ALERT_NOTIFICATION_CHANNEL': 'email', # email | sms | push | all

    # Reservations
    'RESERVATIONS_ENABLED': True,
    'RESERVATION_EXPIRY_HOURS': 24,       # 0 = no expiry

    # GDPR / Audit
    'MOVEMENT_RETENTION_DAYS': 0,         # 0 = retain indefinitely

    # Tenant
    'TENANT_SCOPED': True,
}
```

## GraphQL API

### Queries

| Query                                                              | Returns                                       |
| ------------------------------------------------------------------ | --------------------------------------------- |
| `inventoryItems(category, location, search, page, pageSize)`       | Paginated item list with current stock levels |
| `inventoryItem(id)`                                                | Single item with stock levels per location    |
| `stockMovements(itemId, locationId, movementType, page, pageSize)` | Paginated movement history                    |
| `lowStockItems(locationId)`                                        | Items at or below their low-stock threshold   |
| `inventoryLocations`                                               | All locations for tenant                      |
| `inventoryCategories`                                              | Category tree                                 |
| `stockReservations(reference)`                                     | Reservations for a given order reference      |

### Mutations

| Mutation                                                            | Description                              |
| ------------------------------------------------------------------- | ---------------------------------------- |
| `adjustStock(itemId, locationId, quantityDelta, reasonCode, notes)` | Manual stock adjustment                  |
| `receiveStock(itemId, locationId, quantity, costPrice, reference)`  | Record goods receipt                     |
| `transferStock(itemId, fromLocationId, toLocationId, quantity)`     | Atomic inter-location transfer           |
| `reserveStock(itemId, locationId, quantity, reference, expiresAt)`  | Reserve stock for an order               |
| `releaseReservation(reservationId)`                                 | Release reserved stock back to available |
| `generateBarcode(itemId, format)`                                   | Generate and store barcode/QR image      |
| `scanBarcode(barcodeValue, locationId)`                             | Lookup item by barcode and log scan      |
| `createInventoryItem(input)`                                        | Create a new inventory item              |
| `updateInventoryItem(id, input)`                                    | Update item metadata                     |
| `createInventoryLocation(input)`                                    | Create a new stock location              |
