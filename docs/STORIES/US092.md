# US092 — `@syntek/ui-legal`: Legal Documents & Acceptance UI

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

## User Story

As a developer, I want a legal documents UI package so that I can display versioned T&Cs, Privacy
Policies, and other legal documents, prompt users to re-accept when new versions are published, and
integrate acceptance flows into registration and guarded routes — all configurable without touching
component internals.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the current T&Cs version requires acceptance
When an authenticated user navigates to a guarded route
Then AcceptanceGate intercepts the route and renders LegalAcceptanceModal before allowing access

Given the user accepts the T&Cs in LegalAcceptanceModal
When the acceptance mutation completes
Then the modal closes, the guarded route renders, and the acceptance is persisted to the backend

Given a user visits the legal document page for the Privacy Policy
When the page renders
Then the current published version content is displayed with effective date and version number

Given the RegisterForm has legalDocuments configured
When the registration form renders
Then a checkbox for each configured document is shown; the form cannot submit until all required
documents are checked

Given a user views their legal acceptance history
When the AcceptanceHistory component renders
Then all past acceptances are listed with document type, version, and accepted_at date
```

## Components

### `LegalDocumentPage`

Standalone full-page component for displaying a legal document. Fetches the current active version
via `useLegalDocument(slug)`.

```tsx
<LegalDocumentPage slug="terms" />
<LegalDocumentPage slug="privacy" />
```

### `LegalDocumentDisplay`

Renders the HTML content of a legal document version. Used inside `LegalDocumentPage` and
`LegalAcceptanceModal`. Content is sanitised server-side; this component renders it with
`dangerouslySetInnerHTML` (safe — content is sanitised by `syntek-security-validator` on publish).

### `LegalAcceptanceModal`

Prompted re-acceptance dialog. Triggered automatically by `AcceptanceGate` or called explicitly.
Shows document content (scrollable), accept/decline buttons. Decline either logs out or navigates to
a configurable `onDecline` route.

```tsx
<LegalAcceptanceModal
  documentType="terms"
  versionId={pendingVersionId}
  onAccepted={() => router.replace(intendedPath)}
  onDeclined={() => router.replace("/logout")}
/>
```

### `AcceptanceGate`

HOC / wrapper component that checks whether the current user has accepted all required document
versions. If a pending acceptance exists, renders `LegalAcceptanceModal`. Otherwise renders
`children`. Uses `usePendingAcceptances()` hook.

```tsx
<AcceptanceGate requiredDocuments={["terms", "privacy"]}>
  <ProtectedPageContent />
</AcceptanceGate>
```

### `LegalCheckboxes`

Renders a set of acceptance checkboxes for use in registration forms. Each checkbox links to the
document page (opens in new tab). Exports `legalCheckboxSchema` for Zod validation.

```tsx
<LegalCheckboxes documents={["terms", "privacy"]} value={legalValues} onChange={setLegalValues} />
```

### `AcceptanceHistory`

Read-only list of a user's past legal doc acceptances. Used in the Privacy Centre.

```tsx
<AcceptanceHistory userId={currentUser.id} />
```

## Hooks

| Hook                            | Returns                                              |
| ------------------------------- | ---------------------------------------------------- |
| `useLegalDocument(slug)`        | Current active `LegalDocumentVersion` content        |
| `usePendingAcceptances()`       | List of document types the user has not yet accepted |
| `useAcceptDocument()`           | `[acceptDocument, { loading, error }]` mutation hook |
| `useAcceptanceHistory(userId?)` | List of `UserLegalAcceptance` records                |

## Integration Points

### Registration (`@syntek/ui-auth`)

`SyntekAuthProvider` accepts `legalDocuments` prop — an array of document slugs that must be
accepted at registration. `RegisterForm` renders `LegalCheckboxes` automatically when this prop is
set. No `@syntek/ui-legal` import required in the consuming app — `@syntek/ui-auth` has a peer
dependency on `@syntek/ui-legal`.

```tsx
<SyntekAuthProvider
  legalDocuments={['terms', 'privacy']}
  // ...
>
```

### Privacy Centre (`@syntek/ui-gdpr`)

`PrivacyCentre` includes an `AcceptanceHistory` tab when `@syntek/ui-legal` is installed.

### Route guarding (Next.js)

`withLegalGate` middleware wrapper for Next.js `middleware.ts` — checks acceptance server-side via
the GraphQL API and redirects to an acceptance page if re-acceptance is required.
