## User Story

As a developer, I want a pre-configured Celery task queue so that I can dispatch background tasks to
priority lanes with automatic retries and dead-letter routing.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a task is dispatched to the `critical` queue
When the Celery workers are running
Then it is processed before any tasks in `high`, `default`, or `low` queues

Given a task fails and has remaining retries configured
When the Celery worker processes the failure
Then it retries with exponential back-off at the configured base delay

Given a task exhausts all retries
When the final failure occurs
Then it is routed to the dead-letter queue with full context preserved for manual replay

Given `syntek-logging` is also installed
When a task runs
Then the originating HTTP request ID is present in all task log output

Given a periodic task is defined
When Celery Beat triggers it at the scheduled time
Then the task executes and its result is recorded
```
