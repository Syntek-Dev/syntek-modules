## User Story

As a developer, I want a geospatial location module so that I can store structured address and
coordinate data against any model, auto-geocode addresses using a configurable provider, run
proximity and bounding-box queries, and provide address autocomplete — without being tied to any
specific mapping provider.

## Architecture

`syntek-locations-core` stores and queries geospatial data. It does not render maps (that is
`@syntek/ui-maps` / `@syntek/mobile-maps`) and does not compute routes (that is `syntek-routing`).
Geocoding is delegated to the provider configured in `SYNTEK_LOCATIONS['GEOCODING_PROVIDER']`.
Geospatial queries use PostgreSQL's `earthdistance` extension (no PostGIS required) for proximity,
with optional PostGIS `PointField` support when `SYNTEK_LOCATIONS['USE_POSTGIS'] = True`. When
`SYNTEK_LOCATIONS['USE_TIMESCALEDB'] = True` and `syntek-timescaledb` is installed, a
`LocationHistory` model (recording each geocode result and coordinate change) is converted to a
hypertable — useful for high-volume applications that track many location updates over time.

## Configuration

```python
SYNTEK_LOCATIONS = {
    # Geocoding provider — determines which service auto-geocodes addresses
    # 'google' | 'mapbox' | 'nominatim' (OpenStreetMap, free, self-hostable)
    'GEOCODING_PROVIDER': 'nominatim',

    # Provider API keys (only the active provider's key is required)
    'GOOGLE_MAPS_API_KEY': env('GOOGLE_MAPS_API_KEY', default=''),
    'MAPBOX_ACCESS_TOKEN': env('MAPBOX_ACCESS_TOKEN', default=''),
    'NOMINATIM_USER_AGENT': env('NOMINATIM_USER_AGENT', default='syntek-app'),
    'NOMINATIM_BASE_URL': 'https://nominatim.openstreetmap.org',   # Override for self-hosted

    # Database — use PostGIS PointField if available; falls back to decimal lat/lng
    'USE_POSTGIS': False,

    # Auto-geocode on save — geocodes when address is saved without coordinates
    'AUTO_GEOCODE_ON_SAVE': True,

    # UK postcode lookup — delegates to syntek-geo if installed
    'UK_POSTCODE_LOOKUP_ENABLED': True,

    # TimescaleDB — optional; useful when location history tables accumulate at high volume
    # Requires syntek-timescaledb (US118)
    # Primary candidate: a LocationHistory model (audit trail of location changes)
    # The core Location model itself is not time-series — no benefit from hypertable
    'USE_TIMESCALEDB': False,

    # Proximity query — default radius in kilometres
    'DEFAULT_PROXIMITY_RADIUS_KM': 10,

    # Address autocomplete — provider for typeahead suggestions
    # Uses same provider as GEOCODING_PROVIDER unless overridden
    'AUTOCOMPLETE_PROVIDER': None,   # None = use GEOCODING_PROVIDER
    'AUTOCOMPLETE_MIN_CHARS': 3,
    'AUTOCOMPLETE_MAX_RESULTS': 5,
}
```

## Acceptance Criteria

```gherkin
Scenario 1: Auto-geocode with Nominatim provider
Given GEOCODING_PROVIDER = 'nominatim' and a Location is saved with an address but no coordinates
When the post-save signal fires
Then the address is geocoded via Nominatim and the resulting latitude and longitude are stored on the record

Scenario 2: Provider switching
Given GEOCODING_PROVIDER = 'google'
When the same save occurs
Then the geocoding request is made to Google Maps Geocoding API instead — provider switch requires no code change, only settings change

Scenario 3: Proximity query
Given nearbyLocations(lat, lng, radius_km) is queried
When the query executes
Then only locations within radius_km are returned, ordered by ascending distance, using earthdistance or PostGIS depending on USE_POSTGIS

Scenario 4: Bounding-box query
Given a bounding-box query locationsInBounds(sw_lat, sw_lng, ne_lat, ne_lng) is made
When the query executes
Then only locations within the bounding box are returned

Scenario 5: Address autocomplete
Given addressAutocomplete(query) is called with at least AUTOCOMPLETE_MIN_CHARS characters
When the query executes
Then up to AUTOCOMPLETE_MAX_RESULTS address suggestions are returned from the configured provider

Scenario 6: Coordinate validation
Given an invalid latitude (outside −90 to 90) is supplied
When the Location is saved
Then a ValidationError is raised before any DB write

Scenario 7: Cascade delete
Given a Location is attached to a model via GenericForeignKey and the parent model is deleted
When the cascade runs
Then the Location record is also deleted
```
