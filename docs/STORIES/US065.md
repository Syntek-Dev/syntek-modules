## User Story

As a developer, I want a locations module so that I can store address and coordinate data against
any model, run proximity-based queries, and auto-geocode addresses on save.

## MoSCoW Priority

**Could Have**

## Acceptance Criteria

```gherkin
Given a Location record is saved with an address but no coordinates
When the post-save signal fires
Then `syntek-geo` geocodes the address and the resulting coordinates are stored on the record

Given a `nearbyLocations(lat, lng, radius)` query is made
When the query executes
Then only locations within the given radius are returned, ordered by ascending distance from the reference point

Given a Location is attached to a model via generic FK
When that model record is deleted
Then the associated Location record is also deleted via cascade

Given an invalid latitude or longitude is supplied
When the Location is saved
Then a `ValidationError` is raised with a descriptive message before any DB write
```
