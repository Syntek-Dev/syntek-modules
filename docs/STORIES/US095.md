# US095 — `syntek-payments-stripe`: Stripe Adapter

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

## User Story

As a developer, I want a Stripe-specific payments adapter so that I can process card payments, 3D
Secure, BACS Direct Debit, SEPA, and Stripe Billing subscriptions through the syntek-payments-core
provider interface.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given Stripe adapter is configured
When a payment intent is created via the core API
Then a Stripe PaymentIntent is created and the client secret is returned for front-end confirmation

Given a card requires 3D Secure
When the intent status is requires_action
Then the client secret is returned for the front-end to complete stripe.confirmCardPayment

Given a Stripe webhook arrives
When HMAC-SHA256 signature is verified using WEBHOOK_SECRET
Then the event is routed to the correct handler and transaction state is updated

Given a Stripe webhook arrives with an invalid signature
When verification runs
Then the request is rejected with 401

Given a Stripe subscription is created
When the first invoice is paid
Then a Subscription record is created with status active and billing dates set
```
