## User Story

As a developer, I want a feedback and surveys UI package so that I can render dynamic survey forms
with conditional question logic, provide an admin survey builder, display analytics dashboards, and
wire everything to `syntek-feedback` via GraphQL.

## Acceptance Criteria

```gherkin
Given a survey schema is fetched from the API
When SurveyForm renders
Then each question is displayed with the correct input component — RatingScale, NPSWidget,
  SingleChoice, MultiChoice, FreeText, BooleanToggle, DatePicker, or NumberInput

Given a conditional question has its display condition unmet
When the dependent question's answer changes
Then the conditional question appears or disappears immediately without a network request;
  the visibility engine evaluates conditions client-side from the schema

Given a user completes all required questions and submits
When submitSurveyResponse mutation succeeds
Then a thank-you confirmation screen replaces the form and the submit button is permanently
  disabled so the user cannot resubmit without a page reload

Given the survey spans multiple pages
When the user advances to the next page
Then all required fields on the current page are validated before navigation is permitted;
  page indicator shows current position and total

Given an admin opens the survey builder
When SurveyBuilder renders
Then questions can be added, reordered, edited, and removed; question type and conditional
  logic can be configured inline; a live preview panel reflects every change

Given surveyAnalytics(surveyId) data is available
When SurveyDashboard renders
Then per-question breakdown, response count, completion rate, and NPS score (if applicable)
  are all displayed with accessible chart or table representations

Given NPSWidget is rendered on a page
When the page loads
Then the widget does not appear immediately — it waits for the configured displayDelay before
  sliding in; the user can dismiss it and it does not reappear in the same session
```

## Components

| Component         | Description                                                                                                                          |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `SurveyForm`      | Root form renderer — maps schema to question components; handles multi-page navigation, conditional visibility, and submission state |
| `SurveyBuilder`   | Admin drag-and-drop survey editor — add/reorder/edit/remove questions with live preview pane                                         |
| `SurveyDashboard` | Analytics view — per-question breakdown, NPS gauge, completion funnel, response count                                                |
| `NPSWidget`       | Floating NPS prompt — configurable display delay, dismissible, session-scoped suppression                                            |
| `SurveyThankYou`  | Post-submission confirmation screen with configurable message                                                                        |
| `RatingScale`     | 1–N star or number scale input                                                                                                       |
| `NPSInput`        | 0–10 NPS score input with promoter/passive/detractor colouring                                                                       |
| `SingleChoice`    | Radio button group with optional "other" free text                                                                                   |
| `MultiChoice`     | Checkbox group with optional "other" free text                                                                                       |
| `FreeText`        | Textarea with character counter and optional rich text                                                                               |
| `BooleanToggle`   | Yes/No toggle                                                                                                                        |
| `PageIndicator`   | Multi-page progress indicator                                                                                                        |
| `QuestionCard`    | Wrapper for each question with label, help text, required marker, and error state                                                    |

## Hooks

| Hook                                        | Returns                                                                              |
| ------------------------------------------- | ------------------------------------------------------------------------------------ |
| `useFeedback(surveyId)`                     | `{ schema, loading, error }` — fetches survey schema                                 |
| `useSurveyResponse(surveyId)`               | `{ submit, loading, error, isSubmitted }` — handles submission                       |
| `useSurveyAnalytics(surveyId)`              | `{ analytics, loading, error }` — aggregated analytics data                          |
| `useSurveyBuilder(surveyId)`                | `{ survey, updateQuestion, addQuestion, removeQuestion, reorder, publish, loading }` |
| `useConditionalVisibility(schema, answers)` | `{ visibleQuestions }` — evaluates display conditions from schema client-side        |
| `useNPSWidget(surveyId, options)`           | `{ isVisible, dismiss }` — manages delay and session suppression                     |
