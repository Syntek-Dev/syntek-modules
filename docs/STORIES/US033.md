# US033 — `syntek-integrations`: Third-Party Integrations Framework

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 5 — Backend Compliance & Media

---

## User Story

As a developer, I want an integrations framework so that I can connect external services (EspoCRM, CalDav) via OAuth with per-tenant encrypted credential storage and automatic token refresh.

## MoSCoW Priority

**Could Have**

## Acceptance Criteria

```gherkin
Given an integration is configured for a tenant and the OAuth flow completes
When the callback is processed
Then the access token is stored encrypted via `syntek-crypto` and linked to the tenant

Given a stored access token has expired
When an integration request is made
Then the token is refreshed automatically using the stored refresh token before the request proceeds

Given a tenant disconnects an integration
When the disconnect mutation is called
Then the stored credentials are deleted and the integration is marked inactive

Given two tenants configure the same integration type
When each tenant's integration runs
Then they use their own credentials and cannot access each other's data
```
