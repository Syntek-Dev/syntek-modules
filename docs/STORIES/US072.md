## User Story

As a developer, I want a booking and calendar UI package so that I can embed a configurable booking
flow and calendar display into any project — adapting the UI for restaurants, vets, service
providers, or any bookable resource — without any business-type assumptions in the components.

## Architecture

All components receive `resourceType` and `bookingMode` props derived from the backend
configuration, allowing the same `BookingForm` to render "Select a table" for a restaurant or
"Choose a staff member" for a vet. Labels are fully overridable. Calendar export buttons render only
the providers listed in `calendarExportLinks` from the GraphQL response — the frontend never
hardcodes which providers to show.

## Acceptance Criteria

```gherkin
Scenario: Hard booking mode — exact time slots
Given bookingMode = 'hard'
When AvailabilityGrid renders
Then exact time slots are shown as clickable cells with booked/available state

Scenario: Soft booking mode — arrival windows
Given bookingMode = 'soft'
When AvailabilityGrid renders
Then arrival windows are shown as selectable ranges (e.g. "10:00 – 11:00") rather than exact times

Scenario: Slot selection pre-fills booking form
Given a slot is selected in AvailabilityGrid
When the cell is clicked
Then BookingForm opens pre-filled with the resource, date, and time — and the form adapts its resource label from the resourceType prop

Scenario: Calendar export with dynamic providers
Given a booking is confirmed
When the BookingConfirmation screen renders
Then "Add to Calendar" buttons are displayed — one for each provider returned in calendarExportLinks — no hardcoded provider list

Scenario: Label customisation
Given a project provides custom labels prop
When any component renders
Then all user-facing text (slot labels, button text, resource names) uses the provided overrides

Scenario: Cancelled appointment styling
Given an appointment has cancelled status
When the CalendarView renders
Then cancelled appointments are visually distinct (muted style) from confirmed and pending appointments

Scenario: Timezone-aware slot display
Given the user's timezone differs from the resource's timezone
When AvailabilityGrid renders
Then slot times are displayed in the user's local timezone with a timezone label
```
