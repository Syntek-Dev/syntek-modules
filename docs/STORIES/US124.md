# US124 — `syntek-error`: Error Handling & Custom Error Pages Module

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

---

## User Story

As a developer, I want a configurable error handling module so that I can return consistent, branded
error responses for every HTTP and GraphQL error type — 400, 401, 403, 404, 500, 503, rate limit —
with structured JSON for API consumers and optional HTML pages for browser requests, with
correlation IDs and optional Sentry integration throughout.

**Should Have**

## Acceptance Criteria

```gherkin
Given a 404 error occurs on an HTML request
When Django's error handler fires
Then syntek-error renders the configured 404 template (or the built-in default) with the tenant
  branding context and returns an HTML response with status 404

Given a 404 error occurs on a GraphQL or API request (Accept: application/json)
When Django's error handler fires
Then a structured JSON response is returned:
  { "error": { "code": "NOT_FOUND", "message": "...", "correlationId": "uuid" } }

Given a 403 error occurs and the user is unauthenticated
When the error handler evaluates the request
Then it distinguishes between "not authenticated" (401 Unauthorized) and "not authorised" (403
  Forbidden) and returns the appropriate code and, for HTML requests, the appropriate template;
  the REDIRECT_UNAUTHENTICATED_TO setting controls whether 401s redirect to the login page instead

Given a 500 error occurs
When the error handler fires
Then a structured response is returned with a correlation ID; if SENTRY_ENABLED is True the
  exception is captured with the correlation ID as a tag; the full traceback is NEVER exposed
  in the response body regardless of DEBUG setting

Given a rate limit error is triggered by syntek-security (US012)
When the 429 response is built
Then the Retry-After header is set; for JSON consumers the error code is RATE_LIMITED

Given ERROR_PAGES is configured with a custom template path for 404
When a 404 is rendered
Then the custom template is used instead of the built-in default; custom templates receive the
  same context dict as the defaults (tenant, error_code, correlation_id, request_path)
```

## Configuration

```python
SYNTEK_ERROR = {
    # Response format
    'FORCE_JSON_FOR_API_PATHS': True,      # paths matching API_PATH_PREFIXES always return JSON
    'API_PATH_PREFIXES': ['/graphql/', '/api/'], # used when FORCE_JSON_FOR_API_PATHS is True

    # Error templates (HTML)
    'ERROR_PAGES': {
        '400': '',              # empty = use built-in default
        '401': '',
        '403': '',
        '404': '',
        '429': '',
        '500': '',
        '503': '',
    },

    # Authentication redirect
    'REDIRECT_UNAUTHENTICATED_TO': '',     # e.g. '/login/' — if set, 401 HTML responses redirect
    'REDIRECT_FORBIDDEN_TO': '',           # optional redirect for 403; empty = render 403 page

    # Correlation IDs
    'CORRELATION_ID_HEADER': 'X-Correlation-ID',  # read from request if present, else generate
    'INCLUDE_CORRELATION_ID_IN_RESPONSE': True,

    # Sentry
    'SENTRY_ENABLED': False,              # capture 500 errors to Sentry/Glitchtip via US014

    # Maintenance mode
    'MAINTENANCE_MODE': False,            # when True, all requests return 503
    'MAINTENANCE_MESSAGE': 'We are currently undergoing maintenance. Please try again shortly.',
    'MAINTENANCE_ALLOWED_IPS': [],        # IPs that bypass maintenance mode

    # Custom error codes (extend the built-in set)
    'CUSTOM_ERROR_CODES': {},             # { 'MY_CODE': {'status': 422, 'message': 'Default msg'} }
}
```

## Error Response Format

### JSON (API / GraphQL clients)

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "The requested resource was not found.",
    "correlationId": "a1b2c3d4-e5f6-...",
    "details": {}
  }
}
```

### GraphQL errors (injected into the `errors` array by Strawberry extension)

```json
{
  "data": null,
  "errors": [
    {
      "message": "Not authorised",
      "extensions": {
        "code": "FORBIDDEN",
        "correlationId": "a1b2c3d4-e5f6-..."
      }
    }
  ]
}
```

## Built-In Error Codes

| HTTP Status | Code                    | Scenario                             |
| ----------- | ----------------------- | ------------------------------------ |
| 400         | `BAD_REQUEST`           | Malformed request                    |
| 401         | `UNAUTHENTICATED`       | Missing or invalid credentials       |
| 403         | `FORBIDDEN`             | Authenticated but lacks permission   |
| 404         | `NOT_FOUND`             | Resource does not exist              |
| 405         | `METHOD_NOT_ALLOWED`    | HTTP method not permitted            |
| 409         | `CONFLICT`              | Optimistic lock / duplicate resource |
| 422         | `VALIDATION_ERROR`      | Input validation failure             |
| 429         | `RATE_LIMITED`          | Too many requests                    |
| 500         | `INTERNAL_SERVER_ERROR` | Unhandled exception                  |
| 503         | `SERVICE_UNAVAILABLE`   | Maintenance mode or dependency down  |
