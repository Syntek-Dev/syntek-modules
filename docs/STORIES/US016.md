## User Story

As a developer, I want a per-tenant settings store so that each tenant can have independently
configurable typed values ŌĆö including design token overrides ŌĆö that modules read at runtime via
GraphQL.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a setting is defined with type `bool` and a default value
When `get_setting("feature_x_enabled", tenant)` is called for a tenant with no override
Then the default value is returned

Given tenant A sets `feature_x_enabled` to `True`
When `get_setting("feature_x_enabled", tenant_a)` and `get_setting("feature_x_enabled", tenant_b)` are called
Then `True` is returned for tenant A and the default for tenant B

Given an attempt to store a value that does not match the declared type
When the setting is saved
Then a `SettingTypeError` is raised and nothing is persisted

Given a design token override is written for a tenant (e.g. `--color-primary: #FF6B00`)
When the `designTokens` GraphQL query is called for that tenant
Then the overridden values are returned and the frontend applies them at `:root`

Given no design token override exists for a tenant
When the `designTokens` query is called
Then an empty object is returned and the frontend uses the `shared/tokens` defaults
```
