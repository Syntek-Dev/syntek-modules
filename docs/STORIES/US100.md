# US100 — `syntek-payment-links`: Shareable Payment Links

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a payment links module so that I can generate shareable payment links tied to
invoices, custom amounts, or product purchases that redirect to a hosted checkout page using the
configured payment provider.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a payment link is created for an invoice
When the link is opened
Then the correct amount and invoice reference are shown and the configured provider's checkout is rendered

Given a single-use payment link has been paid
When the link is opened again
Then a 410 Gone response is returned

Given a payment link has passed its expiry date
When it is opened
Then a 410 Gone response is returned

Given a payment link is paid via the hosted checkout
When the provider webhook confirms payment
Then the linked invoice is marked as paid and a receipt is sent via syntek-notifications

Given a payment link is created with email delivery
When the mutation completes
Then an email containing the link is dispatched via syntek-notifications
```
