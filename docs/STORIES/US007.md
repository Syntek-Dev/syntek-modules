# US007 — `syntek-pyo3`: PyO3 Django Bindings

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 2 — Rust Encryption Layer

---

## User Story

As a backend developer, I want to call Rust crypto functions from Django model fields so that encryption and decryption happen transparently with no Python crypto code.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the syntek_pyo3 extension is compiled and installed in .venv
When I run `from syntek_pyo3 import encrypt_field, decrypt_field, hash_password, verify_password`
Then the import succeeds with no errors

Given a Django model field using `EncryptedField`
When a model instance is saved with a plaintext value
Then `pre_save` transparently calls `encrypt_field` and the DB contains only ciphertext

Given a Django model instance is fetched from the DB
When I access an `EncryptedField` attribute in Python
Then `from_db_value` transparently calls `decrypt_field` and returns the plaintext string

Given an invalid decryption key or tampered ciphertext
When `from_db_value` is called
Then a `DecryptionError` is raised and no partial plaintext is returned

Given maturin develop is run
When I import syntek_pyo3 in the active .venv
Then the native extension loads without any import errors
```
