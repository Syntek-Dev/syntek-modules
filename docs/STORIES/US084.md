# US084 — `@syntek/ui-tasks`: Background Job Progress UI

## User Story

As a developer, I want a Next.js package for background job progress so that users can see the
status of long-running operations (bulk imports, report generation, bulk email dispatches) without
custom polling logic or bespoke UI components in every application.

## Configuration Contract

All configuration is passed as props — no `.env` config. Settings come from the GraphQL API provided
by `syntek-tasks` (US015).

```tsx
import { useTaskStatus, JobProgressBar, BackgroundJobsFeed } from '@syntek/ui-tasks';

// Hook — polls taskStatus GraphQL query until terminal state
const { status, progress, result, error, isComplete } = useTaskStatus(taskId, {
    pollIntervalMs: 2000,     // number — polling interval in ms (default 2000)
    stopOnComplete: true,     // boolean — stop polling when SUCCESS or FAILURE (default true)
    onSuccess: (result) => void,  // callback on SUCCESS
    onFailure: (error) => void,   // callback on FAILURE
});

// Component — single task progress bar
<JobProgressBar
    taskId={taskId}
    label="Sending emails..."
    showPercentage={true}
    pollIntervalMs={2000}
    onComplete={(result) => void}
    onError={(error) => void}
/>

// Component — feed of multiple background jobs
<BackgroundJobsFeed
    taskIds={[taskId1, taskId2]}  // or empty — fetches all recent tasks for current user
    maxItems={10}
    showCompleted={true}
    showFailed={true}
    pollIntervalMs={3000}
/>
```

**Key rules:**

- `useTaskStatus` polls the `taskStatus(taskId: ID!)` query from `@syntek/api-client`.
- Polling stops automatically when `status` reaches `SUCCESS` or `FAILURE` and `stopOnComplete` is
  `true`.
- `BackgroundJobsFeed` with no `taskIds` prop calls a `recentTasks` query (paginated, tenant-scoped)
  to show all recent tasks for the current user — requires `syntek-tasks` GraphQL to expose this
  query.
- All components are headless-friendly — they accept a `className` prop and use design tokens from
  `@syntek/ui` for default styling. Consumers can override all visual styles.
- No WebSocket or subscription dependency — polling only. Consumers who need real-time push can
  layer Django Channels subscriptions on top.

## Acceptance Criteria

```gherkin
# --- useTaskStatus hook ---

Given a taskId for a PENDING task
When useTaskStatus is called with pollIntervalMs: 1000
Then status is 'PENDING' and polling begins

Given the task transitions to STARTED with progress 40
When the next poll fires
Then status is 'STARTED' and progress is 40

Given the task transitions to SUCCESS
When the next poll fires
Then isComplete is true, onSuccess callback is invoked, and polling stops

Given the task transitions to FAILURE
When the next poll fires
Then isComplete is true, onFailure callback is invoked with the error message, polling stops

Given stopOnComplete is false
When the task reaches SUCCESS
Then polling continues until the component unmounts

Given the component unmounts before the task completes
When the cleanup function runs
Then no further polls are made — no memory leaks

# --- JobProgressBar component ---

Given taskId resolves to a STARTED task with progress 65
When JobProgressBar renders
Then a progress bar at 65% is shown with the configured label

Given showPercentage is true
When progress is 65
Then '65%' is displayed alongside the bar

Given the task reaches SUCCESS
When onComplete fires
Then the bar shows 100% and a success state

Given the task reaches FAILURE
When onError fires
Then an error message is shown and the bar indicates failure state

# --- BackgroundJobsFeed component ---

Given taskIds contains two task IDs
When BackgroundJobsFeed renders
Then two job rows are shown, each with their current status and progress

Given showCompleted is false
When a task in the list reaches SUCCESS
Then its row is removed from the feed

Given showFailed is false
When a task in the list reaches FAILURE
Then its row is removed from the feed

Given maxItems is 5 and 6 tasks are provided
When BackgroundJobsFeed renders
Then only 5 rows are shown — the oldest is truncated

# --- Accessibility ---

Given JobProgressBar renders
When a screen reader reads the component
Then the progress bar has role="progressbar", aria-valuenow, aria-valuemin, aria-valuemax set

Given status is FAILURE
When the error message renders
Then it has role="alert" for immediate screen reader announcement
```
