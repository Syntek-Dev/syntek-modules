## User Story

As a developer, I want a Cloudinary-backed media module so that images and videos are stored in
Cloudinary CDN, metadata is stored in PostgreSQL, and configurable transformations and optimisations
are applied — with no static file storage on the application server.

## Acceptance Criteria

```gherkin
Given a valid image file is uploaded via the media mutation
When the upload handler runs
Then the file is delivered to Cloudinary and a MediaAsset record is created with public_id,
format, width, height, duration (video), size, alt text, and tenant FK

Given a MediaAsset is queried via GraphQL
When the resolver returns it
Then it includes the Cloudinary CDN URL and pre-built transformation URLs — never the raw API URL

Given a media asset is deleted
When the deletion handler runs
Then the asset is removed from Cloudinary AND the MediaAsset record is removed from PostgreSQL
atomically — partial state is impossible

Given a non-image/non-video MIME type is submitted to the upload mutation
When the MIME type is validated
Then the upload is rejected with 422 and no Cloudinary API call is made

Given SYNTEK_MEDIA IMAGE_OPTIMIZATION is enabled
When an image is uploaded
Then Cloudinary receives the configured eager transformations (thumbnail, WebP, srcset variants)
and the transformation URLs are stored on the MediaAsset record

Given a video is uploaded
When the Cloudinary upload completes asynchronously
Then the MediaAsset transcoding_status is set to "processing" and updated to "ready" or "failed"
when the Cloudinary webhook fires

Given MediaAsset.alt is marked as encrypted via SYNTEK_MEDIA ENCRYPT_ALT_TEXT
When an alt text value is stored
Then it is encrypted via syntek-pyo3 before the DB write and decrypted transparently on read
```

## Configuration

```python
SYNTEK_MEDIA = {
    # --- Cloudinary credentials (required) ---
    'CLOUD_NAME': env('CLOUDINARY_CLOUD_NAME'),
    'API_KEY': env('CLOUDINARY_API_KEY'),
    'API_SECRET': env('CLOUDINARY_API_SECRET'),

    # --- Upload ---
    'UPLOAD_PRESET': '',                   # Optional — Cloudinary unsigned upload preset name
    'FOLDER': 'media',                     # Root Cloudinary folder for all uploads
    'UNIQUE_FILENAME': True,               # Add random suffix to avoid collisions
    'OVERWRITE': False,                    # Reject if public_id already exists
    'RESOURCE_TYPES': ['image', 'video'],  # Allowed resource types; never 'raw'
    'ALLOWED_MIME_TYPES': [                # Explicit allowlist
        'image/jpeg', 'image/png', 'image/gif', 'image/webp',
        'image/avif', 'image/svg+xml',
        'video/mp4', 'video/webm', 'video/quicktime',
    ],
    'MAX_IMAGE_SIZE_MB': 20,
    'MAX_VIDEO_SIZE_MB': 500,

    # --- Image optimisation ---
    'IMAGE_OPTIMIZATION': True,            # Enable auto quality and auto format on delivery
    'AUTO_QUALITY': 'auto:good',           # Cloudinary quality string
    'AUTO_FORMAT': True,                   # Serve WebP/AVIF where supported
    'EAGER_TRANSFORMATIONS': [             # Transformations applied on upload (blocking or async)
        {'width': 400, 'height': 400, 'crop': 'fill', 'format': 'webp', 'quality': 'auto'},    # thumbnail
        {'width': 800, 'format': 'webp', 'quality': 'auto'},                                   # medium
        {'width': 1600, 'format': 'webp', 'quality': 'auto'},                                  # large
    ],
    'EAGER_ASYNC': True,                   # True = eager transforms run after upload returns

    # --- Video ---
    'VIDEO_TRANSCODING_QUALITY': 'auto',   # Cloudinary video quality preset
    'VIDEO_WEBHOOK_SECRET': env('CLOUDINARY_WEBHOOK_SECRET', default=''),

    # --- Delivery ---
    'SECURE_URLS': True,                   # Always serve HTTPS URLs
    'CDN_SUBDOMAIN': True,                 # Use Cloudinary CDN sub-domains for load balancing

    # --- Security ---
    'ENCRYPT_ALT_TEXT': False,             # Encrypt alt text via syntek-pyo3 if True
    'SIGNED_UPLOADS': False,               # Require signed upload parameters if True

    # --- Tenancy ---
    'TENANT_FOLDER_STRATEGY': 'prefix',    # 'prefix' (folder/{tenant_id}/file) | 'tag' (Cloudinary tag)
}
```

### Transformation URL Builder

`syntek-media-core` exposes a `build_url(public_id, **transform_options)` helper that wraps the
Cloudinary SDK URL builder. Consuming modules never construct raw Cloudinary URLs — they call this
helper to ensure `SECURE_URLS`, `AUTO_FORMAT`, and `AUTO_QUALITY` are always applied.

Pre-built URL slots stored on every `MediaAsset`:

| Slot           | Description                                            |
| -------------- | ------------------------------------------------------ |
| `url`          | Original file CDN URL (with auto quality + format)     |
| `thumbnail`    | 400×400 fill crop, WebP                                |
| `medium`       | 800px wide, WebP                                       |
| `large`        | 1600px wide, WebP                                      |
| `srcset`       | Comma-separated srcset string for `<img srcset="...">` |
| `video_poster` | First-frame JPEG poster (video only)                   |

Additional on-demand transforms are generated via `build_url` at request time and not stored.

## Models

| Model        | Field                  | Notes                                            |
| ------------ | ---------------------- | ------------------------------------------------ |
| `MediaAsset` | `public_id`            | Cloudinary public_id — primary key for CDN ops   |
|              | `resource_type`        | `image` or `video`                               |
|              | `format`               | Original format (jpeg, mp4, etc.)                |
|              | `width`, `height`      | Pixel dimensions                                 |
|              | `duration`             | Video only — seconds as float                    |
|              | `size`                 | Bytes                                            |
|              | `alt`                  | Alt text; encrypted if `ENCRYPT_ALT_TEXT = True` |
|              | `tags`                 | ArrayField — Cloudinary tags                     |
|              | `transformation_urls`  | JSONField — pre-built slot URLs                  |
|              | `transcoding_status`   | `pending` / `processing` / `ready` / `failed`    |
|              | `tenant`               | FK — tenant isolation                            |
|              | `uploaded_by`          | FK — user who uploaded                           |
|              | `created_at`           | Timestamp                                        |
| `MediaTag`   | `name`, `assets` (M2M) | Logical tags for gallery filtering               |

## GraphQL API

### Mutations

| Mutation                       | Description                                                             |
| ------------------------------ | ----------------------------------------------------------------------- |
| `uploadMedia(file, alt, tags)` | Upload to Cloudinary, create MediaAsset, return with URLs               |
| `deleteMedia(id)`              | Atomic Cloudinary + DB deletion                                         |
| `updateMediaAlt(id, alt)`      | Update alt text (encrypted if configured)                               |
| `tagMedia(id, tags)`           | Add/replace tags on a MediaAsset                                        |
| `cloudinaryWebhook(payload)`   | Internal — receives Cloudinary notification, updates transcoding_status |

### Queries

| Query                                         | Returns                                          |
| --------------------------------------------- | ------------------------------------------------ |
| `mediaAssets(page, pageSize, mimeType, tags)` | Paginated `[MediaAsset]` — tenant-scoped         |
| `mediaAsset(id)`                              | Single `MediaAsset` with all transformation URLs |
| `buildTransformUrl(id, options)`              | On-demand Cloudinary transform URL — not stored  |

## Dependencies

- US010 — `syntek-tenancy`
- US007 — `syntek-pyo3` (optional — required only when `ENCRYPT_ALT_TEXT = True`)
- US030 depends on `syntek-media-upload` (US106) for the chunked upload pipeline when files exceed
  `MAX_IMAGE_SIZE_MB` or `MAX_VIDEO_SIZE_MB`; direct small-file uploads bypass the upload pipeline
