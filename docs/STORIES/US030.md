# US030 — `syntek-media`: Cloudinary Media Module

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 5 — Backend Compliance & Media

---

## User Story

As a developer, I want a media module so that images and videos are stored in Cloudinary, metadata is stored in PostgreSQL, and assets are served via Cloudinary CDN — with no static file storage on the application server.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a valid image is uploaded via the media endpoint
When the upload handler runs
Then the file is stored in Cloudinary and a `MediaAsset` record is created in PostgreSQL with public_id, format, width, height, and alt text

Given a `MediaAsset` record is queried via GraphQL
When the resolver returns it
Then it includes the Cloudinary CDN URL, not the raw storage URL

Given a media asset is deleted
When the deletion handler runs
Then the asset is removed from Cloudinary AND the `MediaAsset` record is removed from PostgreSQL atomically

Given an attempt to upload a non-image/non-video file (e.g. a PDF)
When the upload handler validates the MIME type
Then it rejects the upload with 422 and makes no Cloudinary API call

Given an image upload succeeds
When the `MediaAsset` is queried
Then Cloudinary transformation URLs are available (thumbnail, webp, srcset variants)
```
