## User Story

As a developer, I want a React forms package so that I can build any form — of any length, any field
type, any conditional logic — that works equally as a full page, a section of a page, a modal, a
drawer, or a multi-step wizard. Forms must support pre-filled values from any source, schema-driven
dynamic rendering from `syntek-forms-core`, or static Zod schemas — all with a single consistent
`useForm` API.

## Acceptance Criteria

```gherkin
Given a Zod schema is passed to FormProvider
When a field value fails validation
Then the error message is displayed beneath the field immediately on blur, and the form cannot
submit until all errors are resolved

Given a dynamic form schema is fetched from the GraphQL API
When DynamicForm renders the schema
Then the correct field components are rendered in the correct step order with all validation rules,
conditional logic, and pre-fill values applied

Given a conditional field has its display condition unmet
When the user changes a controlling field's value
Then the dependent field is hidden immediately and its value is excluded from the submission payload

Given a multi-step form has 5 steps
When the user completes step 2 and navigates forward
Then step 3 renders; the progress bar shows 40%; step 1 and 2 are reachable via back navigation

Given a form is used inside a modal
When FormProvider receives the compact prop
Then all internal field spacing and padding is reduced to fit the constrained container without
any external layout changes required

Given a form has defaultValues set
When the form first renders
Then all configured fields are pre-populated with those values — the user sees a filled-in form

Given an async initialValues prop is set
When the data resolves
Then the form fields update to the resolved values without remounting

Given a form submission is in flight
When the submit mutation is pending
Then all fields and the submit button are disabled and a loading indicator is shown

Given useForm is used without any of the provided field components
When the hook is called
Then it returns value, error, onChange, handleSubmit, and reset — fully usable headlessly
```

## Deployment Contexts

`@syntek/forms` has no layout opinions. It works in all of these contexts with no extra
configuration:

| Context        | How to use                                                          |
| -------------- | ------------------------------------------------------------------- |
| Full page      | Mount `FormProvider` at page level; use `FormSection` to group      |
| Page section   | Mount `FormProvider` inside any container — no full-page assumption |
| Modal/drawer   | Pass `compact` prop to `FormProvider` for reduced spacing           |
| Multi-step     | Use `MultiStepForm` with `FormStep` children                        |
| Pre-filled     | Pass `defaultValues` (static) or `initialValues` (async)            |
| Dynamic schema | Use `DynamicForm` with backend schema — all logic driven by JSON    |
| Headless       | Use `useForm` hook alone without any provided components            |

## Hooks

### `useForm(options)`

Primary API. Works with Zod schemas and dynamic backend schemas.

```tsx
const form = useForm({
  schema: myZodSchema, // static Zod schema (optional if using DynamicForm)
  defaultValues: { name: "" }, // static pre-fill
  initialValues: fetchUser(), // async pre-fill — Promise resolving to partial field values
  onSubmit: async (data) => {
    await submitMutation(data);
  },
  onSuccess: () => router.push("/thank-you"),
  onError: (errors) => console.error(errors),
  mode: "onBlur", // 'onBlur' | 'onChange' | 'onSubmit'
});
```

Returns: `{ register, handleSubmit, reset, setValue, watch, formState, control, step }`

### `useFormField(name)`

Access a single field's state outside a field component.

```tsx
const { value, error, onChange, onBlur } = useFormField("email");
```

### `useFormWatch(fieldName?)`

Observe field value changes for conditional rendering outside the form context.

```tsx
const enquiryType = useFormWatch("enquiry_type");
```

### `useFormStep()`

Multi-step navigation state.

```tsx
const { currentStep, totalSteps, goNext, goPrev, canGoNext, canGoPrev, progress } = useFormStep();
```

## Components

### `FormProvider`

Context root. Must wrap all form components and hooks.

```tsx
<FormProvider
  form={form}
  compact={isModal} // reduces padding for modal/drawer use
  disabled={isReadOnly} // disables all fields (for prefilled read-only view)
>
  {children}
</FormProvider>
```

### Field Components

All field components accept `name` (wired to form context), `label`, `description`, `required`,
`disabled`, and `className`. Validation errors are displayed automatically.

| Component            | Renders                         | Key Props                                   |
| -------------------- | ------------------------------- | ------------------------------------------- |
| `TextField`          | Single-line text input          | `maxLength`, `pattern`, `autoComplete`      |
| `TextareaField`      | Multi-line textarea             | `rows`, `maxLength`, `autoResize`           |
| `NumberField`        | Numeric input                   | `min`, `max`, `step`                        |
| `EmailField`         | Email input                     |                                             |
| `PhoneField`         | Phone input with country prefix | `defaultCountry`                            |
| `URLField`           | URL input                       |                                             |
| `SelectField`        | Single-choice dropdown          | `options`, `placeholder`, `searchable`      |
| `MultiSelectField`   | Multi-choice dropdown           | `options`, `maxSelections`                  |
| `CheckboxField`      | Single boolean checkbox         |                                             |
| `CheckboxGroupField` | Multiple checkboxes             | `options`, `minSelections`, `maxSelections` |
| `RadioGroupField`    | Radio button group              | `options`, `layout` (vertical/horizontal)   |
| `ToggleField`        | Boolean switch                  |                                             |
| `DateField`          | Date picker                     | `minDate`, `maxDate`, `format`              |
| `TimeField`          | Time picker                     | `minuteStep`                                |

| `DateRangeField`     | From/to date pair               | `minDate`, `maxDate`                        |

### Layout Components

```tsx
<FormSection title="Contact Details" description="All fields are required.">
  <TextField name="first_name" label="First Name" />
  <EmailField name="email" label="Email" />
</FormSection>
```

```tsx
<MultiStepForm form={form} showProgress onComplete={() => router.push("/done")}>
  <FormStep title="Your Details" validate={["first_name", "email"]}>
    <TextField name="first_name" label="First Name" />
    <EmailField name="email" label="Email" />
  </FormStep>
  <FormStep title="Your Message">
    <TextareaField name="message" label="Message" />
  </FormStep>
</MultiStepForm>
```

`MultiStepForm` props: `showProgress`, `stepNavigationLabel`, `nextLabel`, `prevLabel`,
`submitLabel`, `onComplete`, `draftKey` (localStorage key for auto-save draft).

### `DynamicForm`

Renders a `syntek-forms-core` schema fetched from the GraphQL API. No other configuration required —
all field types, steps, conditional logic, and pre-fill are driven by the backend schema.

```tsx
<DynamicForm
  formId="contact-form"
  onSuccess={(submission) => router.push("/thank-you")}
  compact={isModal}
/>
```

Internally: fetches schema via `useFormSchema(formId)`, resolves pre-fill, evaluates conditions,
maps `schema.steps[].fields[]` to the correct field components, and calls `submitForm` mutation.

### `FormDraftSaver`

Auto-saves form progress to `localStorage` (or the backend draft API if `draftKey` is set to a
`formId`). Restores on mount.

```tsx
<FormDraftSaver draftKey="contact-form-draft" debounceMs={1000} />
```

## Conditional Logic

Conditions from the backend schema are evaluated client-side in real time via `useFormWatch`. Static
Zod forms support conditional visibility via the `showWhen` prop on any field component:

```tsx
<TextField
  name="account_ref"
  label="Account Reference"
  showWhen={{ field: "enquiry_type", op: "eq", value: "billing" }}
/>
```

When a field is hidden, its value is excluded from the submission payload automatically.
