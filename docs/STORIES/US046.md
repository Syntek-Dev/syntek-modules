# US046 — `@syntek/forms`: Headless Form Primitives

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 6 — Web Frontend Packages

---

## User Story

As a developer, I want headless form primitives so that I can build validated, accessible forms that integrate with both static schemas and the dynamic schemas generated by `syntek-forms` on the backend.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a Zod schema is passed to the form
When a field value fails validation
Then the error message is displayed beneath the field and the form cannot be submitted

Given a dynamic form schema is received from the GraphQL API
When the form renderer processes the schema
Then the correct field types are rendered in the correct order with the correct validation rules

Given a conditional field has its display condition unmet
When the form renders
Then the conditional field is hidden and its value is excluded from the submission payload

Given a form is submitted
When the submission is in flight
Then all fields are disabled and a loading indicator is shown
```
