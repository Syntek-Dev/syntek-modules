# US001 — Monorepo Workspace Configuration

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: ✅ Completed **Epic**: Epic 1 —
Foundation & Workspace

---

## User Story

As a module contributor, I want a fully configured monorepo workspace so that I can develop any
package layer (Python, TypeScript, Rust) from a single root with consistent tooling.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a fresh clone of the repository
When I run `pnpm install` from the root
Then all TypeScript/JavaScript workspace packages are linked and resolvable

Given a fresh clone of the repository
When I run `uv venv && source .venv/bin/activate && uv pip install -e "packages/backend/syntek-auth[dev]"`
Then the backend package installs with all dev dependencies

Given a fresh clone of the repository
When I run `cargo build` from the root
Then all Rust crates compile without errors

Given I am on any operating system
When I run `./dev.sh`
Then watch modes start for all active layers without errors

Given I run `./test.sh`
When the script completes
Then unit and integration tests run across all layers and a combined pass/fail is reported
```
