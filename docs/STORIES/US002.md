## User Story

As a frontend developer, I want a shared TypeScript types package so that all web and mobile
packages use consistent type definitions without duplication.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the shared/types package is installed in a workspace package
When I import a type
Then TypeScript resolves it without errors

Given a type is defined in shared/types
When I change its shape in a breaking way
Then TypeScript compilation fails in all consuming packages that are incompatible

Given shared/types is built
When I check the dist output
Then declaration files (.d.ts) are present for all exports
```
