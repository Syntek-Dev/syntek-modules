## User Story

As a frontend developer, I want pre-generated, typed GraphQL operations so that all packages query
the backend consistently with full TypeScript inference.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the shared/graphql package is built
When I import a query hook in any web package
Then TypeScript infers the correct response and variable types

Given a query is defined in a .graphql file
When I run `pnpm codegen`
Then typed React Query hooks are generated for that operation

Given the backend GraphQL schema changes
When I run `pnpm codegen`
Then the generated types update and TypeScript surfaces any breaking changes in consuming packages

Given codegen runs in CI
When the output differs from the committed files
Then the CI pipeline fails, requiring the developer to commit updated generated files
```
