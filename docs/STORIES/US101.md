## User Story

As a developer, I want a report export module so that I can generate PDF, Excel, and CSV files from
completed report runs, deliver them by email, and provide pre-signed download URLs — reusing the
storage infrastructure from `syntek-bulk-export`.

## Acceptance Criteria

```gherkin
Given a ReportRun completes with status "done"
When an export is requested for that run in PDF format
Then a branded, paginated PDF is generated asynchronously via Celery and stored with a pre-signed URL

Given an Excel export is requested
When the file is generated
Then it contains formatted headers, correct column data types, and freeze panes on row 1

Given a CSV export is requested with a custom delimiter
When the file is generated
Then all rows use the specified delimiter and encoding and the file is UTF-8 BOM for Excel compatibility

Given a ScheduledReport has an email recipient list
When a report run completes
Then the export file is attached (or linked by URL) and dispatched via syntek-notifications (US087)

Given the export storage backend is MinIO
When an export is created
Then the file is stored via syntek-bulk-export's storage layer and the download URL respects
DOWNLOAD_URL_TTL

Given a custom PDF template hook is registered for a report ID
When a PDF export is generated for that report
Then the custom Jinja2 template is used instead of the default paginated table layout
```

## Configuration

```python
SYNTEK_REPORTING_EXPORT = {
    # --- Formats ---
    'SUPPORTED_FORMATS': ['pdf', 'excel', 'csv'],

    # --- Storage ---
    'STORAGE_BACKEND': 'minio',          # 'minio' | 'local' | 's3'
    'DOWNLOAD_URL_TTL': 3600,            # Pre-signed URL expiry (seconds)

    # --- PDF ---
    'PDF_ENGINE': 'weasyprint',          # Fixed; reserved for future adapters
    'PDF_PAGE_SIZE': 'A4',               # 'A4' | 'Letter'
    'PDF_ORIENTATION': 'landscape',      # 'portrait' | 'landscape'
    'PDF_TEMPLATE_HOOKS': {},            # Dict[report_id, str] — Jinja2 template path per report

    # --- Excel ---
    'EXCEL_FREEZE_HEADER': True,
    'EXCEL_AUTO_COLUMN_WIDTH': True,

    # --- CSV ---
    'CSV_DELIMITER': ',',
    'CSV_ENCODING': 'utf-8-sig',         # BOM for Excel compatibility
    'CSV_QUOTING': 'minimal',            # 'minimal' | 'all' | 'nonnumeric'

    # --- Delivery ---
    'EMAIL_DELIVERY': True,              # Requires syntek-notifications (US087); skipped if absent
    'EMAIL_ATTACH_SMALL_FILES': True,    # Attach if file < 5 MB; link otherwise
    'EMAIL_LINK_TTL': 86400,            # Link expiry for emailed download links (seconds)

    # --- Cleanup ---
    'EXPORT_RETENTION_DAYS': 30,         # Auto-delete stored export files after N days
}
```

### PDF Template Hooks

When `PDF_TEMPLATE_HOOKS` maps a `report_id` to a Jinja2 template path, that template is used for
PDF export of that report. Template context variables:

| Variable       | Type         | Description                                   |
| -------------- | ------------ | --------------------------------------------- |
| `report`       | `dict`       | Report definition metadata                    |
| `run`          | `dict`       | ReportRun metadata (parameters, generated_at) |
| `rows`         | `list[dict]` | Output rows from the ReportRunResult          |
| `tenant`       | `dict`       | Tenant context (name, logo, branding)         |
| `generated_at` | `datetime`   | Export generation timestamp                   |

When no hook is registered, the default template renders a paginated table with the report title,
tenant logo (if available), column headers from `output_fields`, and a footer with page numbers and
generation timestamp.

## Integration with `syntek-bulk-export`

`syntek-reporting-export` reuses the storage backend and pre-signed URL infrastructure from
`syntek-bulk-export` (US093) for CSV and Excel files. PDF generation uses its own WeasyPrint
pipeline. Both packages share the same `STORAGE_BACKEND` adapter interface — consuming projects that
have installed both modules configure storage once.

If `syntek-bulk-export` is not installed, `syntek-reporting-export` falls back to its own local file
storage with a warning at startup.
