# US008 — `syntek-graphql-crypto`: GraphQL Encryption Middleware

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 2 — Rust Encryption Layer

---

## User Story

As a security engineer, I want a GraphQL middleware layer so that no encrypted field value ever reaches a resolver response as raw ciphertext, and all decryption is performed server-side before serialisation.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a GraphQL resolver returns a field annotated with `@encrypted`
When the middleware processes the response
Then the field value is decrypted before serialisation

Given a decryption failure for an `@encrypted` field
When the middleware processes the response
Then the field is set to `null`, an error is appended to the `errors` array, and the request is not aborted

Given a field is NOT annotated with `@encrypted`
When the middleware processes the response
Then it passes through unchanged with no performance overhead

Given an unauthenticated request attempts to resolve an `@encrypted` field
When the middleware runs
Then it returns `null` with an authorisation error and logs the attempt via syntek-logging

Given middleware is configured at application startup
When individual resolvers run
Then they do not need any decryption logic — it is entirely handled by the middleware
```
