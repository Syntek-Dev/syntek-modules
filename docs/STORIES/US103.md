## User Story

As a developer, I want an analytics UI package so that I can inject analytics scripts with GDPR
consent gating, track page views automatically on Next.js route changes, and render an admin
analytics dashboard using the chart components from `@syntek/ui-reporting`.

## Acceptance Criteria

```gherkin
Given AnalyticsProvider wraps the app with GDPR consent required
When a user has not consented
Then no analytics script is injected and no page view fires

Given a user consents and navigates between Next.js routes
When AnalyticsPageTracker detects a route change
Then a page view is tracked via the syntek-analytics GraphQL mutation without PII

Given the AdminAnalyticsDashboard is rendered
When analyticsStats data loads
Then page views, unique visitors, top pages, and top referrers are displayed using
@syntek/ui-reporting chart components

Given useAnalytics is called
When trackEvent is invoked with a name and properties
Then the syntek-analytics trackEvent mutation fires and the local event count increments

Given the analytics provider changes from Plausible to GA4 in backend settings
When the app restarts
Then AnalyticsProvider renders no provider-specific script (all tracking is server-side) and the
dashboard data reflects the new provider's data
```

## Components

| Component                 | Description                                                                                                 |
| ------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `AnalyticsProvider`       | Root provider — reads `analyticsConfig` from GraphQL, gates script injection on GDPR consent                |
| `AnalyticsPageTracker`    | Next.js router event listener — fires `trackPageView` mutation on route change                              |
| `AdminAnalyticsDashboard` | Full admin dashboard — page views chart, unique visitors, top pages table, top referrers, date range picker |
| `AnalyticsStatCard`       | Single stat display — value, label, trend indicator (up/down with percentage)                               |
| `TopPagesTable`           | Sortable table of top pages with view counts and percentage of total                                        |
| `TopReferrersTable`       | Sortable table of top referrers with visit counts                                                           |

### `AnalyticsProvider` Props

| Prop              | Type        | Default       | Description                                              |
| ----------------- | ----------- | ------------- | -------------------------------------------------------- |
| `requireConsent`  | `boolean`   | `true`        | Gate script injection on GDPR consent state              |
| `consentCategory` | `string`    | `'analytics'` | Consent category to check (matches `syntek-gdpr` config) |
| `children`        | `ReactNode` | —             | App content                                              |

`AnalyticsProvider` does not inject any client-side analytics script — all tracking is server-side
via `syntek-analytics`. The provider exists to coordinate consent state with `@syntek/ui-gdpr` and
to provide the `useAnalytics` context.

## Hooks

| Hook                                              | Description                                                              |
| ------------------------------------------------- | ------------------------------------------------------------------------ |
| `useAnalytics()`                                  | Returns `{ trackEvent, trackPageView, hasConsented, isReady }`           |
| `useAnalyticsStats(dateRange)`                    | Wraps `analyticsStats` GraphQL query; returns `{ data, loading, error }` |
| `useTopPages(dateRange, limit?)`                  | Wraps `analyticsTopEvents` query filtered by page view events            |
| `useTopReferrers(dateRange, limit?)`              | Wraps `analyticsStats.topReferrers`                                      |
| `useAnalyticsTimeSeries(dateRange, granularity?)` | Wraps `analyticsPageViews` query; returns time-series rows for charts    |

### `useAnalytics` Return Shape

```ts
interface UseAnalyticsReturn {
  trackEvent: (name: string, properties?: Record<string, string | number | boolean>) => void;
  trackPageView: (path?: string) => void;
  hasConsented: boolean; // Current GDPR consent state for analytics category
  isReady: boolean; // True when provider has resolved consent state
}
```

`trackEvent` and `trackPageView` are no-ops when `hasConsented` is false.

## `AdminAnalyticsDashboard` Layout

The dashboard uses chart components from `@syntek/ui-reporting`:

- **Top row:** three `AnalyticsStatCard` components — total page views, unique visitors, pages per
  session — each with a trend indicator compared to the previous period
- **Main chart:** `LineChart` (from `@syntek/ui-reporting`) showing page views and unique visitors
  over time — granularity auto-selected from date range
- **Bottom row:** `TopPagesTable` (left) and `TopReferrersTable` (right)
- **Controls:** `DateRangePicker` (from `@syntek/ui`) — defaults to last 30 days; options: 7d, 30d,
  90d, custom range
