# US071 — `@syntek/ui-maps`: Maps, Routing & Live Tracking UI Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

---

## User Story

As a developer, I want a maps UI package so that I can embed interactive maps, location pickers,
route displays, live tracking views, and "open in navigation app" buttons — all driven by
configurable providers with no hardcoded map service.

## Architecture

The map renderer is configured via `SYNTEK_MAPS['PROVIDER']` on the backend. `@syntek/ui-maps` reads
this from the GraphQL config response and lazy-loads the appropriate SDK (Mapbox GL JS, Google Maps
JS API, or Leaflet). Consuming projects only ship the SDK bundle for their chosen provider. Live
tracking updates arrive via the `positionUpdated` GraphQL subscription (Strawberry WebSocket).

## Configuration

Resolved from GraphQL `mapsConfig` query — not set in frontend code:

```typescript
interface MapsConfig {
  provider: "mapbox" | "google" | "leaflet";
  accessToken: string; // Provider API key/token
  defaultCentre: [number, number]; // [lat, lng]
  defaultZoom: number;
  trafficLayerEnabled: boolean;
  navigationApps: string[]; // From SYNTEK_ROUTING['NAVIGATION_APPS']
}
```

## Acceptance Criteria

```gherkin
Scenario 1: Provider lazy-load — Mapbox
Given provider = 'mapbox'
When <Map> renders
Then Mapbox GL JS is lazy-loaded and the map tiles display using the configured Mapbox access token — no Google Maps SDK is loaded

Scenario 2: Provider lazy-load — Google
Given provider = 'google'
When <Map> renders
Then the Google Maps JS API is lazy-loaded instead — provider switch requires no frontend code change

Scenario 3: Route polyline rendering
Given a Route is passed to <RouteMap>
When the component renders
Then the optimised route polyline is drawn on the map, each stop is marked with its sequence number, and the estimated duration for each leg is shown in a sidebar

Scenario 4: Live tracker animation
Given a live tracking session is active
When <TrackingMap> renders
Then the tracker's marker updates in real time via the positionUpdated WebSocket subscription, with a smooth animated movement between positions

Scenario 5: Navigation app filtering
Given navigationApps = ['google_maps', 'waze']
When <NavigationLinks> renders
Then exactly two "Open in..." buttons are shown — no Apple Maps button

Scenario 6: LocationPicker autocomplete and pin
Given the LocationPicker field is used in a form
When a user types an address
Then autocomplete suggestions are fetched from the addressAutocomplete GraphQL query and a pin is placed on the map preview when a suggestion is selected

Scenario 7: Marker clustering
Given a map has more than 50 markers
When the zoom level is low
Then markers are clustered automatically with a count badge on the cluster icon
```
