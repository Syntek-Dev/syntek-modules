# US020 — `syntek-webhooks`: Inbound & Outbound Webhook Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a webhooks module so that I can receive signed inbound webhooks and dispatch
signed outbound events with automatic retry and full delivery logging.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given an inbound webhook arrives with an invalid HMAC signature
When the ingestion endpoint processes it
Then it returns 401 and the payload is not processed or stored

Given an inbound webhook has an idempotency key seen in the last 24 hours
When the ingestion endpoint processes it
Then it returns 200 without re-processing the payload

Given an outbound webhook delivery fails
When the retry policy triggers
Then it retries up to 10 times with exponential back-off and jitter

Given a domain event is fired within Django
When `syntek-webhooks` has an outbound rule matching that event
Then the payload is dispatched to the configured endpoint signed with HMAC-SHA256

Given a delivery attempt completes (success or failure)
When the delivery log is queried
Then the full request, response status, and duration are available
```
