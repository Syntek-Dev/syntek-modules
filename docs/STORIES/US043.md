# US043 — `@syntek/session`: Session Management Package

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 6 — Web Frontend Packages

---

## User Story

As a developer, I want a session management package so that I can handle JWT token storage, silent refresh, idle timeout, and session context across all Syntek web applications.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a user logs in and receives a JWT
When the session context provider mounts
Then the token is stored in an httpOnly cookie (not localStorage) and the session state is available via hook

Given the JWT is within 60 seconds of expiry
When the silent refresh interval fires
Then a refresh request is made and the new token replaces the old one without user interruption

Given the user has been idle for the configured timeout period
When the idle timeout fires
Then the session is cleared, the user is redirected to login, and a timeout notification is shown

Given the user closes the browser and returns within the session window
When the page reloads
Then the session is restored from the httpOnly cookie without requiring re-login
```
