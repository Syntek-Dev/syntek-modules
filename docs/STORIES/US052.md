## User Story

As a developer, I want a Stripe checkout UI package so that I can embed payment forms, subscription management, and billing history screens that connect to `syntek-payments` via GraphQL.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the checkout component is rendered
When Stripe Elements loads
Then the card input is embedded via Stripe Elements (never handling raw card numbers directly)

Given a payment is submitted
When Stripe confirms the payment intent
Then the backend mutation is called with the payment intent ID and the success state is shown

Given a subscription is active
When the subscription management panel renders
Then the current plan, next billing date, and cancel/upgrade options are displayed

Given a user cancels their subscription
When the cancel mutation completes
Then the subscription status updates to cancelled and the user sees a confirmation with the access end date
```
