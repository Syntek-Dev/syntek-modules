# US052 — `@syntek/ui-payments`: Provider-Agnostic Payment UI

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

---

## User Story

As a developer, I want a provider-agnostic payments UI package so that I can embed payment forms for
Stripe, Square, PayPal, and SumUp, manage subscriptions, apply discount codes, and view billing
history — all driven by the configured backend provider.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the provider is set to Stripe
When CheckoutForm renders
Then Stripe Elements are embedded and no raw card numbers are handled directly

Given the provider is set to Square
When CheckoutForm renders
Then the Square Web Payments SDK card widget is rendered

Given the provider is set to PayPal
When CheckoutForm renders
Then the PayPal button is rendered

Given the provider is set to SumUp
When CheckoutForm renders
Then the SumUp card widget is rendered

Given a valid discount code is entered in DiscountCodeInput
When validation completes
Then the discounted total is shown before payment submission

Given a payment is submitted
When the provider confirms it
Then the backend confirmPayment mutation is called and the success state is displayed
```
