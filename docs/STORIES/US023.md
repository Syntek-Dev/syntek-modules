# US023 — `syntek-forms-core`: Dynamic Form Engine

**Last Updated**: 10/03/2026 **Version**: 0.2.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a schema-driven form engine so that I can define any form — of any length,
any field type, any conditional logic — once in JSON and have the backend handle rendering schemas,
validation, conditional evaluation, pre-fill resolution, draft persistence, submission storage, spam
protection, and webhook dispatch automatically. No code changes are required per form.

## MoSCoW Priority

**Must Have**

## Architecture — Schema-Driven, Not Code-Driven

Form definitions are stored as JSON in the database. The frontend receives the schema via GraphQL
and renders it — the backend never generates HTML. This means:

- Adding or editing a form requires no deployment — update the schema in the database
- All validation, conditional logic, and pre-fill rules live in the schema — both backend and
  frontend evaluate them (backend is authoritative)
- Any consuming project can define any form shape via the builder API or direct JSON upload
- Forms work in any context: full page, page section, modal, multi-step wizard, or embedded widget

## Acceptance Criteria

```gherkin
Given a form schema defines field B as required when field A equals "yes"
When a submission arrives with A="yes" and B empty
Then a 422 is returned with field-level validation errors and processing stops before storage

Given a form has conditional logic that hides field C when field D is empty
When a submission arrives with D empty and C populated
Then C's value is stripped from the submission before storage and no validation runs for C

Given a form schema defines a pre-fill source for the email field
When an authenticated user requests the form schema
Then the pre-fill value is resolved server-side and returned in the schema response

Given a multi-step form is partially completed and the user leaves
When they return and resume the form
Then their progress is restored from the last saved draft at the last completed step

Given a submission is saved
When a webhook URL is configured on the form definition
Then the signed payload is dispatched via syntek-webhooks within 1 second of save

Given a form has spam protection configured as 'honeypot'
When a submission arrives with the honeypot field populated
Then it returns 200 (to avoid signalling to bots) but the submission is silently discarded

Given a form schema is updated (new version published)
When a new submission is saved
Then the schema version at submission time is snapshotted on the submission record

Given a file upload field is included in a form
When a file is submitted
Then it is stored via syntek-documents and the presigned URL is recorded on the submission

Given a consuming project defines a custom field type in SYNTEK_FORMS['CUSTOM_FIELD_TYPES']
When a form schema references that type
Then the backend applies the registered validation callable for that field
```

## Configuration

All behaviour is controlled via `SYNTEK_FORMS` in Django `settings.py`. Nothing is hardcoded.

```python
SYNTEK_FORMS = {
    # Draft persistence backend — 'session' (default) or 'redis'
    # redis requires syntek-cache (US077); session works out of the box
    'DRAFT_BACKEND': 'session',
    'DRAFT_TTL_HOURS': 48,

    # File upload routing — 'syntek-documents' (US031) or dotted path to custom callable
    'FILE_UPLOAD_BACKEND': 'syntek-documents',
    'MAX_FILE_SIZE_MB': 10,

    # Allowed MIME types for file fields (overridable per-field in schema)
    'ALLOWED_FILE_TYPES': ['image/*', 'application/pdf'],

    # Spam protection — 'honeypot' (no external service), 'turnstile', 'recaptcha', or None
    'SPAM_PROTECTION': 'honeypot',
    'TURNSTILE_SECRET': env('TURNSTILE_SECRET', default=''),
    'RECAPTCHA_SECRET': env('RECAPTCHA_SECRET', default=''),

    # Encrypt submission data at rest using syntek-crypto (US006)
    'ENCRYPT_SUBMISSIONS': True,

    # Per-tenant rate limit for form submissions
    'RATE_LIMIT': '100/hour',

    # Email notification on every submission (optional; requires syntek-notifications-email US087)
    'SUBMISSION_NOTIFY_EMAIL': None,

    # Custom field types — key = type slug; value = dotted path to validator callable
    # callable signature: (value, field_config) -> list[str] (error messages, empty if valid)
    'CUSTOM_FIELD_TYPES': {
        'uk_postcode': 'myapp.validators.validate_uk_postcode',
    },
}
```

## Form Schema

Form schemas are stored as JSON and versioned. The schema structure supports any form layout:

```json
{
  "id": "contact-form",
  "title": "Contact Us",
  "description": "We'll get back to you within 2 working days.",
  "steps": [
    {
      "id": "step-1",
      "title": "Your Details",
      "fields": [
        {
          "id": "first_name",
          "type": "text",
          "label": "First Name",
          "required": true,
          "placeholder": "Enter your first name",
          "max_length": 100
        },
        {
          "id": "enquiry_type",
          "type": "select",
          "label": "Enquiry Type",
          "options": [
            { "value": "general", "label": "General" },
            { "value": "billing", "label": "Billing" }
          ]
        }
      ]
    },
    {
      "id": "step-2",
      "title": "Your Message",
      "fields": [
        { "id": "message", "type": "textarea", "label": "Message", "required": true },
        { "id": "account_ref", "type": "text", "label": "Account Reference" }
      ]
    }
  ],
  "conditions": [
    {
      "when": { "field": "enquiry_type", "op": "eq", "value": "billing" },
      "then": { "show": ["account_ref"], "require": ["account_ref"] }
    }
  ],
  "prefill": {
    "first_name": { "source": "user_profile", "field": "first_name" },
    "email": { "source": "user_profile", "field": "email" },
    "ref": { "source": "url_param", "param": "ref" }
  },
  "spam_protection": "honeypot",
  "submit_label": "Send Message",
  "success_message": "Thank you! We'll be in touch.",
  "success_redirect": null
}
```

Single-step forms use a single `steps` entry. Multi-step forms use multiple entries. The schema is
identical — the frontend decides how to present them.

| ---------------- | ---------------------- | ---------------------------------------------- |

| `date_range`     | From/to date pair      |                                                |
| `select`         | Single-choice dropdown | Static options or API-sourced                  |
| `multi_select`   | Multi-choice dropdown  |                                                |

| `toggle`         | Boolean switch         |                                                |

| `divider`        | Non-input display      | Visual separator                               |

## Conditional Logic

Conditions evaluate against submitted field values. Operators: `eq`, `neq`, `gt`, `lt`, `gte`,
`lte`, `contains`, `not_contains`, `in`, `not_in`, `is_empty`, `is_not_empty`.

Actions: `show`, `hide`, `require`, `unrequire`, `set_value`.

Multiple conditions use `all` (AND) or `any` (OR):

```json
{
  "match": "any",
  "conditions": [
    { "field": "type", "op": "eq", "value": "business" },
    { "field": "employees", "op": "gt", "value": 10 }
  ],
  "then": { "show": ["vat_number"] }
}
```

## Pre-fill Sources

| Source         | Description                                                           |
| -------------- | --------------------------------------------------------------------- |
| `user_profile` | Resolved from the authenticated user's profile via `syntek-users`     |
| `url_param`    | Read from the URL query string at form load time                      |
| `api`          | Dotted path to a callable `(user, tenant) -> value` — project-defined |
| `constant`     | Hardcoded value embedded in schema                                    |

Pre-fill values are resolved server-side when the schema is fetched. Hidden fields are always
resolved server-side and never exposed to the client.

## Data Model

### `FormDefinition`

| Field             | Type      | Notes                                              |
| ----------------- | --------- | -------------------------------------------------- |
| `id`              | UUID      |                                                    |
| `slug`            | SlugField | Unique per tenant                                  |
| `current_version` | FK        | `FormVersion` — null until first version published |
| `created_by`      | FK        | Auth user                                          |
| `is_active`       | bool      | Inactive forms reject submissions with 404         |
| `tenant`          | FK (null) |                                                    |

### `FormVersion`

Immutable once published. Schema changes create a new version.

| Field          | Type      | Notes                            |
| -------------- | --------- | -------------------------------- |
| `form`         | FK        | `FormDefinition`                 |
| `version`      | IntField  | Auto-incrementing per form       |
| `schema`       | JSONField | Full form schema at this version |
| `published_at` | datetime  | Null until published             |
| `published_by` | FK        | Auth user                        |
| `is_published` | bool      | Immutable once True              |

### `FormSubmission`

| Field             | Type      | Notes                                            |
| ----------------- | --------- | ------------------------------------------------ |
| `id`              | UUID      |                                                  |
| `form`            | FK        | `FormDefinition`                                 |
| `form_version`    | FK        | `FormVersion` — snapshot at submission time      |
| `data`            | JSONField | Submitted field values (encrypted if configured) |
| `submitted_by`    | FK (null) | Auth user — null for anonymous submissions       |
| `submitted_at`    | datetime  |                                                  |
| `ip_hash`         | CharField | SHA-256 of submitter IP                          |
| `user_agent_hash` | CharField | SHA-256 of UA string                             |
| `tenant`          | FK (null) |                                                  |

### `FormSubmissionFile`

| Field        | Type      | Notes                          |
| ------------ | --------- | ------------------------------ |
| `submission` | FK        | `FormSubmission`               |
| `field_id`   | CharField | Field ID from schema           |
| `doc_id`     | CharField | `syntek-documents` document ID |
| `filename`   | CharField | Original filename              |
| `mime_type`  | CharField |                                |

### `FormDraft`

| Field         | Type      | Notes                         |
| ------------- | --------- | ----------------------------- |
| `form`        | FK        | `FormDefinition`              |
| `session_key` | CharField | Session or Redis key          |
| `data`        | JSONField | Partial field values          |
| `step`        | IntField  | Last completed step index     |
| `expires_at`  | datetime  | Enforced by `DRAFT_TTL_HOURS` |
| `tenant`      | FK (null) |                               |
