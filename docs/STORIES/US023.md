# US023 — `syntek-forms`: Dynamic Forms Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a schema-driven forms module so that form definitions stored as JSON in the
database drive rendering, validation, and submission storage without any code changes per form.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a form schema defines field B as required when field A equals "yes"
When a submission arrives with A="yes" and B empty
Then a 422 is returned with a field-level validation error for B

Given a form has a configured outbound webhook
When a submission is saved
Then the signed payload is dispatched via `syntek-webhooks`

Given a form schema is updated (new version)
When a new submission is saved
Then the schema version at submission time is snapshotted with the submission record

Given a multi-step form is partially completed and the user leaves
When they return and resume the form
Then their progress is restored from the last completed step

Given a file upload field is included in a form
When a file is submitted
Then it is stored via `syntek-documents` and the presigned URL is recorded on the submission
```
