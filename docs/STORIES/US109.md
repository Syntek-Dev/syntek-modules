## User Story

As a developer, I want accounting service connectors so that I can sync invoices, payments, and
journal entries to Xero, QuickBooks Online, FreeAgent, and Sage Business Cloud — using direct OAuth2
REST APIs where available and the automation bridge (n8n) as a fallback for services without clean
APIs.

## Architecture

Each accounting service adapter implements the `IntegrationAdapter` protocol from
`syntek-integrations-core`. Direct REST API adapters (Xero, QuickBooks Online, FreeAgent, Sage
Business Cloud) are provided out of the box. For accounting services not listed here — or for
desktop accounting software (e.g. Sage 50, Kashflow) that lacks a REST API — the recommended path is
`syntek-integrations-automation` (US108) with an n8n workflow handling the sync. This avoids brittle
screen-scraping adapters inside Syntek.

Sync direction is always **Syntek → accounting service** (one-way push). Consuming projects that
need bidirectional sync should implement reconciliation logic in their own codebase via the domain
event bus.

## Configuration

```python
SYNTEK_INTEGRATIONS_ACCOUNTING = {
    # Enable only the services the project uses
    'ENABLED_ADAPTERS': ['xero'],   # 'xero' | 'quickbooks' | 'freeagent' | 'sage'

    # Xero
    'XERO': {
        'CLIENT_ID': env('XERO_CLIENT_ID'),
        'CLIENT_SECRET': env('XERO_CLIENT_SECRET'),
        'SCOPES': ['accounting.transactions', 'accounting.contacts'],
        'TENANT_ID_ENV': 'XERO_TENANT_ID',   # Resolved per-tenant at runtime
    },

    # QuickBooks Online
    'QUICKBOOKS': {
        'CLIENT_ID': env('QBO_CLIENT_ID'),
        'CLIENT_SECRET': env('QBO_CLIENT_SECRET'),
        'SANDBOX': False,
    },

    # FreeAgent
    'FREEAGENT': {
        'CLIENT_ID': env('FREEAGENT_CLIENT_ID'),
        'CLIENT_SECRET': env('FREEAGENT_CLIENT_SECRET'),
    },

    # Sage Business Cloud
    'SAGE': {
        'CLIENT_ID': env('SAGE_CLIENT_ID'),
        'CLIENT_SECRET': env('SAGE_CLIENT_SECRET'),
        'COUNTRY_CODE': 'GB',   # Sage country-specific endpoint routing
    },

    # Sync behaviour
    'SYNC_ON_INVOICE_PAID': True,      # Auto-sync when syntek-invoicing marks an invoice paid
    'SYNC_ON_PAYMENT': True,           # Auto-sync when syntek-payments records a transaction
    'SYNC_JOURNAL_ENTRIES': False,     # Opt-in — syncs syntek-accounting journal entries
    'SYNC_RETRY_COUNT': 3,
}
```

## Acceptance Criteria

```gherkin
Given Xero adapter is enabled and OAuth2 credentials are stored
When syncInvoice(invoice_id) is called
Then the invoice is posted to Xero Accounting API and the AccountingSyncRecord is created with status 'synced' and the Xero invoice ID

Given a Xero sync fails due to a 422 validation error
When the error is returned
Then an AccountingSyncRecord is created with status 'failed', the error detail is stored, and the event 'accounting.sync_failed' is fired via syntek-bus

Given SYNC_ON_INVOICE_PAID = True and a syntek-invoicing invoice transitions to 'paid'
When the domain event is received via syntek-bus
Then syncInvoice is enqueued as a Celery task for the configured adapter

Given QuickBooks Online adapter is enabled
When an invoice is synced
Then the invoice is created in QBO as a SalesReceipt or Invoice depending on payment status, and the QBO entity ID is stored

Given FreeAgent adapter is enabled
When a contact does not yet exist in FreeAgent
Then the contact is created first, then the invoice is posted against the new contact

Given ENABLED_ADAPTERS = ['xero', 'quickbooks']
When both adapters are active for a tenant
Then sync runs for both services independently and failures in one do not block the other
```
