# US113 — `@syntek/ui-email-builder`: Drag-and-Drop Email Template Builder

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

## User Story

As a developer, I want a React email template builder so that users can visually compose email
templates with drag-and-drop blocks, insert personalisation variables, preview in desktop and mobile
viewports, and save directly to `syntek-email-core` — without writing HTML.

## Acceptance Criteria

```gherkin
Given the EmailBuilder is rendered
When a user drags a TextBlock from the block library onto the canvas
Then the block is inserted at the drop position and the canvas renders the updated order

Given a TextBlock is selected
When the user opens the variable picker
Then the available variables from the template's variables_schema are listed; selecting one
inserts the {{ variable_name }} token at the cursor position in the text editor

Given the user switches to mobile preview
When the preview tab is selected
Then the email canvas re-renders at 375px width with mobile-specific layout adjustments

Given the user clicks "Send Test"
When they enter an email address and confirm
Then the previewEmailTemplate mutation fires to render the current blocks, then sendEmail is
called with the rendered HTML — the recipient receives the actual email

Given the user clicks Save
When the save mutation resolves
Then the current block state is serialised to JSON, stored as EmailTemplate.html_body, and the
template version is not published until the user explicitly clicks Publish

Given a template is loaded from syntek-email-core
When emailTemplate is queried
Then the block JSON is deserialised and the canvas restores the exact block configuration
```

## Components

| Component         | Description                                                                |
| ----------------- | -------------------------------------------------------------------------- |
| `EmailBuilder`    | Root composer — canvas + sidebar + top toolbar; manages builder state      |
| `EmailCanvas`     | Ordered list of rendered blocks; drop target for new and reordered blocks  |
| `BlockLibrary`    | Sidebar panel — draggable block type cards grouped by category             |
| `BlockEditor`     | Contextual property editor — appears in sidebar when a block is selected   |
| `EmailPreview`    | Iframe rendering of rendered HTML; desktop (600px) and mobile (375px) tabs |
| `VariablePicker`  | Dropdown listing template variables; inserts `{{ name }}` at cursor        |
| `TestSendDialog`  | Modal — recipient email input + send test button                           |
| `TemplateManager` | Browse, load, duplicate, and delete templates from syntek-email-core       |
| `PublishButton`   | Triggers `publishEmailTemplate` mutation; shows current publish status     |

### Block Types

| Block              | Properties                                                                            |
| ------------------ | ------------------------------------------------------------------------------------- |
| `TextBlock`        | Rich text (Tiptap) with variable insertion, font size, colour, alignment              |
| `ImageBlock`       | Image URL (or picker from syntek-media), alt text, link URL, width, alignment         |
| `ButtonBlock`      | Label, URL (variable-aware), background colour, text colour, border radius, alignment |
| `ColumnsBlock`     | Layout: 50/50, 33/67, 67/33, 33/33/33 — each column contains a nested block list      |
| `DividerBlock`     | Colour, thickness, margin top/bottom                                                  |
| `SpacerBlock`      | Height (px)                                                                           |
| `SocialLinksBlock` | Icon set: Twitter/X, LinkedIn, Facebook, Instagram, YouTube — per-icon URL            |
| `HeaderBlock`      | Logo image, company name, background colour, padding                                  |
| `FooterBlock`      | Company name, address lines, unsubscribe link text, legal text, background            |
| `HtmlBlock`        | Raw HTML escape hatch — for advanced users; renders as-is in preview                  |

### Hooks

| Hook                              | Description                                                        |
| --------------------------------- | ------------------------------------------------------------------ |
| `useEmailBuilder(options?)`       | Builder state — blocks array, selected block ID, undo/redo stack   |
| `useEmailTemplate(id?)`           | Fetch existing template; save/publish mutations                    |
| `useEmailPreview(blocks, schema)` | Renders preview HTML from current block state (debounced)          |
| `useBlockDrag()`                  | DnD state — wraps `@hello-pangea/dnd` for block reorder and insert |
| `useVariableSchema(templateId?)`  | Fetches `variables_schema` from template or accepts prop           |

### `useEmailBuilder` State Shape

```ts
interface EmailBuilderState {
  blocks: EmailBlock[]; // Ordered array of block definitions
  selectedBlockId: string | null;
  isDirty: boolean; // True when unsaved changes exist
  isPublished: boolean;
  undo: () => void;
  redo: () => void;
  canUndo: boolean;
  canRedo: boolean;
  insertBlock: (type: BlockType, afterId?: string) => void;
  removeBlock: (id: string) => void;
  updateBlock: (id: string, patch: Partial<EmailBlock>) => void;
  reorderBlocks: (fromIndex: number, toIndex: number) => void;
}
```

### Block Serialisation

Blocks are serialised as a JSON array stored in `EmailTemplate.html_body`. At render time,
`syntek-email-core` deserialises the JSON and produces email-safe HTML via table-based layout
(compatible with Outlook, Gmail, Apple Mail). The serialised format is:

```json
[
  {
    "id": "uuid",
    "type": "TextBlock",
    "props": { "html": "<p>Hello {{ first_name }}</p>", "align": "left" }
  },
  {
    "id": "uuid",
    "type": "ButtonBlock",
    "props": { "label": "Activate Account", "url": "{{ activation_url }}", "bg": "#4F46E5" }
  }
]
```

When `previewEmailTemplate` is called, the server-side renderer converts this JSON + Jinja2
variables to final HTML. The client never generates the final email HTML directly.
