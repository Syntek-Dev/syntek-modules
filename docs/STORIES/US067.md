## User Story

As a developer, I want a fully configurable surveys and feedback module so that I can create
multi-question surveys with conditional logic, collect and analyse responses, calculate NPS scores,
and emit webhooks on completion.

## Acceptance Criteria

```gherkin
Given a survey with conditional questions is published
When surveySchema(id) is queried
Then the full question schema — including display conditions and question types — is returned so
  the frontend can apply conditional logic client-side without additional requests

Given a user submits a survey response
When the response is saved
Then a SurveyResponse record is created, attributed to the user or a session fingerprint if
  anonymous, and a duplicate submission from the same attribution is rejected with a clear error

Given ALLOW_MULTIPLE_RESPONSES is False and the user has already responded
When submitSurveyResponse is called
Then a DuplicateResponseError is returned and no new SurveyResponse is created

Given a survey contains an NPS question
When surveyAnalytics(surveyId) is queried
Then response counts per 0–10 score, the promoter/passive/detractor breakdown, and the calculated
  NPS score are all returned

Given a survey has a configured completion webhook
When a SurveyResponse is created successfully
Then an HMAC-SHA256 signed POST is delivered to the webhook URL with the response payload within
  30 seconds via a Celery task

Given an erasure request is processed for a user
When the GDPR erasure hook fires
Then all SurveyResponse records attributed to that user are anonymised — free text answers are
  replaced with the ERASURE_PLACEHOLDER and the respondent FK is set to null

Given a survey export is requested
When exportSurveyResponses(surveyId, format) mutation completes
Then a Celery task assembles the export (CSV or Excel) and delivers it via the bulk-export
  storage backend (US093) or local fallback
```

## Configuration

```python
SYNTEK_FEEDBACK = {
    # General
    'ALLOW_ANONYMOUS': False,          # permit responses without an authenticated user
    'ALLOW_MULTIPLE_RESPONSES': False, # True = multiple responses per user per survey
    'RESPONSE_WINDOW_DAYS': 0,         # 0 = no expiry; otherwise survey closes after N days
    'MAX_QUESTIONS_PER_SURVEY': 50,

    # Question types enabled
    'ENABLED_QUESTION_TYPES': [        # subset controls which types can be created
        'free_text', 'single_choice', 'multi_choice',
        'rating_scale', 'nps', 'boolean', 'date', 'number',
    ],

    # Conditional logic
    'CONDITIONAL_LOGIC_ENABLED': True, # reuses syntek-forms condition evaluator
    'MAX_CONDITIONS_PER_QUESTION': 5,

    # NPS settings
    'NPS_PROMOTER_THRESHOLD': 9,       # 9–10 = promoter
    'NPS_PASSIVE_THRESHOLD': 7,        # 7–8 = passive; 0–6 = detractor

    # Webhooks
    'WEBHOOK_ENABLED': False,
    'WEBHOOK_URL': '',
    'WEBHOOK_SECRET': '',              # HMAC-SHA256 signing secret
    'WEBHOOK_TIMEOUT_SECONDS': 10,
    'WEBHOOK_MAX_RETRIES': 3,

    # GDPR
    'ERASURE_PLACEHOLDER': '[response removed]',
    'RESPONSE_RETENTION_DAYS': 0,      # 0 = retain indefinitely

    # Export
    'EXPORT_FORMATS': ['csv', 'excel'],

    # Tenant
    'TENANT_SCOPED': True,
    'TENANT_CAN_OVERRIDE_ANONYMOUS': True,
}
```

## GraphQL API

### Queries

| Query                                       | Returns                                                          |
| ------------------------------------------- | ---------------------------------------------------------------- |
| `surveys(status)`                           | List of surveys for tenant                                       |
| `surveySchema(id)`                          | Full schema including questions and conditions                   |
| `surveyResponses(surveyId, page, pageSize)` | Paginated response list (moderator/admin only)                   |
| `surveyAnalytics(surveyId)`                 | Aggregate stats: counts per question, NPS score, completion rate |
| `myResponse(surveyId)`                      | Current user's own response if submitted                         |

### Mutations

| Mutation                                  | Description                         |
| ----------------------------------------- | ----------------------------------- |
| `createSurvey(input)`                     | Create a new survey in draft status |
| `updateSurvey(id, input)`                 | Update survey metadata or questions |
| `publishSurvey(id)`                       | Transition draft → published        |
| `closeSurvey(id)`                         | Transition published → closed       |
| `submitSurveyResponse(surveyId, answers)` | Submit a complete survey response   |
| `exportSurveyResponses(surveyId, format)` | Enqueue async CSV/Excel export      |
