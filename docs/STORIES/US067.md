# US067 — `syntek-feedback`: Surveys & Feedback Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

---

## User Story

As a developer, I want a feedback and surveys module so that I can define multi-question surveys
with conditional logic, collect responses, calculate NPS scores, and export results.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a survey with conditional questions is defined
When `surveySchema(id)` is queried
Then the full question schema including display conditions is returned so the frontend can apply conditional logic client-side

Given a user submits a survey response
When the response is saved
Then a SurveyResponse record is created and duplicate submission from the same user is rejected with a clear error

Given a survey contains an NPS question
When `surveyAnalytics(surveyId)` is queried
Then the response counts for each 0–10 score, the promoter/passive/detractor breakdown, and the calculated NPS score are all returned

Given an erasure request is processed for a user
When the GDPR erasure hook fires
Then all attributed SurveyResponse records for that user are anonymised
```

## Dependencies

- US010 — `syntek-tenancy`
- US023 — `syntek-forms` (shared conditional logic engine)
- US029 — `syntek-gdpr` (erasure hook)
