## Acceptance Criteria

```gherkin
Given a contact is on the global unsubscribe list
When a campaign is dispatched
Then that contact is excluded from the send — no exception, no error, no re-queuing

Given a campaign email is sent and the recipient opens it
When the tracking pixel is loaded
Then a CampaignEmailOpen record is created with recipient ID, timestamp, and user agent hash
And unique_opens and open_rate on CampaignStats are updated

Given a campaign email contains tracked links
When the recipient clicks a link
Then a CampaignEmailClick record is created with the resolved URL, timestamp, and position
And the CampaignClickMap entry for that URL is incremented
And the recipient is redirected to the original URL with no perceptible delay

Given a campaign is sent and the provider reports a hard bounce
When the bounce webhook is processed
Then a CampaignBounce record (type: hard) is created
And the recipient's email is flagged as hard-bounced and excluded from all future campaign sends

Given a campaign is sent and the provider reports a soft bounce
When the bounce webhook is processed
Then a CampaignBounce record (type: soft) is created
And the recipient is not excluded unless soft_bounce_threshold is reached

Given a campaign send completes
When CampaignStats are recomputed
Then open_rate = unique_opens / delivered, click_rate = unique_clicks / delivered,
     bounce_rate = (hard_bounces + soft_bounces) / sent — all as percentages rounded to 4dp

Given an automation sequence is configured with three steps at 0h, 24h, and 72h delays
When a subscriber is enrolled
Then Step 1 sends immediately, Step 2 at 24h, Step 3 at 72h — paused if they unsubscribe

Given `syntek-gdpr` is installed and an erasure request is processed for a subscriber
When the erasure handler runs
Then their email address and all open/click/bounce records are anonymised — not deleted
And anonymised records retain aggregate counts for analytics accuracy

Given `syntek-payments` is installed and a subscriber purchases after clicking a campaign link
When the order is confirmed within the attribution window
Then a CampaignRevenue record is created with order value and campaign attribution
```

## Architecture Decision — Marketing vs Transactional

`syntek-email-marketing` handles **proactive marketing campaigns** (newsletters, drip sequences,
broadcast sends). It is a separate module from the transactional notification sub-modules
(`syntek-notifications-email`, US087):

| Dimension        | `syntek-notifications-email` (US087)         | `syntek-email-marketing` (US038)                |
| ---------------- | -------------------------------------------- | ----------------------------------------------- |
| Trigger          | System event (order placed, password reset)  | Marketer-scheduled or automation-triggered      |
| Recipients       | One-to-one (specific user)                   | One-to-many (subscriber lists)                  |
| GDPR legal basis | Legitimate interest / contract               | Explicit consent (opt-in only)                  |
| Content          | Notification template                        | Campaign HTML / MJML builder                    |
| Analytics        | Delivery + bounce log                        | Full campaign analytics (see below)             |
| Infrastructure   | `syntek-notifications-core` channel registry | Uses `syntek-notifications-email` as send layer |

## Analytics Model

### Campaign-Level Models

#### `CampaignSendRecord`

One row per recipient per campaign send. Tracks the delivery lifecycle.

| Field              | Type        | Notes                                           |
| ------------------ | ----------- | ----------------------------------------------- |
| `campaign`         | FK          | `Campaign`                                      |
| `send_batch`       | FK          | `CampaignSendBatch` — groups concurrent sends   |
| `recipient_email`  | CharField   | Stored hashed post-GDPR-erasure                 |
| `subscriber`       | FK (null)   | `ContactSubscription` — null if contact deleted |
| `sent_at`          | datetime    |                                                 |
| `status`           | CharField   | queued / sent / delivered / failed / bounced    |
| `first_opened_at`  | datetime    | null until first open                           |
| `total_opens`      | PositiveInt | incremented on each open event                  |
| `unique_opens`     | bool        | true after first open recorded                  |
| `first_clicked_at` | datetime    | null until first click                          |
| `total_clicks`     | PositiveInt | incremented on each click event                 |
| `unique_clicks`    | bool        | true after first click recorded                 |
| `unsubscribed_at`  | datetime    | null until unsubscribe event                    |
| `provider_msg_id`  | CharField   | Provider's message ID for webhook correlation   |

#### `CampaignEmailOpen`

One row per open event (pixel load). Deduplicated for unique open tracking via `CampaignSendRecord`.

| Field             | Type      | Notes                                              |
| ----------------- | --------- | -------------------------------------------------- |
| `send_record`     | FK        | `CampaignSendRecord`                               |
| `opened_at`       | datetime  |                                                    |
| `email_client`    | CharField | Extracted from user agent (e.g. Gmail, Apple Mail) |
| `user_agent_hash` | CharField | SHA-256 of raw UA — not stored in clear            |
| `ip_hash`         | CharField | SHA-256 of IP — not stored in clear; GDPR-safe     |

#### `CampaignEmailClick`

One row per link click event.

| Field             | Type        | Notes                                  |
| ----------------- | ----------- | -------------------------------------- |
| `send_record`     | FK          | `CampaignSendRecord`                   |
| `url`             | URLField    | Original (pre-redirect) URL            |
| `link_position`   | PositiveInt | Ordinal position of link in email body |
| `clicked_at`      | datetime    |                                        |
| `user_agent_hash` | CharField   | SHA-256 of raw UA                      |
| `ip_hash`         | CharField   | SHA-256 of IP                          |

#### `CampaignBounce`

| Field         | Type      | Notes                               |
| ------------- | --------- | ----------------------------------- |
| `send_record` | FK        | `CampaignSendRecord`                |
| `bounce_type` | CharField | hard / soft                         |
| `bounce_code` | CharField | Provider bounce classification code |
| `message`     | TextField | Provider bounce reason              |
| `bounced_at`  | datetime  |                                     |

#### `CampaignUnsubscribe`

| Field             | Type      | Notes                               |
| ----------------- | --------- | ----------------------------------- |
| `send_record`     | FK        | `CampaignSendRecord`                |
| `unsubscribed_at` | datetime  |                                     |
| `method`          | CharField | link / api / admin / spam-complaint |

### Aggregate / Materialised Stats

#### `CampaignStats`

One row per campaign, recomputed after send completion and updated incrementally via signals.

| Field              | Type         | Notes                                           |
| ------------------ | ------------ | ----------------------------------------------- |
| `campaign`         | OneToOne FK  |                                                 |
| `computed_at`      | datetime     | Last recomputation time                         |
| `total_recipients` | PositiveInt  |                                                 |
| `total_sent`       | PositiveInt  |                                                 |
| `total_delivered`  | PositiveInt  | sent − hard_bounces − soft_bounces              |
| `total_failed`     | PositiveInt  | Provider-reported send failures                 |
| `hard_bounces`     | PositiveInt  |                                                 |
| `soft_bounces`     | PositiveInt  |                                                 |
| `bounce_rate`      | DecimalField | (hard + soft) / sent — 4dp                      |
| `total_opens`      | PositiveInt  | Sum of all open events                          |
| `unique_opens`     | PositiveInt  | Count of send_records with unique_opens = True  |
| `open_rate`        | DecimalField | unique_opens / delivered — 4dp                  |
| `total_clicks`     | PositiveInt  | Sum of all click events                         |
| `unique_clicks`    | PositiveInt  | Count of send_records with unique_clicks = True |
| `click_rate`       | DecimalField | unique_clicks / delivered — 4dp                 |
| `unsubscribes`     | PositiveInt  |                                                 |
| `unsubscribe_rate` | DecimalField | unsubscribes / delivered — 4dp                  |
| `spam_complaints`  | PositiveInt  |                                                 |

| -------------------- | ------------ | ---------------------------------------------- |
