## User Story

As a developer, I want a fully configurable groups module so that consuming projects can define
their own group types, membership roles, custom metadata fields, and invitation flows — enabling any
organisational model (teams, departments, chapters, cohorts, committees) without code changes.

## Configuration Contract

```python
# settings.py
SYNTEK_GROUPS = {
    # GROUP_TYPES — define the group shapes available in this project.
    # Each key becomes a valid `type` value on the Group model.
    # All keys default to an empty dict — every GROUP_TYPES entry is required to be explicit.
    'GROUP_TYPES': {
        # Example — consuming projects replace/extend this entirely:
        # 'team': {
        #     'label': 'Team',                 # Human-readable singular name
        #     'label_plural': 'Teams',         # Human-readable plural name
        #     'roles': ['owner', 'admin', 'member', 'viewer'],
        #     'default_role': 'member',        # Role assigned on join / accepted invite
        #     'allow_nesting': True,           # Can this group type have child groups?
        #     'max_depth': 3,                  # 0 = flat; 1 = one level of nesting; etc.
        #     'self_join_allowed': False,      # User can join without an invitation
        #     'invitation_required': True,     # Must receive an invitation to join
        #     'single_membership': False,      # User can only be in one group of this type
        #                                      # per tenant (e.g. one 'department' per user)
        # },
    },

    # GROUP_SCHEMA — custom metadata fields stored on every Group record.
    # Applied to all group types; per-type schemas are not yet supported.
    'GROUP_SCHEMA': {
        # 'field_name': {
        #     'type': 'str|text|int|float|bool|date|email|url|choice|json',
        #     'required': bool,         # default False
        #     'default': Any,           # used when field is absent on create
        #     'choices': [(value, label)],  # required when type == 'choice'
        #     'description': str,       # surfaced in GraphQL introspection
        # }
    },

    # Invitation settings
    'INVITATION_ENABLED': True,
    'INVITATION_EXPIRY_HOURS': 72,     # int — 0 = never expires
    'INVITATION_MAX_USES': 1,          # int — 0 = unlimited (shareable link behaviour)

    # Permission inheritance — when True, assigning a permission bundle to a group
    # propagates those permissions to all direct and descendant members.
    'PERMISSION_INHERITANCE_ENABLED': True,

    # Maximum users that can be bulk-assigned to a group in a single operation.
    'BULK_ASSIGN_MAX': 500,

    # Soft delete — archived groups are hidden from default querysets but
    # remain in the database. Access via Group.all_objects.
    'SOFT_DELETE': True,

    # GraphQL exposure — set False to disable all group-related resolvers.
    'EXPOSE_VIA_GRAPHQL': True,

    # Require the requesting user to have the 'manage_groups' permission for
    # all write operations (create, update, archive, membership mutations).
    'REQUIRE_ADMIN_FOR_WRITE': True,

    # Enable full-text search across group name, slug, and GROUP_SCHEMA fields
    # with 'searchable: True'.
    'SEARCH_ENABLED': True,
}
```

## Acceptance Criteria

```gherkin
# GROUP_TYPES registry

Given GROUP_TYPES defines 'team' with max_depth = 3
When an attempt is made to nest a group at depth 4
Then a MaxDepthExceededError is raised

Given GROUP_TYPES defines 'department' with single_membership = True
When a user is already a member of one department
And they attempt to join a second department
Then a SingleMembershipViolationError is raised

Given GROUP_TYPES defines 'cohort' with self_join_allowed = True
When an authenticated user calls addGroupMember without an invitation
Then they are added as a member with the configured default_role

Given GROUP_TYPES defines 'committee' with invitation_required = True
When a user attempts to join without an invitation token
Then an InvitationRequiredError is returned

# Organisational hierarchy

Given a group hierarchy Organisation > Department > Team
When a user is a member of Team
Then they are queryable as a descendant of Organisation without a cross-tenant query

Given PERMISSION_INHERITANCE_ENABLED is True
And a permission bundle is assigned to a Department group
When a member of a nested Team makes a request
Then they inherit all permissions in that bundle

# Invitations

Given INVITATION_ENABLED is True and INVITATION_MAX_USES is 1
When a user accepts an invitation token
Then the invitation is consumed and subsequent accepts return InvitationAlreadyUsedError

Given INVITATION_MAX_USES is 0 (shareable link)
When five different users accept the same invitation token within the expiry window
Then all five are added as members with the configured role

Given INVITATION_EXPIRY_HOURS is 72
When an invitee attempts to accept an invitation older than 72 hours
Then an InvitationExpiredError is raised

# GROUP_SCHEMA custom fields

Given GROUP_SCHEMA defines a required 'region' choice field
When a group is created without the 'region' field
Then a ValidationError is raised referencing the missing required field

Given GROUP_SCHEMA defines a 'budget_code' str field
When a group is created with a valid budget_code
Then the value is stored in custom_fields and returned in the GraphQL response

# Soft delete

Given SOFT_DELETE is True
When archiveGroup is called for a group
Then Group.objects.get raises DoesNotExist
But Group.all_objects.get returns the group with is_active = False

# Bulk assign

Given BULK_ASSIGN_MAX is 500
When bulkAddGroupMembers is called with 501 user IDs
Then a BulkAssignLimitExceededError is raised before any members are written

Given bulkAddGroupMembers is called with 100 user IDs
When 3 of those users already have single_membership violations
Then those 3 are rejected individually with errors and the remaining 97 succeed

# Tenant isolation

Given two tenants A and B each with a group named 'Engineering'
When tenant A queries their groups
Then only tenant A's group is returned — tenant B's record is never visible
```

## Python API

```python
from syntek_groups import (
    create_group,
    get_group,
    update_group,
    archive_group,
    add_member,
    remove_member,
    bulk_add_members,
    invite_member,
    accept_invitation,
    revoke_invitation,
    get_members,
    get_user_groups,
    get_descendants,
    get_ancestors,
)

# Create a group
group = create_group(
    tenant=request.tenant,
    name="Engineering",
    type_key="team",
    parent=department_group,     # optional; None = root group
    custom_fields={"region": "UK"},
)

# Membership
add_member(group_id=group.id, user_id=user.id, role="member", tenant=request.tenant)
bulk_add_members(group_id=group.id, user_ids=[...], role="member", tenant=request.tenant)
remove_member(group_id=group.id, user_id=user.id, tenant=request.tenant)

# Invitations
invitation = invite_member(
    group_id=group.id,
    email="alice@example.com",
    role="member",
    invited_by=request.user,
    tenant=request.tenant,
)
membership = accept_invitation(token=invitation.token, user=user, tenant=request.tenant)
revoke_invitation(token=invitation.token, group_id=group.id, tenant=request.tenant)

# Hierarchy traversal
descendants = get_descendants(group_id=root.id, tenant=request.tenant)
ancestors   = get_ancestors(group_id=leaf.id, tenant=request.tenant)
user_groups = get_user_groups(user_id=user.id, tenant=request.tenant, type_key="team")
```

## Data Model

```python
class Group(models.Model):
    tenant          = TenantForeignKey(...)
    name            = models.CharField(max_length=150)
    slug            = models.SlugField(max_length=150)
    type            = models.CharField(max_length=100)      # key in GROUP_TYPES
    parent          = models.ForeignKey('self', null=True, blank=True, on_delete=models.PROTECT)
    depth           = models.PositiveSmallIntegerField(default=0)  # denormalised for fast queries
    custom_fields   = models.JSONField(default=dict)
    is_active       = models.BooleanField(default=True)
    created_at      = models.DateTimeField(auto_now_add=True)
    updated_at      = models.DateTimeField(auto_now=True)

    objects         = ActiveGroupManager()      # excludes is_active=False
    all_objects     = models.Manager()

class GroupMembership(models.Model):
    group      = models.ForeignKey(Group, on_delete=models.CASCADE)
    user       = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE)
    role       = models.CharField(max_length=100)
    joined_at  = models.DateTimeField(auto_now_add=True)
    invited_by = models.ForeignKey(
        settings.AUTH_USER_MODEL, null=True, blank=True,
        on_delete=models.SET_NULL, related_name='+'
    )

class GroupInvitation(models.Model):
    group          = models.ForeignKey(Group, on_delete=models.CASCADE)
    invited_by     = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE)
    invited_email  = models.EmailField()
    token          = models.UUIDField(default=uuid.uuid4, unique=True)
    role           = models.CharField(max_length=100)
    expires_at     = models.DateTimeField(null=True, blank=True)  # null = never expires
    max_uses       = models.PositiveIntegerField(default=1)       # 0 = unlimited
    uses_count     = models.PositiveIntegerField(default=0)
    revoked        = models.BooleanField(default=False)
    created_at     = models.DateTimeField(auto_now_add=True)
```
