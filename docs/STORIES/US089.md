# US089 — `syntek-notifications-push`: Push Notification Channel

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a push notification channel adapter so that I can send FCM and APNs push
notifications to mobile devices — with device token management, automatic invalid token removal, and
a React Native registration helper.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a user has registered a device token
When a push notification is dispatched to their account
Then the notification payload is delivered to that device via the correct provider (FCM or APNs)

Given FCM or APNs reports that a device token is invalid or unregistered
When the delivery response is processed
Then the token is immediately removed from the DeviceToken table — it is never retried

Given a user has multiple registered devices
When a push notification is dispatched
Then the notification is delivered to all registered devices for that user in parallel

Given a push notification is dispatched when the user has no registered device tokens
When the dispatch runs
Then the push channel is skipped silently — no exception raised

Given a user logs out of a device
When the logout mutation is called
Then the device token for that session is deleted from the DeviceToken table
```

## Tasks

- [ ] **Red phase:** write failing tests — FCM delivery, APNs delivery, invalid token removal,
      multi-device fan-out, no-token no-op, logout token deletion
- [ ] Implement `SYNTEK_NOTIFICATIONS_PUSH` settings schema (FCM server key / service account JSON,
      APNs key ID, team ID, bundle ID, APNs environment: sandbox/production)
- [ ] Implement `DeviceToken` model — fields: user FK, token, platform (ios/android/web),
      created_at, last_seen_at; unique constraint on (user, token, platform)
- [ ] Implement FCM adapter — uses `firebase-admin` Python SDK; sends to individual tokens; handles
      `messaging/invalid-registration-token` and `messaging/registration-token-not-registered`
      responses by queuing token deletion
- [ ] Implement APNs adapter — uses `aioapns` or `httpx` with HTTP/2; constructs standard APS
      payload; handles 410 Gone and 400 BadDeviceToken by queuing token deletion
- [ ] Implement invalid token auto-removal Celery task — batches deletions after each push send;
      never blocks the delivery path
- [ ] Implement multi-device fan-out — dispatches in parallel using `celery.group`; each token is an
      independent sub-task so a single bad token does not block others
- [ ] Implement Strawberry mutation: `registerDeviceToken(token, platform)` — upserts the token for
      the current authenticated user
- [ ] Implement logout hook — `deregisterDeviceToken(token)` mutation removes the token; called by
      `syntek-auth` on session termination
- [ ] Implement React Native registration helper — exported as a thin JS/TS client library in
      `packages/backend/syntek-notifications-push/client/` that wraps `registerDeviceToken` over
      GraphQL; consumed by `@syntek/mobile-notifications` (US059)
- [ ] Register the push adapter with the `syntek-notifications-core` channel registry via
      `AppConfig.ready()`
- [ ] Create `syntek.manifest.toml` — module ID, settings keys with types and defaults, and
      `INSTALLED_APPS` entry
- [ ] **Green phase:** all push channel tests pass
