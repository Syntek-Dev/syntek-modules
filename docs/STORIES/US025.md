# US025 — `syntek-payments-core`: Multi-Provider Payment Core

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a provider-agnostic payments core module so that I can accept one-off
payments and manage subscriptions through a consistent API regardless of which payment provider
(Stripe, Square, PayPal, SumUp) is configured.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a SYNTEK_PAYMENTS['PROVIDER'] is set to any supported provider
When a payment intent is created
Then the core module delegates to the configured adapter without the consumer needing provider-specific code

Given a discount code with a 20% discount and 100 usage limit is applied at checkout
When the payment intent is created
Then the total is reduced by 20% and the usage counter increments atomically

Given a discount code has reached its usage limit
When another user attempts to apply it
Then the mutation returns a validation error and the payment intent is not created

Given a payment link is generated for an invoice
When the link is opened
Then the configured payment form renders with the correct amount and reference

Given a webhook arrives for any provider
When the handler is looked up from SYNTEK_PAYMENTS['WEBHOOK_HANDLERS']
Then the correct handler processes the event and updates transaction state

Given a webhook arrives with an invalid signature
When verification runs
Then the payload is rejected with 401 and not processed

Given a subscription payment fails
When the retry schedule runs
Then payment is retried per configured intervals and subscription transitions to past_due
```
