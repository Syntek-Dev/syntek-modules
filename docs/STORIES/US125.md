## User Story

As a developer, I want a web error pages package so that I can render fully branded, accessible
error pages for every error type (404, 500, 403, 401, 429, 503), catch React render errors via an
ErrorBoundary, and configure the action buttons, messaging, and redirect behaviour per error type —
with no hardcoded content.

## Acceptance Criteria

```gherkin
Given the Next.js app receives a 404 response
When NotFoundPage renders
Then the configured title, description, and CTA button (e.g. "Go Home") are shown; if
  NOT_FOUND_REDIRECT is set, a countdown timer auto-redirects after NOT_FOUND_REDIRECT_DELAY seconds

Given a React render error is thrown inside an ErrorBoundary
When the boundary catches it
Then the configured fallback UI renders (default or custom via fallback prop); the error and
  correlationId are passed to the registered onError callback; the page never shows a blank screen

Given the user is unauthenticated and tries to access a protected route
When UnauthorizedPage renders
Then if REDIRECT_TO_LOGIN is True, the page auto-redirects to LOGIN_URL after the configured
  delay; otherwise it renders the 401 page with a "Sign In" CTA

Given the user is authenticated but forbidden from the route
When ForbiddenPage renders
Then the 403 page renders with a "Go Back" and optionally a "Contact Support" CTA; no redirect
  is performed unless FORBIDDEN_REDIRECT is configured

Given a 429 Too Many Requests response is received
When RateLimitedPage renders
Then a "please wait" message renders with the Retry-After duration displayed to the user; a
  retry button re-attempts the last navigation after the wait period

Given the site is in maintenance mode (503)
When MaintenancePage renders
Then the configured maintenance message and estimated recovery time are displayed; no other
  navigation is possible from this page
```

## Configuration

```typescript
// Pass to SyntekErrorProvider at app root
interface SyntekErrorConfig {
  // 404
  notFound?: {
    title?: string;
    description?: string;
    ctaLabel?: string;
    ctaHref?: string; // default: '/'
    redirectTo?: string; // auto-redirect target; omit to disable
    redirectDelaySeconds?: number; // default: 5
  };

  // 401 Unauthorized
  unauthorized?: {
    title?: string;
    description?: string;
    redirectToLogin?: boolean; // default: true
    loginUrl?: string; // default: '/login'
    redirectDelaySeconds?: number; // default: 3
  };

  // 403 Forbidden
  forbidden?: {
    title?: string;
    description?: string;
    showContactSupport?: boolean; // default: false
    contactSupportHref?: string;
    redirectTo?: string;
  };

  // 429 Rate Limited
  rateLimited?: {
    title?: string;
    description?: string;
    showRetryButton?: boolean; // default: true
  };

  // 500 Server Error
  serverError?: {
    title?: string;
    description?: string;
    showCorrelationId?: boolean; // display correlation ID to user for support; default: true
    ctaLabel?: string;
    ctaHref?: string;
  };

  // 503 Maintenance
  maintenance?: {
    title?: string;
    message?: string;
    estimatedRecoveryTime?: string;
    logoSrc?: string;
  };

  // Global
  onError?: (error: Error, correlationId: string) => void; // e.g. Sentry.captureException
}
```

## Components

| Component             | Description                                                                                   |
| --------------------- | --------------------------------------------------------------------------------------------- |
| `SyntekErrorProvider` | Context provider — wraps app root; provides error config to all components                    |
| `NotFoundPage`        | 404 page with configurable title, description, CTA, and optional auto-redirect with countdown |
| `ForbiddenPage`       | 403 page — authenticated but lacks permission; optional redirect                              |
| `UnauthorizedPage`    | 401 page — unauthenticated; optional auto-redirect to login with countdown                    |
| `ServerErrorPage`     | 500 page — shows correlation ID for support; optional Sentry link                             |
| `RateLimitedPage`     | 429 page — shows Retry-After wait period with countdown and retry button                      |
| `MaintenancePage`     | 503 page — maintenance message, logo, estimated recovery time                                 |
| `ErrorBoundary`       | React class error boundary — catches render errors; renders configurable fallback             |
| `ErrorFallback`       | Default fallback UI for ErrorBoundary — minimal, always renders without JS dependencies       |
| `ErrorPageShell`      | Shared layout wrapper used by all error pages — centres content, applies design tokens        |

## Hooks

| Hook                 | Returns                                                                |
| -------------------- | ---------------------------------------------------------------------- |
| `useErrorConfig()`   | Current error configuration from `SyntekErrorProvider`                 |
| `useErrorBoundary()` | `{ resetBoundary }` — programmatically reset the nearest ErrorBoundary |

## Next.js Integration

```tsx
// app/not-found.tsx
export default function NotFound() {
  return <NotFoundPage />;
}

// app/error.tsx
("use client");
export default function Error({ error, reset }) {
  return (
    <ErrorBoundary onReset={reset}>
      <ServerErrorPage />
    </ErrorBoundary>
  );
}

// app/global-error.tsx — root-level catch
("use client");
export default function GlobalError({ error, reset }) {
  return <ErrorFallback error={error} reset={reset} />;
}
```
