# US098 — `syntek-payments-sumup`: SumUp Adapter

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a SumUp-specific payments adapter so that I can process card-present and
card-not-present payments via SumUp through the syntek-payments-core interface.

## MoSCoW Priority

**Could Have**

## Acceptance Criteria

```gherkin
Given SumUp adapter is configured
When a checkout is created
Then a SumUp checkout ID is returned for the front-end to render the SumUp payment widget

Given a SumUp payment succeeds
When the webhook is received
Then a Transaction record is created with status succeeded

Given a SumUp webhook arrives with an invalid signature
When verification runs
Then the request is rejected with 401

Given a refund is issued for a SumUp transaction
When the refund is processed
Then a Refund record is created and the original Transaction is updated
```
