# US061 — `@syntek/mobile-sync`: Offline SQLite Sync

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Mobile Packages

---

## User Story

As a developer, I want an offline SQLite sync package so that I can cache data locally on device,
allow reads and writes while offline, and sync changes to the backend when connectivity is restored.

## MoSCoW Priority

**Could Have**

## Acceptance Criteria

```gherkin
Given the device loses network connectivity
When the user creates or updates a record
Then the change is written to the local SQLite database and queued for sync

Given network connectivity is restored
When the sync engine runs
Then all queued local changes are sent to the backend GraphQL mutations in order

Given the backend returns a conflict (the record was updated remotely while offline)
When the conflict resolver runs
Then the configured conflict strategy (last-write-wins or manual merge) is applied and the user is notified if manual intervention is needed

Given a large dataset is cached locally
When the user queries the local store
Then the query returns in under 50ms with correct results from SQLite
```
