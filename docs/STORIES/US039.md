# US039 — `syntek-api-keys`: Developer API Keys Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 5 — Backend
Compliance & Media

---

## User Story

As a developer, I want an API keys module so that I can issue scoped developer keys that
authenticate requests to my GraphQL API without requiring a user session.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given an API key is issued with scope `orders:read`
When a request is authenticated with that key
Then only `orders:read` resolvers are accessible — `orders:write` returns 403

Given an API key is revoked
When a subsequent request uses that key
Then a 401 is returned immediately

Given an API key is issued
When the key is stored
Then only a hashed version is persisted — the plaintext key is shown to the developer once only

Given a request is authenticated with an API key
When the request is processed
Then `syntek-audit` records the API key ID (not the plaintext key) as the actor
```
