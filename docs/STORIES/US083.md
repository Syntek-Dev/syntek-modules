# US083 — `@syntek/mobile-security`: React Native Security Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a React Native security package so that Syntek mobile applications have
consistent input sanitisation, deep link validation, WebView URL injection protection, and
screenshot prevention — with clear documentation of which web security concepts do not apply to
mobile and why.

## MoSCoW Priority

**Must Have**

## Design Rationale

React Native applications do not serve HTTP responses — they are clients, not servers. HTTP security
headers (CSP, HSTS, X-Frame-Options, etc.) are server-side concepts. This package's role is:

1. Provide input sanitisation utilities consistent with `@syntek/ui-security/sanitise`
2. Provide deep link and WebView URL validation to prevent URL injection attacks
3. Provide a `SyntekMobileSecurityConfig` component for mobile-specific security settings
4. Clearly document what is and is not applicable in the mobile context

## Entry Points

### `@syntek/mobile-security/sanitise` — Input Sanitisation

```typescript
import { stripTags, escapeHtml, sanitiseText } from "@syntek/mobile-security/sanitise";

// stripTags(html: string): string
// Plain text extraction. Removes all HTML tags. Identical API to web equivalent.
// Example: stripTags('<p>Hello <strong>world</strong></p>') → 'Hello world'
// Nested tags handled correctly.
//
// Implementation: regex-based stripping only. No DOMPurify — not available in React Native.
// Consumers handling rich text in mobile should sanitise server-side via
// syntek-security-validator before sending to the client.

// escapeHtml(str: string): string
// Escapes HTML entities. Identical API to web equivalent.
// Escapes: <, >, &, ", '
// Example: escapeHtml('<img src="x" onerror="alert()">') → '&lt;img src=&quot;x&quot; onerror=&quot;alert()&quot;&gt;'

// sanitiseText(input: string): string
// Strips HTML tags and trims leading/trailing whitespace.
// Example: sanitiseText('  <p>Hello</p>  ') → 'Hello'
```

### `@syntek/mobile-security/deeplinks` — Link Validation

```typescript
import {
  validateDeepLink,
  validateWebViewUrl,
  type WebViewUrlOptions,
} from "@syntek/mobile-security/deeplinks";

// validateDeepLink(url: string, allowedSchemes: string[]): boolean
// Validates that a deep link URL matches one of the allowed schemes.
// Prevents open redirect and URL-injection attacks via deep links.
//
// Returns true only if:
//   - url is a non-empty string
//   - url matches exactly one scheme in allowedSchemes
//   - url is not malformed
//
// Returns false for: empty/null URL, non-matching scheme, or malformed URL.
//
// Example:
//   validateDeepLink('myapp://home', ['myapp://', 'https://myapp.com'])  → true
//   validateDeepLink('http://evil.com', ['myapp://', 'https://myapp.com']) → false
//   validateDeepLink('', ['myapp://'])  → false

// validateWebViewUrl(url: string, options?: WebViewUrlOptions): boolean
// Validates URLs before loading them in expo-web-view or bare React Native WebView.
//
// WebView URL injection is a real mobile attack surface — a malicious URL passed to
// a WebView can exfiltrate native context, execute JS in a trusted origin, or redirect
// to phishing pages. This helper enforces an allowlist-based approach.
//
// Options:
//   allowedOrigins: string[]       — exact origins allowed (e.g. ['https://myapp.com'])
//                                    Subdomains must be listed explicitly — exact match required.
//   requireHttps?: boolean         — default true; rejects any non-HTTPS URL
//   blockDataUrls?: boolean        — default true; rejects data: URIs (XSS vector)
//   blockJavascriptUrls?: boolean  — default true; rejects javascript: URIs
//
// Returns true only if:
//   - url is non-empty
//   - url's origin is in allowedOrigins (exact match)
//   - if requireHttps: true, scheme is 'https' (or 'http' if requireHttps: false)
//   - if blockDataUrls: true, url does not start with 'data:'
//   - if blockJavascriptUrls: true, url does not start with 'javascript:'
//
// Returns false for: empty allowedOrigins, URL outside allowedOrigins, or blocked protocol.
//
// Usage:
//   if (!validateWebViewUrl(url, { allowedOrigins: ['https://myapp.com'] })) {
//     throw new Error('Blocked unsafe WebView URL');
//   }
//   return <WebView source={{ uri: url }} />;
```

### `@syntek/mobile-security` (Default Export) — Security Configuration

```typescript
import { SyntekMobileSecurityConfig } from '@syntek/mobile-security';

<SyntekMobileSecurityConfig
  preventScreenCapture={false}           // boolean — prevent screenshots on all screens
  screensWithCapturePrevention={[]}      // string[] — selective prevention by screen name
>
  <NavigationContainer>
    <AppNavigator />
  </NavigationContainer>
</SyntekMobileSecurityConfig>

// Key rules:
// - preventScreenCapture: true — activates expo-screen-capture globally
// - preventScreenCapture: false and screensWithCapturePrevention is empty — capture allowed
// - screensWithCapturePrevention — activation/deactivation is tied to navigation context
//   Requires screen name matching from the NavigationContainer.
//   If a screen name in the list is navigated to, capture prevention is activated.
//   If navigation away from a guarded screen, capture prevention is deactivated.
```

### `MOBILE_SECURITY_NOTES` — Documented No-Ops

The package exports a `MOBILE_SECURITY_NOTES` string constant. It is also included in a
`SECURITY-MOBILE-NOTES.md` file (shipped in the package).

This constant and documentation explain:

- **CSP (Content Security Policy)**: Not applicable to mobile apps. API requests from mobile go to
  Django, which sets CSP headers via `syntek-security` (US012). The mobile client never serves HTTP
  responses.

- **HSTS (HTTP Strict Transport Security)**: Not applicable to mobile apps. Enforced by the
  Django/Nginx layer. Expo enforces HTTPS in production builds at the platform level.

- **X-Frame-Options**: Not applicable. Mobile apps cannot be framed.

- **CSRF (Cross-Site Request Forgery)**: Not applicable in the traditional sense. Django CSRF tokens
  are exempted for mobile API clients using token-based auth (JWT). The backend validates all
  requests; see `syntek-security` for exemption details.

- **Certificate Pinning**: `expo-ssl-pinning` is the mechanism for pinning. This package documents
  the recommended approach for developers who need pinning but does not implement pinning directly.
  See `SECURITY-MOBILE-NOTES.md` for setup steps.

## Acceptance Criteria

```gherkin
# --- stripTags ---

Given a string containing HTML tags
When stripTags is called
Then all opening and closing tags are removed
And plain text content is preserved
And nested tags are handled correctly

Example:
  stripTags('<p>Hello <strong>world</strong></p>') → 'Hello world'
  stripTags('<img>test</img>') → 'test'
  stripTags('No tags') → 'No tags'
  stripTags('<div><span>Nested</span></div>') → 'Nested'

# --- escapeHtml ---

Given a string containing HTML-sensitive characters
When escapeHtml is called
Then < is escaped to &lt;
And > is escaped to &gt;
And & is escaped to &amp;
And " is escaped to &quot;
And ' is escaped to &#39;
And all five characters are escaped in the output

Example:
  escapeHtml('<img src="x" onerror="alert()">') → '&lt;img src=&quot;x&quot; onerror=&quot;alert()&quot;&gt;'
  escapeHtml('a & b') → 'a &amp; b'
  escapeHtml("it's") → 'it&#39;s'

# --- sanitiseText ---

Given a string with HTML tags and whitespace
When sanitiseText is called
Then all HTML tags are removed
And leading and trailing whitespace is trimmed
And internal whitespace is preserved

Example:
  sanitiseText('  <p>Hello</p>  ') → 'Hello'
  sanitiseText('<p>  Text with spaces  </p>') → 'Text with spaces'

# --- validateDeepLink: allowed scheme ---

Given an allowed scheme list ['myapp://', 'https://myapp.com']
When validateDeepLink('myapp://home', allowedSchemes) is called
Then the result is true

# --- validateDeepLink: disallowed scheme ---

Given an allowed scheme list ['myapp://', 'https://myapp.com']
When validateDeepLink('http://evil.com', allowedSchemes) is called
Then the result is false

# --- validateDeepLink: empty URL ---

Given any allowed scheme list
When validateDeepLink('', allowedSchemes) is called
Then the result is false

# --- validateDeepLink: null/undefined ---

Given any allowed scheme list
When validateDeepLink(null, allowedSchemes) or validateDeepLink(undefined, allowedSchemes) is called
Then the result is false

# --- validateDeepLink: malformed URL ---

Given any allowed scheme list
When validateDeepLink('not a valid url at all', allowedSchemes) is called
Then the result is false

# --- validateWebViewUrl: allowed origin ---

Given options { allowedOrigins: ['https://myapp.com'], requireHttps: true }
When validateWebViewUrl('https://myapp.com/page', options) is called
Then the result is true

# --- validateWebViewUrl: disallowed origin ---

Given options { allowedOrigins: ['https://myapp.com'], requireHttps: true }
When validateWebViewUrl('https://evil.com/phish', options) is called
Then the result is false

# --- validateWebViewUrl: HTTP with requireHttps true ---

Given options { requireHttps: true }
When validateWebViewUrl('http://myapp.com', options) is called
Then the result is false

# --- validateWebViewUrl: HTTP with requireHttps false ---

Given options { requireHttps: false, allowedOrigins: ['http://myapp.com', 'https://myapp.com'] }
When validateWebViewUrl('http://myapp.com/page', options) is called
Then the result is true

# --- validateWebViewUrl: data: URI with blockDataUrls true ---

Given options { blockDataUrls: true }
When validateWebViewUrl('data:text/html,<script>alert(1)</script>', options) is called
Then the result is false

# --- validateWebViewUrl: javascript: URI with blockJavascriptUrls true ---

Given options { blockJavascriptUrls: true }
When validateWebViewUrl('javascript:alert(1)', options) is called
Then the result is false

# --- validateWebViewUrl: empty allowedOrigins ---

Given options { allowedOrigins: [] }
When validateWebViewUrl('https://myapp.com', options) is called
Then the result is false

# --- validateWebViewUrl: subdomain not in allowedOrigins ---

Given options { allowedOrigins: ['https://myapp.com'] }
When validateWebViewUrl('https://api.myapp.com/endpoint', options) is called
Then the result is false (exact match required)

# --- SyntekMobileSecurityConfig: mount without error ---

Given a valid SyntekMobileSecurityConfig component with preventScreenCapture: false
When the component mounts
Then no error is thrown
And the NavigationContainer child renders

# --- SyntekMobileSecurityConfig: preventScreenCapture true ---

Given preventScreenCapture: true
When the component mounts
Then expo-screen-capture activate() is called exactly once

# --- SyntekMobileSecurityConfig: preventScreenCapture false ---

Given preventScreenCapture: false
When the component mounts
Then expo-screen-capture activate() is not called

# --- SyntekMobileSecurityConfig: screensWithCapturePrevention ---

Given screensWithCapturePrevention: ['SettingsScreen', 'PaymentScreen']
When navigation occurs to SettingsScreen
Then expo-screen-capture activate() is called

When navigation occurs away from SettingsScreen to HomeScreen
Then expo-screen-capture deactivate() is called

When navigation occurs to a screen not in the list (e.g., ProfileScreen)
Then expo-screen-capture deactivate() is called if previously active

# --- MOBILE_SECURITY_NOTES export ---

Given the MOBILE_SECURITY_NOTES constant
When it is imported
Then it is a non-empty string
And it documents that CSP, HSTS, X-Frame-Options, and traditional CSRF are not applicable
And it documents that certificate pinning can be implemented via expo-ssl-pinning
```
