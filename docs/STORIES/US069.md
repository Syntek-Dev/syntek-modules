# US069 — `@syntek/ui-comments`: Comments UI Package

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

## User Story

As a developer, I want a comments UI package so that I can render threaded comment threads, reaction
pickers, and moderation controls that connect to `syntek-comments` via GraphQL.

**Could Have**

## Acceptance Criteria

```gherkin
Given a CommentThread is rendered for a target model
When the component mounts
Then comments are fetched for the correct contentType and objectId and rendered as a nested tree up to the configured max depth

Given a user submits a new comment
When the addComment mutation completes
Then the new comment appears at the top of the thread without a page reload and the form is cleared

Given a user clicks a reaction on a comment
When the addReaction mutation completes
Then the reaction count increments immediately via optimistic update and rolls back if the mutation fails

Given the current user has the moderator role
When a held-for-review comment is displayed
Then an Approve button is visible and clicking it calls the approveComment mutation
```
