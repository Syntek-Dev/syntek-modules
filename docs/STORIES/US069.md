## User Story

As a developer, I want a comments UI package so that I can render threaded discussion threads,
review widgets with star ratings, reaction pickers, voting controls, and moderation tools — all
wired to `syntek-comments` via GraphQL and fully configurable.

## Acceptance Criteria

```gherkin
Given a CommentThread is rendered for a target model
When the component mounts
Then comments are fetched for the correct contentType and objectId and rendered as a nested tree
  up to the configured maxDepth; threads beyond maxDepth show a "load more replies" control

Given a user submits a new comment
When the addComment mutation completes
Then the new comment appears optimistically in the thread, the form is cleared, and an error
  rolls back the optimistic update cleanly without leaving a ghost comment

Given the backend is in reviews mode (COMMENT_MODE = reviews)
When CommentThread renders
Then each comment displays a StarRating widget; the aggregate rating and count are shown in the
  ReviewSummary header above the thread

Given a user clicks a reaction button
When addReaction mutation completes
Then the reaction count increments immediately via optimistic update and rolls back if the
  mutation fails; the current user's active reaction is visually highlighted

Given VOTING_MODE is updown
When a user clicks upvote or downvote
Then the vote is toggled; clicking the active direction again removes the vote; the net score
  updates optimistically

Given the current user has the moderator permission
When a pending comment is rendered
Then an Approve button and a Reject button are visible; clicking Approve calls approveComment;
  clicking Reject opens an inline reason input before calling rejectComment

Given a user clicks the flag (report) button on a comment
When flagComment mutation completes
Then the button changes to a reported state and cannot be clicked again for that comment
```

## Components

| Component           | Description                                                                                                               |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `CommentThread`     | Root component — renders top-level comments and nested replies to maxDepth; accepts `contentType` + `objectId` props      |
| `CommentItem`       | Single comment — body, author, timestamp, edit indicator, reactions/votes, reply button, moderation controls, flag button |
| `CommentForm`       | Inline form — body textarea with character counter, optional star rating (reviews mode), submit/cancel                    |
| `ReactionPicker`    | Reaction type selector with aggregate counts; renders reaction buttons or up/down vote controls based on VOTING_MODE      |
| `StarRating`        | 1–5 star input and display widget (reviews mode only); read-only and interactive variants                                 |
| `ReviewSummary`     | Aggregate star distribution bar chart and overall rating header                                                           |
| `ModerationBadge`   | Status badge (pending/spam/rejected) visible to moderators                                                                |
| `ModerationToolbar` | Approve/Reject controls with inline reason input for rejected comments                                                    |
| `FlagButton`        | Report-to-moderator control with confirmed reported state                                                                 |
| `LoadMoreReplies`   | Progressive disclosure for threads exceeding initial load depth                                                           |

## Hooks

| Hook                                          | Returns                                                                               |
| --------------------------------------------- | ------------------------------------------------------------------------------------- |
| `useComments(contentType, objectId, options)` | `{ comments, loading, error, fetchMore, hasNextPage }` — paginated top-level comments |
| `useCommentThread(id, depth)`                 | `{ thread, loading, error }` — single comment with nested replies                     |
| `useAddComment(options)`                      | `{ addComment, loading, error }` — mutation with optimistic update                    |
| `useEditComment()`                            | `{ editComment, loading }`                                                            |
| `useDeleteComment()`                          | `{ deleteComment, loading }`                                                          |
| `useReaction(commentId)`                      | `{ reactions, currentReaction, addReaction, removeReaction, loading }`                |
| `useVote(commentId)`                          | `{ upCount, downCount, userVote, castVote, loading }`                                 |
| `useModerationQueue(contentType, objectId)`   | Pending/held comments for moderators                                                  |
| `useCommentAnalytics(contentType, objectId)`  | Aggregate counts and average rating                                                   |
