# US013 — `syntek-audit`: Immutable Audit Logging Module

**Last Updated**: 06/03/2026
**Version**: 0.1.0
**Status**: To Do
**Epic**: Epic 3 — Core Backend Modules

---

## User Story

As a compliance officer, I want an immutable audit log so that all significant actions are recorded with actor, resource, action, timestamp, IP, and an HMAC chain that detects tampering.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given any significant action occurs (login, data change, admin action)
When the audit middleware records it
Then an `AuditEntry` is written with actor, resource_type, resource_id, action, ip, user_agent, and timestamp

Given an audit entry is written
When a batch integrity check runs against that entry
Then any entry whose HMAC does not match the chain is flagged as tampered

Given the application database user attempts to UPDATE or DELETE an audit entry
When the ORM or raw SQL executes
Then the operation is rejected by a database-level constraint

Given a GDPR retention policy is set to 24 months for a given action category
When the retention task processes entries older than 24 months
Then PII fields are anonymised and the entry record is retained for chain integrity

Given syntek-audit is installed
When any Strawberry mutation runs
Then the mutation is automatically recorded without any per-resolver boilerplate
```
