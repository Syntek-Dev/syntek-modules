# US073 — `@syntek/mobile-media`: Mobile Media Capture & Upload

**Last Updated**: 10/03/2026 **Version**: 0.2.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a mobile media package so that I can capture photos and video from the camera
or photo library, pick documents, upload with chunked progress tracking, and handle preview and crop
— all wired to `syntek-media-upload` for virus-scanned delivery to Cloudinary or MinIO.

## Acceptance Criteria

```gherkin
Given the CameraCapture component is rendered
When the user taps the shutter button
Then the captured photo is held in local state for preview and crop before upload — it is not
uploaded automatically

Given a media item is confirmed for upload
When the chunked upload begins via syntek-media-upload
Then a progress bar updates in real time as chunks are sent and the upload can be cancelled at any
point before completion

Given an upload is interrupted by a network loss
When connectivity is restored
Then the upload resumes from the last confirmed chunk — a full re-upload is never required

Given a file that does not match the allowed MIME types is selected from the library
When the selection is confirmed
Then an inline error is shown and the file is rejected before any upload session is initiated

Given the upload session is rejected with reason virus_detected
When the rejection response is received
Then a clear error message is displayed and the local preview is cleared

Given the upload completes successfully
When the commit response is received
Then the returned Cloudinary CDN URL or MinIO file record is passed to the onComplete callback
and the local preview state is cleared

Given the user selects multiple photos from the library
When useMediaUpload processes the selection
Then each file is uploaded sequentially with individual progress tracking and per-file
onComplete / onError callbacks
```

| `CameraCapture`      | `expo-camera` with permission request flow, shutter button, flash toggle      |

| `UploadProgressBar`  | Native progress bar with cancel button; wired to `useUploadProgress`          |

### Hooks

| Hook                              | Description                                                                    |
| --------------------------------- | ------------------------------------------------------------------------------ |
| `useMediaUpload(options)`         | Chunked upload via syntek-media-upload; handles initiate/chunk/complete/cancel |
| `useUploadProgress(sessionToken)` | Polls `uploadProgress`; returns `{ status, percent, cancel }`                  |
| `useCameraPermission()`           | Requests and tracks `expo-camera` permission state                             |
| `useMediaLibraryPermission()`     | Requests and tracks `expo-image-picker` permission state                       |

### `useMediaUpload` Options

```ts
interface UseMobileMediaUploadOptions {
  commitTarget: "media" | "files";
  onProgress?: (percent: number, sessionToken: string) => void;
  onComplete?: (result: MediaAsset | MobileFile) => void;
  onError?: (error: UploadError) => void;
  resumable?: boolean; // default true — resume on network restore
}
```

### Resumable Upload Strategy

When `resumable: true` (default), `useMediaUpload` stores the session token and confirmed chunk
index in `AsyncStorage`. On network restore (detected via `@react-native-community/netinfo`), it
re-queries `uploadProgress` to identify the last confirmed chunk index and resumes from the next
chunk. Sessions expired on the server trigger a clean restart with a new `initiateUpload` call.
