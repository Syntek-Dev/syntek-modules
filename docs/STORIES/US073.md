# US073 — `@syntek/mobile-media`: Mobile Media Capture & Upload

**Last Updated**: 06/03/2026

**Status**: To Do
**Epic**: Epic 8 — Mobile Packages

## User Story

As a developer, I want a mobile media package so that I can capture photos and video from the camera or photo library and upload them to `syntek-media` via GraphQL with real-time progress feedback.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the camera capture component is rendered
When the user taps the shutter button
Then the captured photo is held in local state for preview before upload — it is not uploaded automatically

Given a media item is confirmed for upload
When the upload begins
Then a progress bar updates in real time as chunks are sent and the upload can be cancelled mid-flight

Given a file that does not match the allowed MIME types is selected from the library
When the selection is confirmed
Then an inline error is shown and the file is rejected before any upload is attempted

Given the upload completes successfully
When the mutation response is received
Then the returned Cloudinary CDN URL is passed to the onSuccess callback and the local preview is cleared
```
