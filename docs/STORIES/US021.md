## User Story

As a developer, I want a bulk import module so that large data files ‚Äî CSV, Excel, JSON, and PDF ‚Äî
can be uploaded and processed asynchronously with column mapping, per-row validation, duplicate
detection, error reporting, and rollback ‚Äî all configurable without touching module internals.

**Must Have**

## Acceptance Criteria

```gherkin
Given a CSV file with 50,000 rows is uploaded for import
When the import Celery task runs
Then it processes in configurable chunks, tracks progress in Redis, and does not exhaust application
memory

Given a row in the import file fails validation
When the task processes it
Then the row is recorded in an error report with its line number and validation message, and
processing continues for remaining rows ‚Äî no early abort

Given an import job completes with errors
When the user requests the error report
Then they receive a downloadable file in the original format with error annotations per failing row

Given an import job is in progress
When the user polls the progress endpoint
Then they receive current_row, total_rows, and percentage_complete

Given a column mapping is saved for an import type
When the same file type is imported again
Then the saved column-to-field mapping is pre-populated automatically

Given a duplicate row is detected during import
When SYNTEK_BULK_IMPORT['DUPLICATE_STRATEGY'] is 'skip'
Then the duplicate is skipped and recorded in the error report with reason 'duplicate'

Given a duplicate row is detected during import
When SYNTEK_BULK_IMPORT['DUPLICATE_STRATEGY'] is 'update'
Then the existing record is updated with the imported values

Given an import job is rolled back
When the rollback task completes
Then all records created by that job are deleted and the job status is set to 'rolled_back'

Given a project configures SYNTEK_BULK_IMPORT with a custom validation schema for a model
When a row fails that custom validation
Then the per-row error message uses the consuming project's configured error message, not a default

Given a PDF file containing one or more data tables is uploaded for import
When the import task runs
Then each detected table is extracted across all pages and processed through the same validation,
duplicate detection, and error accumulation pipeline as CSV/Excel

Given a PDF import job completes with errors
When the user requests the error report
Then they receive a CSV file with error annotations ‚Äî PDF format is never used for error reports
regardless of SYNTEK_BULK_IMPORT['ERROR_REPORT_FORMAT']
```

## Configuration

All behaviour is controlled via `SYNTEK_BULK_IMPORT` in Django `settings.py`. Nothing is hardcoded.
Consuming projects configure which models are importable, custom validation, duplicate detection
strategy, and file format support.

```python
SYNTEK_BULK_IMPORT = {
    # Models available for import ‚Äî key = slug used in API; value = dotted model path
    'IMPORTABLE_MODELS': {
        'contacts': 'myapp.models.Contact',
        'products': 'myapp.models.Product',
    },

    # Chunk size for Celery task processing (rows per task iteration)
    'CHUNK_SIZE': 500,

    # Maximum file size in MB
    'MAX_FILE_SIZE_MB': 50,

    # Supported formats ‚Äî 'csv', 'excel', 'json', 'pdf'
    'SUPPORTED_FORMATS': ['csv', 'excel', 'pdf'],

    # PDF extraction backend ‚Äî 'pdfplumber' (default) or 'camelot'
    # pdfplumber: best for text-based PDFs; camelot: best for scanned/bordered tables
    'PDF_EXTRACTION_BACKEND': 'pdfplumber',

    # PDF table index to extract ‚Äî 0-indexed per page; None = extract all tables found
    'PDF_TABLE_INDEX': None,

    # Duplicate detection field(s) per model slug
    'DUPLICATE_FIELDS': {
        'contacts': ['email'],
        'products': ['sku'],
    },

    # Duplicate strategy: 'skip' | 'update' | 'error'
    'DUPLICATE_STRATEGY': 'skip',

    # Rollback support ‚Äî enables ImportJob.rollback() to delete all rows from a job
    'ENABLE_ROLLBACK': True,

    # Rollback window in days ‚Äî jobs older than this cannot be rolled back
    'ROLLBACK_WINDOW_DAYS': 30,

    # Custom per-model validation functions ‚Äî dotted path to callable(row_dict) -> list[str]
    'VALIDATION_HOOKS': {
        'contacts': 'myapp.validators.validate_contact_row',
    },

    # Error report format ‚Äî 'original' (same as input) or 'csv'
    # Note: PDF imports always produce a CSV error report regardless of this setting
    'ERROR_REPORT_FORMAT': 'original',
}
```

## Data Model

### `ImportJob`

| Field            | Type      | Notes                                                             |
| ---------------- | --------- | ----------------------------------------------------------------- |
| `id`             | UUID      | Primary key                                                       |
| `model_slug`     | CharField | Maps to `IMPORTABLE_MODELS` key                                   |
| `status`         | CharField | `pending` / `processing` / `completed` / `failed` / `rolled_back` |
| `total_rows`     | IntField  | Populated after file parse                                        |
| `processed`      | IntField  | Rows processed so far                                             |
| `created`        | IntField  | Rows successfully created                                         |
| `updated`        | IntField  | Rows updated (when strategy = update)                             |
| `skipped`        | IntField  | Rows skipped (duplicates or filtered)                             |
| `error_count`    | IntField  | Rows that failed validation                                       |
| `uploaded_by`    | FK        | Auth user                                                         |
| `uploaded_at`    | datetime  |                                                                   |
| `completed_at`   | datetime  | Null until complete                                               |
| `file_format`    | CharField | `csv` / `excel` / `json` / `pdf`                                  |
| `column_mapping` | JSONField | Column ‚Üí field mapping used for this job                          |
| `tenant`         | FK (null) |                                                                   |

### `ImportError`

| Field        | Type      | Notes                                 |
| ------------ | --------- | ------------------------------------- |
| `job`        | FK        | `ImportJob`                           |
| `row_number` | IntField  | 1-indexed source row number           |
| `row_data`   | JSONField | Raw row as parsed                     |
| `errors`     | JSONField | List of validation error strings      |
| `reason`     | CharField | `validation` / `duplicate` / `system` |

### `ColumnMapping`

Saved per (model_slug, file_hash) so repeat imports pre-populate the mapping.

| Field        | Type      | Notes                                |
| ------------ | --------- | ------------------------------------ |
| `model_slug` | CharField |                                      |
| `file_hash`  | CharField | SHA-256 of column header row         |
| `mapping`    | JSONField | `{"source_col": "model_field", ...}` |
| `tenant`     | FK (null) |                                      |
