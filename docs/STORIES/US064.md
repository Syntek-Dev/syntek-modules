## User Story

As a developer, I want a scheduling module so that I can define availability slots for resources,
accept appointment bookings, manage recurring events, and sync confirmed appointments to Google
Calendar.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a resource has availability slots defined
When `availableSlots(resourceId, dateRange)` is queried
Then only unbooked slots within the date range are returned — no already-booked or past slots

Given a booking request is made for an available slot
When `bookAppointment` is called
Then the slot is reserved atomically (no double-booking possible), an Appointment record is created with `pending` status, and a confirmation notification is dispatched

Given an appointment is confirmed
When `confirmAppointment` is called
Then the appointment status transitions to `confirmed` and the event is pushed to the linked Google Calendar via syntek-integrations

Given a cancelled appointment frees a slot
When the cancellation is processed
Then the slot becomes available again and any waitlisted bookings are notified
```
