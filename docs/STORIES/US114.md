# US114 — `@syntek/mobile-email-builder`: Mobile Email Template Editor

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a React Native email template editor so that users can compose and edit email
templates on mobile devices — using a tap-to-add block model, variable insertion, a WebView preview,
and save to `syntek-email-core` — without needing to use a desktop browser for email authoring.

## Acceptance Criteria

```gherkin
Given the MobileEmailEditor is rendered
When the user taps "Add Block"
Then a bottom sheet appears showing the available block types; tapping one inserts it at the
end of the block list

Given a TextBlock is tapped for editing
When the text editing sheet opens
Then the keyboard accessory toolbar shows the VariablePicker; tapping a variable inserts
{{ variable_name }} at the cursor

Given two blocks are in the list
When the user long-presses a block and drags
Then the block reorders with a real-time visual drag indicator (react-native-draggable-flatlist)

Given the user switches to the Preview tab
When the tab renders
Then the current blocks are rendered to HTML server-side via previewEmailTemplate and displayed
in a WebView at the correct width — not a simulated local render

Given the user taps Save
When the save mutation resolves
Then the block JSON is stored to EmailTemplate.html_body and the editor isDirty flag clears

Given the user taps Publish
When the confirmPublish dialog is confirmed
Then publishEmailTemplate is called and the template status shows "Published"
```

| --------------------- | ------------------------------------------------------------------------ |
| `MobileEmailEditor`   | Root — tab view: Edit and Preview tabs                                   |
| `BlockList`           | Draggable FlatList of block items (react-native-draggable-flatlist)      |
| `BlockItem`           | Block row — label, type icon, edit button, delete button, drag handle    |

| `EmailWebViewPreview` | WebView rendering server-rendered HTML; loading spinner; refresh button  |
| `TemplatePickerSheet` | Bottom sheet — browse and load existing templates from syntek-email-core |

### Supported Block Types on Mobile

The mobile editor supports a subset of the full block library — focusing on the most common types:

| Block          | Editable Properties                                         |
| -------------- | ----------------------------------------------------------- |
| `TextBlock`    | Plain text with variable insertion (no rich text on mobile) |
| `ImageBlock`   | Image URL, alt text, alignment                              |
| `ButtonBlock`  | Label, URL (variable-aware), background colour              |
| `DividerBlock` | Colour only                                                 |
| `SpacerBlock`  | Height                                                      |
| `FooterBlock`  | Address line, unsubscribe text                              |

`ColumnsBlock`, `HtmlBlock`, `SocialLinksBlock`, `HeaderBlock` are view-only on mobile (rendered in
preview) but not editable — full editing requires the desktop builder (US113).

### Hooks

| Hook                             | Description                                                     |
| -------------------------------- | --------------------------------------------------------------- |
| `useMobileEmailEditor(options?)` | Block state — add, remove, reorder, update, undo, save, publish |
| `useEmailPreviewWebView(blocks)` | Calls `previewEmailTemplate`; returns `{html, loading, error}`  |
| `useVariableSchema(templateId?)` | Fetches variables_schema from template or accepts prop          |

### Keyboard Accessory Toolbar

When a `TextBlock` is in edit mode, the keyboard accessory toolbar renders above the system keyboard
with:

- Variable picker button (opens `VariablePicker` inline list)
- Bold / Italic toggles (applies `**text**` markdown-style tokens rendered server-side)
- Done button (dismisses keyboard)
