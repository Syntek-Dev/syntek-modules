# US128 — `@syntek/ui-ai-chat` — React AI Chat UI

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Story Points**: 8

**Epic**: Epic 7 — Web Frontend Features

---

## User Story

**As a** developer

**I want** a React AI chat UI package

**So that** I can embed a full-page chat interface or a floating chat widget in any Next.js project,
with streaming message rendering, attachment support, and full token-driven styling — wired to
`syntek-ai` via GraphQL and SSE.

---

## MoSCoW Priority

- **Must Have:**
  - Full-page chat component (`ChatPage`)
  - Floating chat widget (`ChatWidget`)
  - Streaming message rendering (incremental token display)
  - Message list with role-based alignment (user/assistant)
  - Input bar with send button
  - Session creation and selection
  - GraphQL integration (`@syntek/api-client`)
  - SSE stream consumption

- **Should Have:**
  - Attachment preview (image thumbnails)
  - Markdown rendering in messages
  - Code block syntax highlighting
  - Typing indicator (three-dot animation)
  - Auto-scroll to latest message
  - Keyboard shortcuts (Enter to send, Shift+Enter for newline)
  - Previous conversation sidebar (optional)

- **Could Have:**
  - Voice input (audio → transcription)
  - Message copying to clipboard
  - Message regeneration
  - Conversation export (PDF, JSON)
  - Search within conversation
  - Message reactions/upvote-downvote
  - Conversation branching/forking

- **Won't Have:**
  - Mobile-responsive design (see US129 for mobile)
  - Voice output (TTS integration)
  - Multi-user collaboration (real-time co-chat)
  - File upload beyond images

## Architecture

### Core Design

The package provides two entry points: `ChatPage` (full-page chat interface) and `ChatWidget`
(floating button that expands to an overlay). Both use the same `useChat` hook internally.

The `useChat` hook manages session creation, message history from GraphQL, and SSE stream
consumption. Streaming tokens render incrementally using a text buffer. The consuming project passes
its own `systemPrompt` prop — starter prompts from the `syntek-ai` repository are intended as the
baseline.

All styling uses design tokens from `@syntek/ui`; no styles are hardcoded. Avatars, colour scheme,
title, and placeholder text are all configurable via props.

### Components

| Component            | Description                                                                                                     |
| -------------------- | --------------------------------------------------------------------------------------------------------------- |
| `ChatPage`           | Full-page chat — message list, input bar, attachment button, session selector sidebar (optional)                |
| `ChatWidget`         | Floating button (configurable position) that opens an overlay chat panel                                        |
| `ChatMessageList`    | Scrollable list of messages with role-based alignment and streaming indicator                                   |
| `ChatMessage`        | Single message bubble — user or assistant, with avatar slot, markdown rendering, code block syntax highlighting |
| `ChatInput`          | Multi-line input with send button, attachment button, and character limit indicator                             |
| `ChatAttachment`     | Attachment preview thumbnail with remove button                                                                 |
| `ChatSessionSidebar` | Optional sidebar listing previous conversation sessions                                                         |
| `StreamingIndicator` | Three-dot animation shown while assistant is typing                                                             |

### Hooks

| Hook                        | Description                                                                   |
| --------------------------- | ----------------------------------------------------------------------------- |
| `useChat(options)`          | Core hook — manages session, messages, SSE stream, send, and attachment state |
| `useChatHistory(sessionId)` | Wraps `conversationHistory` GraphQL query                                     |
| `useChatSessions()`         | Wraps `conversationSessions` paginated query                                  |

**`useChat` Options Interface**

```typescript
interface UseChatOptions {
  sessionId?: string; // Resume existing session
  systemPrompt?: string; // Override default system prompt
  model?: string; // Override default model
  onError?: (error: Error) => void;
}
```

**`useChat` Return Interface**

```typescript
interface UseChatReturn {
  messages: ChatMessage[];
  isStreaming: boolean;
  sessionId: string | null;
  sendMessage: (content: string, attachments?: File[]) => void;
  startNewSession: () => void;
  clearHistory: () => void;
  isLoading: boolean;
  error?: Error;
}

interface ChatMessage {
  id: string;
  role: "user" | "assistant" | "system";
  content: string;
  attachmentUrl?: string;
  attachmentType?: "image" | "document";
  createdAt: string;
}
```

### Component Examples

**`ChatPage` Usage**

```typescript
import { ChatPage } from '@syntek/ui-ai-chat'

export default function ChatRoute() {
  return (
    <ChatPage
      title="Ask me anything"
      placeholder="Type your question here..."
      systemPrompt="You are a helpful assistant specializing in React."
      showSessionSidebar={true}
      avatarUrl="/images/assistant-avatar.png"
    />
  )
}
```

**`ChatWidget` Usage**

```typescript
import { ChatWidget } from '@syntek/ui-ai-chat'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        {children}
        <ChatWidget
          position="bottom-right"
          label="Chat with us"
          systemPrompt="You are a helpful support agent."
        />
      </body>
    </html>
  )
}
```

### Styling Strategy

All components accept:

- `className` prop for Tailwind CSS overrides
- Design tokens from `@syntek/ui` (colors, spacing, typography)
- Dark mode support via `dark:` variants
- CSS custom properties for branding

No hardcoded hex colours or pixel values; all values derive from design tokens.

## Acceptance Criteria

### Scenario 1: Session Creation on First Send

**Given** a user opens `ChatPage` without a `sessionId` prop

**When** they type a message and click send

**Then** the system:

- Calls `startConversation` mutation (no system prompt override by default)
- Creates a new session server-side (via `syntek-ai`)
- Returns a session ID
- Sends the user message immediately
- Opens the SSE stream for the response

**And** the session ID is stored in React state for subsequent messages

---

### Scenario 2: SSE Token Rendering

**Given** the SSE stream is open

**When** tokens arrive from `text/event-stream`

**Then** each token:

- Is appended to a text buffer
- Renders incrementally in the assistant message bubble
- Updates within 100ms of arrival
- Does NOT block the UI

**And** the user sees smooth, real-time token streaming

---

### Scenario 3: Message List Chronological Order

**Given** `conversationHistory(sessionId)` returns multiple messages

**When** the component mounts or history is refreshed

**Then** messages are:

- Rendered in chronological order (oldest first, newest last)
- Aligned left (assistant) or right (user) based on role
- Grouped visually by sender

---

### Scenario 4: Attachment Preview

**Given** the user clicks the attachment button

**When** they select an image file

**Then** the system:

- Displays a thumbnail preview in the input area
- Includes a remove button (×) to cancel
- Sends the image with the message on send
- Stores the image via `syntek-media` (or configured backend)
- Displays the image inline in the message history (not just a link)

---

### Scenario 5: Widget Open/Close

**Given** `ChatWidget` is rendered

**When** the floating button is clicked

**Then** the system:

- Expands an overlay panel (from the configured position)
- Displays the chat interface inside the overlay
- Shows a close button (×) in the top-right corner of the panel

**When** the close button is clicked

**Then** the panel collapses and the floating button is shown

---

### Scenario 6: Keyboard Send (Enter Key)

**Given** the user has typed a message in `ChatInput`

**When** they press Enter

**Then** the message is sent immediately

**When** they press Shift+Enter

**Then** a newline is inserted (no send)

---

### Scenario 7: Streaming Indicator Visibility

**Given** the SSE stream is active and tokens are arriving

**When** the assistant message is being built

**Then** a `StreamingIndicator` (three-dot animation) appears at the end of the message

**When** the stream ends

**Then** the indicator disappears and the final message is displayed

## Notes

- SSE stream is consumed client-side via native `EventSource` API (no external library needed)
- Markdown rendering uses `remark` + `rehype` ecosystem; syntax highlighting via Shiki
- All styling is token-driven; Tailwind CSS variables are sourced from `@syntek/ui`
- The floating widget is positioned fixed; consuming projects should ensure their layout allows
  fixed positioning
- System prompt override is a consuming-project concern; this package accepts it as a prop but does
  not validate it
- Attachment upload is delegated to `syntek-ai` backend (consuming projects configure
  `ATTACHMENT_BACKEND`)
