## User Story

As a developer installing `syntek-auth`, I want a fully configured authentication system so that my Django project supports MFA, passkeys, social OAuth, and secure session management with zero custom auth code.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given `SYNTEK_AUTH` is configured in settings.py
When `AppConfig.ready()` runs
Then all required settings are validated and `ImproperlyConfigured` is raised for any missing required value

Given a user registers with a valid email and password
When the registration mutation is called
Then the password is hashed via Argon2id (through syntek-pyo3) and no plaintext is stored

Given a user with MFA enabled submits correct first-factor credentials
When the login mutation is called
Then a partial session token is returned and the TOTP/OTP input is required before a full session is issued

Given 5 consecutive failed login attempts from one account
When a 6th attempt is made within the lockout window
Then the account is locked and a 429 is returned with `Retry-After`

Given a valid JWT access token has expired
When the client sends the refresh token
Then a new access token is issued and the refresh token is rotated

Given an unauthenticated request hits any protected endpoint
When the request is processed
Then a 401 is returned with no user data or stack trace leaked

Given a passkey is registered for a user
When the user authenticates with that passkey
Then a full session is issued without a password prompt
```
