## User Story

As a platform administrator, I want a branding section in `syntek-platform` where I can edit
colours, fonts, typography, spacing, and breakpoints through a form with a colour picker, so that my
site's visual identity updates live without touching any code.

## Architecture Note

`TOKEN_MANIFEST` is a **static, read-only schema** published in the `@syntek/tokens` package. It
defines what tokens exist, what UI widget to render, and what the default value is. It is never
stored in or mutated by the database.

What `syntek-platform` stores in `syntek-settings` (per-tenant) is the **token override map** — a
key-value record of CSS custom property names to user-chosen values. On each SSR request the
platform reads this map and injects a `<style>` block. The published package is never modified at
runtime.

```text
TOKEN_MANIFEST (static, in @syntek/tokens)
  → platform reads it → renders branding form with correct widgets
  → user picks a colour via colour picker (oklch, hex, hsl, etc.)
  → override saved to syntek-settings: { "--color-primary": "oklch(0.55 0.2 250)" }
  → SSR: reads overrides → injects <style>:root{ --color-primary: oklch(0.55 0.2 250) }</style>
  → all var(--color-primary) references update — no package change, no rebuild
```

## Supported Colour Formats

The platform branding form and colour picker must accept and store any valid CSS colour string, plus
Tailwind CSS palette references which are resolved to hex before storage:

| Format         | Example                                  | Notes                                   |
| -------------- | ---------------------------------------- | --------------------------------------- |
| Hex            | `#2563eb`, `#fff`, `#2563ebcc`           | 3, 6, and 8-digit (with alpha)          |
| rgb()          | `rgb(37, 99, 235)`                       |                                         |
| rgba()         | `rgba(37, 99, 235, 0.8)`                 |                                         |
| hsl()          | `hsl(221, 83%, 53%)`                     |                                         |
| hsla()         | `hsla(221, 83%, 53%, 0.8)`               |                                         |
| hwb()          | `hwb(221 15% 8%)`                        | CSS Color Level 4                       |
| lab()          | `lab(46 -8 -45)`                         | CSS Color Level 4                       |
| lch()          | `lch(46 46 264)`                         | CSS Color Level 4                       |
| oklab()        | `oklab(0.55 -0.05 -0.15)`                | CSS Color Level 4, perceptual           |
| oklch()        | `oklch(0.55 0.2 250)`                    | CSS Color Level 4, perceptual           |
| Named          | `cornflowerblue`, `transparent`, `white` | All CSS named colours                   |
| Tailwind scale | `blue-600`, `emerald-400`, `slate-900`   | Resolved to hex on save; see note below |

`oklch` is the preferred format for new tokens — it provides perceptually uniform colour scaling and
is well-supported in modern browsers (Chrome 111+, Firefox 113+, Safari 16.4+).

### Tailwind colour names

The colour picker exposes the full Tailwind CSS v4 palette (`slate`, `gray`, `zinc`, `neutral`,
`stone`, `red`, `orange`, `amber`, `yellow`, `lime`, `green`, `emerald`, `teal`, `cyan`, `sky`,
`blue`, `indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose`) at scales 50–950 as a named swatch
palette tab. When a user selects `blue-600` from the palette:

1. The picker resolves it to its hex value (`#2563eb`) using the bundled Tailwind v4 colour map
   exported from `@syntek/tokens` as `TAILWIND_COLOURS`
2. The **resolved hex** is what gets stored in `syntek-settings` — not the Tailwind name string
3. This keeps the stored value a valid CSS colour regardless of Tailwind version changes

The `TAILWIND_COLOURS` export is a flat record: `{ "blue-600": "#2563eb", ... }` covering all
palette entries. The colour picker uses it to render swatches and resolve selections.

  ```ts
  type TokenCategory =
    | "colour"
    | "spacing"
    | "typography"
    | "radius"
    | "shadow"
    | "z-index"
    | "transition";

  type TokenWidgetType =
    | "color" // colour picker — accepts all CSS colour formats
    | "px" // number input with px unit
    | "rem" // number input with rem unit
    | "font-family" // font selector
    | "font-weight" // select: 100–900
    | "number" // plain number (e.g. z-index)
    | "duration" // number input with ms unit
    | "easing"; // text input / cubic-bezier editor

  interface TokenDescriptor {
    key: string; // e.g. "COLOR_PRIMARY"
    cssVar: string; // e.g. "--color-primary"
    category: TokenCategory;
    type: TokenWidgetType;
    default: string | number; // resolved value — hex for colours, number for spacing/breakpoints
    label: string; // human-readable label for the branding form
  }
  ```
