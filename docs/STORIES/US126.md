# US126 — `@syntek/mobile-error`: Mobile Error Screens Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a React Native error screens package so that I can display branded,
accessible error screens for every error type (404, 403, 401, 500, 429, 503, network offline), catch
React render errors via an ErrorBoundary, and configure messaging, retry logic, and redirect
behaviour per error type.

## Acceptance Criteria

```gherkin
Given a 404 error is returned from the API
When MobileNotFoundScreen renders
Then the configured title, description, and "Go Back" button are shown with the correct
  design-token styling; tapping Go Back calls navigation.goBack()

Given a React render error is thrown inside MobileErrorBoundary
When the boundary catches it
Then the MobileErrorFallback screen renders; the onError callback is called with the error
  and correlationId; the app does not crash or show a blank screen

Given the device has no network connection
When MobileOfflineScreen renders
Then a "No internet connection" message is shown with a Retry button; tapping Retry calls the
  registered onRetry callback and shows an ActivityIndicator while retrying

Given a 401 response is received
When MobileUnauthorizedScreen renders
Then a "Session expired" or "Sign in" message is shown with a "Sign In" button that navigates
  to the configured loginRoute

Given a 503 maintenance response is received
When MobileMaintenanceScreen renders
Then the maintenance message and estimated recovery time are shown; no navigation away is
  possible; a pull-to-refresh triggers a re-check of the maintenance endpoint
```

## Configuration

```typescript
// Pass to MobileErrorProvider at app root
interface MobileErrorConfig {
  notFound?: {
    title?: string;
    description?: string;
    showGoBack?: boolean; // default: true
    ctaLabel?: string;
    ctaRoute?: string;
  };
  unauthorized?: {
    title?: string;
    description?: string;
    loginRoute: string; // required — navigation route name for login screen
  };
  forbidden?: {
    title?: string;
    description?: string;
    showGoBack?: boolean; // default: true
  };
  serverError?: {
    title?: string;
    description?: string;
    showCorrelationId?: boolean; // default: true
    showRetry?: boolean; // default: true
  };
  rateLimited?: {
    title?: string;
    description?: string;
    showRetry?: boolean; // default: true
  };
  maintenance?: {
    title?: string;
    message?: string;
    estimatedRecoveryTime?: string;
    logoSource?: ImageSourcePropType;
  };
  offline?: {
    title?: string;
    description?: string;
    onRetry?: () => void | Promise<void>;
  };
  onError?: (error: Error, correlationId: string) => void;
}
```

## Components

| Component                  | Description                                                                      |
| -------------------------- | -------------------------------------------------------------------------------- |
| `MobileErrorProvider`      | Context provider — wraps navigator root; provides error config                   |
| `MobileErrorScreen`        | Generic error screen — accepts `errorCode`, `title`, `description`, optional CTA |
| `MobileNotFoundScreen`     | 404 screen with Go Back / CTA button                                             |
| `MobileUnauthorizedScreen` | 401 screen — session expired; Sign In button navigates to loginRoute             |
| `MobileForbiddenScreen`    | 403 screen — access denied; Go Back button                                       |
| `MobileServerErrorScreen`  | 500 screen — shows correlationId; Retry button                                   |
| `MobileRateLimitedScreen`  | 429 screen — shows wait time; Retry button with countdown                        |
| `MobileMaintenanceScreen`  | 503 screen — message, pull-to-refresh to re-check                                |
| `MobileOfflineScreen`      | No-network screen — Retry button with @react-native-community/netinfo listener   |
| `MobileErrorBoundary`      | React class error boundary — catches render errors; renders MobileErrorFallback  |
| `MobileErrorFallback`      | Minimal safe fallback — renders without context dependency                       |
| `MobileErrorIllustration`  | Simple SVG illustration per error type (bundled, no network required)            |

## Hooks

| Hook                       | Returns                                                |
| -------------------------- | ------------------------------------------------------ |
| `useMobileErrorConfig()`   | Current error configuration                            |
| `useMobileErrorBoundary()` | `{ resetBoundary }`                                    |
| `useMobileNetworkStatus()` | `{ isConnected, isInternetReachable }` — wraps netinfo |
