# US111 — `@syntek/mobile-scheduling`: Mobile Booking UI Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a mobile booking UI package so that I can embed availability browsing,
appointment booking, and calendar export into React Native applications — with the same configurable
booking modes and label overrides as the web package.

## Architecture

`@syntek/mobile-scheduling` provides React Native components and hooks that mirror the web package's
functionality. Key exports include `useScheduling()` for reading configuration and managing booking
state, and UI components (`AvailabilityPicker`, `BookingForm`, `BookingConfirmation`,
`AppointmentList`, `CancellationSheet`) that adapt to both iOS and Android. The "Add to Calendar"
action opens the device's native calendar application via a deep-link to the iCal file served by
`syntek-scheduling-calendar`.

## Acceptance Criteria

```gherkin
Scenario: Hard booking mode — time slot picker
Given bookingMode = 'hard'
When AvailabilityPicker renders
Then exact time slots are shown in a scrollable grid in the user's device timezone

Scenario: Soft booking mode — arrival window picker
Given bookingMode = 'soft'
When AvailabilityPicker renders
Then arrival windows are shown as selectable ranges

Scenario: Native calendar integration
Given a booking is confirmed
When BookingConfirmation renders
Then an "Add to Calendar" button is shown that, when tapped, opens the device's native calendar app with the appointment pre-filled via the iCal URL

Scenario: Cancellation with policy display
Given a user taps cancel on AppointmentList
When CancellationSheet opens
Then the cancellation policy is shown (free window or fee notice) and confirming calls the cancelAppointment mutation

Scenario: Label customisation on mobile
Given custom labels are provided
When any component renders
Then all text uses the provided overrides
```
