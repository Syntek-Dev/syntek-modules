## User Story

As a developer, I want a notification UI package so that I can display a real-time notification bell
and feed, surface toast alerts, and provide a full notification preference centre where users can
opt in or out of each notification category per channel — all connected to `syntek-notifications`
via WebSocket and GraphQL.

## Acceptance Criteria

```gherkin
Given a new notification is delivered via WebSocket
When the notification event is received
Then the bell icon updates its unread count immediately without a page refresh

Given the notification feed is opened
When the panel renders
Then all unread notifications are listed with timestamp, icon, and action link

Given the user marks all notifications as read
When the bulk-read mutation completes
Then all notifications in the feed show as read and the bell count resets to zero

Given a notification is dispatched to a user
When the notification feed is closed
Then a toast appears in the corner of the screen with the notification message and action link

Given the user opens the notification preference centre
When the panel renders
Then each notification category is listed with its display name, current opt-in state per channel,
and a toggle — categories with legal_basis = contract or legitimate_interest are shown as
read-only (always on) with a tooltip explaining why they cannot be disabled

Given a user toggles off a notification category that requires explicit consent
When the withdraw mutation completes
Then the consent record is closed immediately and no further notifications are sent for that category

Given a user toggles on a notification category that requires explicit consent
When the grant mutation completes
Then a NotificationConsentRecord is created and notifications for that category resume
```

## Components

### `NotificationBell`

Compact header icon with unread badge. Clicking opens `NotificationFeed`.

### `NotificationFeed`

Slide-out panel listing notifications. Supports single mark-read, bulk mark-read, and infinite
scroll for older items. Each item shows icon, message, timestamp, and optional action link.

### `NotificationToast`

Auto-dismissing toast alert triggered by incoming WebSocket notifications when the feed is not open.
Configurable position (top-right / bottom-right / bottom-left) and auto-dismiss delay.

### `NotificationPreferenceCentre`

Full preference management UI. Presents all configured `NotificationConsentCategory` rows grouped by
section.

**Layout:**

```text
Category               Email    SMS    Push   In-App
─────────────────────────────────────────────────────
Security Alerts        🔒 On    🔒 On  🔒 On  🔒 On    (contract — read-only)
Transactional          🔒 On    🔒 On   Off   🔒 On    (contract — read-only per channel)
Marketing Emails        ●        —      —      —       (consent — user-toggleable)
Marketing SMS           —        ●      —      —       (consent — user-toggleable)
Push Notifications      —        —      ●      —       (consent — user-toggleable)
Product Updates         ●        —      ●      ●       (configurable — user-toggleable)
```

Rules:

- Categories with `legal_basis = contract` or `legitimate_interest` render as locked toggles with an
  info tooltip explaining the legal basis; they cannot be turned off
- Categories with `legal_basis = consent` render as interactive toggles; toggling calls
  `grantNotificationConsent` or `withdrawNotificationConsent`
- Channel columns that are not supported for a category show `—`
- The preference centre is embeddable in an account settings page or surfaced as a standalone route

### `ConsentStatusBadge`

Compact inline badge showing consent status for a single category. Used in account settings
summaries and onboarding flows.

## Hooks

| Hook                           | Returns                                              |
| ------------------------------ | ---------------------------------------------------- |
| `useNotifications()`           | Paginated notification list + WebSocket subscription |
| `useUnreadCount()`             | Live unread count from WebSocket                     |
| `useMarkAsRead()`              | `[markAsRead(id), { loading }]` mutation hook        |
| `useMarkAllRead()`             | `[markAllRead(), { loading }]` mutation hook         |
| `useNotificationPreferences()` | Per-category per-channel preference state            |
| `useGrantConsent()`            | `[grantConsent(category, method), { loading }]`      |
| `useWithdrawConsent()`         | `[withdrawConsent(category), { loading }]`           |

## Integration Points

### Registration (`@syntek/ui-auth`)

`SyntekAuthProvider` accepts a `notificationConsent` prop — an array of category slugs to collect
consent for at registration. `RegisterForm` renders a `NotificationConsentCheckboxes` section
(similar to `LegalCheckboxes`) for these categories. Consent is recorded with
`method = 'registration'`.

### Settings (`@syntek/ui-settings`)

`@syntek/ui-settings` includes a `NotificationPreferenceCentre` tab when `@syntek/ui-notifications`
is installed. No additional configuration required.
