## User Story

As a developer, I want a notification feed UI package so that I can display a real-time notification
bell and feed that connects to `syntek-notifications` via WebSocket and GraphQL.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a new notification is delivered via WebSocket
When the notification event is received
Then the bell icon updates its unread count immediately without a page refresh

Given the notification feed is opened
When the panel renders
Then all unread notifications are listed with timestamp, icon, and action link

Given the user marks a notification as read
When the read mutation completes
Then the notification is removed from the unread list and the bell count decrements

Given the user marks all notifications as read
When the bulk-read mutation completes
Then all notifications in the feed show as read and the bell count resets to zero
```
