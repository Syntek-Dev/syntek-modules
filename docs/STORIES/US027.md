## User Story

As a developer building for a charity or church, I want a provider-agnostic donations module so that
I can accept one-off and recurring donations via any configured payment provider, capture UK Gift
Aid declarations, generate and email donation receipts, and configure all behaviour via
`SYNTEK_DONATIONS` settings.

## Acceptance Criteria

```gherkin
Given a donor makes a Gift Aid eligible donation and confirms taxpayer status
When the declaration is saved
Then a `GiftAidDeclaration` record is created linked to the donation with all required HMRC fields populated

Given Gift Aid declarations are requested for export
When the export runs
Then the output conforms to HMRC Gift Aid XML schema with all mandatory fields

Given a recurring donation is configured for monthly billing
When the billing date arrives
Then the charge is processed automatically via syntek-payments-core using the configured provider without manual intervention

Given a donor chooses to donate anonymously
When the donation is processed
Then no PII is stored and the donation amount is recorded without any donor reference

Given a fundraising campaign has a target amount
When donations are recorded against it
Then the total raised and percentage of target are accurate in real time

Given a donation is completed
When the receipt generation task runs
Then a donation receipt PDF is generated and emailed to the donor via syntek-notifications using the configured email channel

Given `SYNTEK_DONATIONS['GIFT_AID_ENABLED']` is False
When the donation form is submitted
Then no Gift Aid declaration fields are present or required

Given a recurring donation webhook event arrives from the configured provider
When the event is processed via syntek-payments-core webhook dispatcher
Then the `RecurringDonation` record is updated and the donor is notified via syntek-notifications
```
