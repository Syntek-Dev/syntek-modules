## User Story

As a developer, I want a central email engine so that I can manage email templates with versioning,
send transactional and bulk emails via any supported provider (SendGrid, Mailgun, SMTP), track
delivery events via webhooks, handle bounces and complaints automatically, and maintain a
suppression list — without rebuilding this infrastructure in every consuming module.

## Acceptance Criteria

```gherkin
Given an EmailTemplate is created with a Jinja2 subject and html_body
When previewEmailTemplate is called with sample variables
Then the rendered HTML and plain-text fallback are returned with all variables substituted

Given sendEmail is called with a valid template ID and recipient
When the recipient's email is not in the suppression list
Then the email is delivered via the configured provider and an EmailMessage record is created with
status "sent" and the provider message ID

Given a recipient email is in the suppression list with reason "hard_bounce"
When sendEmail is called with that address
Then the send is skipped, the EmailMessage is created with status "suppressed", and no provider
API call is made

Given the provider webhook fires a "hard_bounce" event
When the webhook endpoint verifies the provider signature and processes the payload
Then an EmailEvent record is created, the provider message ID is resolved to an EmailMessage,
and the recipient address is added to EmailSuppression with reason "hard_bounce"

Given SOFT_BOUNCE_THRESHOLD is 3 and a recipient has received 3 consecutive soft bounce events
When the third soft bounce webhook is processed
Then the recipient is added to EmailSuppression with reason "soft_bounce_threshold"

Given a recipient clicks the unsubscribe link
When the signed unsubscribe URL is verified and processed
Then an EmailSuppression record is created with reason "unsubscribed" and all future sends to
that address are suppressed

Given BACKEND is switched from "sendgrid" to "mailgun" in settings
When Django restarts
Then all subsequent sends route through the Mailgun adapter with no code changes in calling modules
```

### Provider Adapters

All adapters satisfy the `EmailBackend` protocol:

```python
class EmailBackend(Protocol):
    def send(self, message: OutboundEmail) -> str: ...            # Returns provider message ID
    def send_bulk(self, messages: list[OutboundEmail]) -> list[str]: ...
    def verify_webhook_signature(self, request: HttpRequest) -> bool: ...
    def parse_webhook_event(self, payload: dict) -> ParsedEmailEvent: ...
```

| Adapter           | BACKEND key | Webhook events supported                                      |
| ----------------- | ----------- | ------------------------------------------------------------- |
| `SendGridBackend` | `sendgrid`  | delivered, opened, clicked, bounced, complained, unsubscribed |
| `MailgunBackend`  | `mailgun`   | delivered, opened, clicked, bounced, complained, unsubscribed |
| `SMTPBackend`     | `smtp`      | sent only — no webhook events (no tracking)                   |

## Public API

```python
from syntek_email_core import send_email, send_bulk_email, is_suppressed

# Send a single templated email (async via Celery)
send_email(
    template_id='welcome',
    to_email='user@example.com',
    variables={'first_name': 'Alice', 'activation_url': 'https://...'},
    from_email='noreply@tenant.com',  # Optional override
)

# Send to a list (batched Celery tasks)
send_bulk_email(
    template_id='newsletter',
    recipients=[{'email': 'a@b.com', 'variables': {...}}, ...],
)

# Check suppression before attempting send
if not is_suppressed('user@example.com'):
    send_email(...)
```

## GraphQL API

### Mutations

| Mutation                                         | Description                                         |
| ------------------------------------------------ | --------------------------------------------------- |
| `createEmailTemplate(input)`                     | Create; `is_published = False` by default           |
| `publishEmailTemplate(id)`                       | Publish — creates `EmailTemplateVersion` snapshot   |
| `updateEmailTemplate(id, input)`                 | Update draft; does NOT create a version             |
| `deleteEmailTemplate(id)`                        | Soft-delete                                         |
| `sendEmail(templateId, to, variables, options)`  | Send one email (async)                              |
| `sendBulkEmail(templateId, recipients, options)` | Enqueue bulk send (Celery batched)                  |
| `emailWebhook(payload)`                          | Internal — receives provider event, updates records |
| `reinstateEmailAddress(email)`                   | Admin — remove from suppression list                |

### Queries

| Query                                    | Returns                                                           |
| ---------------------------------------- | ----------------------------------------------------------------- |
| `emailTemplates(page, locale)`           | Paginated `[EmailTemplate]` — tenant-scoped                       |
| `emailTemplate(id)`                      | Single template with version history                              |
| `previewEmailTemplate(id, variables)`    | `{html, text}` — rendered preview; does not send                  |
| `emailMessages(page, status, dateRange)` | Paginated message log — tenant-scoped                             |
| `emailStats(dateRange)`                  | Delivery rate, open rate, click rate, bounce rate, complaint rate |
| `emailSuppressions(page, reason)`        | Paginated suppression list                                        |
| `checkSuppression(email)`                | `{suppressed: bool, reason: str}` — instant lookup                |
