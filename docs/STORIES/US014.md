# US014 — `syntek-logging`: Structured Logging Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

---

## User Story

As a developer, I want structured JSON logging with request ID propagation so that logs from any
layer of the stack can be correlated and ingested by Loki and Sentry.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given an HTTP request arrives
When the logging middleware runs
Then a unique UUID request ID is generated and threaded through all log statements for that request

Given `LOG_LEVEL: debug` is configured for development
When any log statement runs
Then output is human-readable coloured console format

Given `LOG_LEVEL: error` is configured for production
When a log statement at INFO level runs
Then it is suppressed entirely

Given a field matching a configured sensitive pattern (e.g. `password`, `token`, `card_number`) is logged
When the log emitter runs
Then the field value is replaced with `[REDACTED]`

Given a Celery task is dispatched
When the task runs on a worker
Then the originating request ID is present in all task log output
```
