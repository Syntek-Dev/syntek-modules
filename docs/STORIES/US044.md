## User Story

As a developer, I want a configurable typed GraphQL client so that I can make type-safe API calls,
handle authentication headers automatically, retry on transient errors, subscribe via WebSocket, and
intercept requests and responses — all without writing boilerplate code in every consuming package.

**Must Have**

## Acceptance Criteria

```gherkin
Given the GraphQL schema is introspected
When graphql-codegen runs
Then fully typed TypeScript hooks and document nodes are generated for every query, mutation, and
  subscription in shared/graphql/ and emitted to packages/web/api-client/src/generated/

Given a generated hook is called in a component
When TypeScript compiles
Then incorrect field names or argument types are caught at compile time — no `any` types are
  present in the generated output

Given the user's session token is present in the session context
When the client makes any request
Then the Authorization: Bearer <token> header is injected automatically by the auth interceptor
  without any manual configuration at the call site

Given the session token has expired and a 401 response is received
When the auth interceptor detects the 401
Then it automatically calls the token refresh endpoint, updates the session, and retries the
  original request exactly once; if refresh also fails the user is signed out gracefully

Given a network error or 5xx response occurs
When the retry interceptor fires
Then the request is retried up to MAX_RETRIES times with exponential backoff using RETRY_DELAY_MS
  as the base; requests with mutations are not retried unless RETRY_MUTATIONS is True

Given a WebSocket connection is established for a subscription
When the connection drops
Then the client reconnects automatically with RECONNECT_DELAY_MS back-off and re-subscribes to
  all active subscriptions without any manual intervention

Given REQUEST_TIMEOUT_MS is configured
When a request exceeds the timeout
Then it is cancelled and a TimeoutError is returned to the caller

Given an error interceptor is registered via addErrorInterceptor(fn)
When any request returns a GraphQL or network error
Then the interceptor function is called with the full error context before the error propagates
  to the component — enabling global error handling (Sentry reporting, toast notifications, etc.)
```

## Configuration

```typescript
// createSyntekClient(options) — call once at app root
interface SyntekClientOptions {
  // Endpoint
  httpUrl: string; // GraphQL HTTP endpoint
  wsUrl?: string; // WebSocket endpoint for subscriptions

  // Authentication
  getToken: () => string | null; // called before every request
  onTokenRefresh: () => Promise<string | null>; // called on 401 — return new token or null
  onAuthFailure: () => void; // called when refresh fails — typically sign out

  // Retry
  maxRetries?: number; // default: 3 (network errors and 5xx only)
  retryDelayMs?: number; // base delay for exponential back-off; default: 500
  retryMutations?: boolean; // default: false

  // Timeouts
  requestTimeoutMs?: number; // default: 30000

  // WebSocket / Subscriptions
  reconnectDelayMs?: number; // default: 1000
  connectionParams?: Record<string, unknown>; // e.g. auth headers for WS handshake

  // Caching (urql/Apollo)
  cacheConfig?: NormalizedCacheConfig; // pass-through to underlying client cache

  // Interceptors
  requestInterceptors?: RequestInterceptor[];
  responseInterceptors?: ResponseInterceptor[];
  errorInterceptors?: ErrorInterceptor[];

  // Development
  devTools?: boolean; // enable Apollo/urql DevTools; default: process.env.NODE_ENV === 'development'
}
```

## API Surface

### Client Factory

```typescript
// Create and configure the client instance
const client = createSyntekClient(options: SyntekClientOptions): SyntekClient

// Wrap your app root — provides client to all hooks
<SyntekClientProvider client={client}>
  {children}
</SyntekClientProvider>
```

### Interceptors

```typescript
// Add a global request interceptor (e.g. add correlation ID header)
client.addRequestInterceptor((request) => modifiedRequest)

// Add a global response interceptor (e.g. extract custom headers)
client.addResponseInterceptor((response) => response)

// Add a global error interceptor (e.g. report to Sentry)
client.addErrorInterceptor((error, operation) => void)
```

### Code Generation

```bash
# Run via syntek-dev — regenerates from shared/graphql/ schema
syntek-dev codegen
```

Generated output: `packages/web/api-client/src/generated/`

- `types.ts` — all GraphQL scalar, enum, and object types
- `operations.ts` — typed document nodes for all operations
- `hooks.ts` — typed React hooks for all queries/mutations/subscriptions

### Hooks (generated)

All generated hooks follow the same pattern as the underlying urql/Apollo hooks but are fully typed.
Custom shared hooks:

```typescript
// Low-level execute — use when generated hooks are insufficient
useQuery<T>(document, variables?, options?): QueryResult<T>
useMutation<T, V>(document): MutationResult<T, V>
useSubscription<T>(document, variables?): SubscriptionResult<T>

// Access the raw client instance (for imperative calls)
useSyntekClient(): SyntekClient
```

## Underlying Client Choice

The package wraps **urql** by default (lighter, simpler exchange model suited to a modular library).
Consuming projects that require Apollo-specific features (Apollo Studio, fine-grained cache control)
can switch via `cacheConfig`. The generated hooks and types are identical regardless of the
underlying client — the switch is a build-time option, not a runtime flag.
