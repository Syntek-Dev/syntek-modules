# US044 — `@syntek/api-client`: Generated GraphQL Client

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 6 — Web Frontend
Packages

---

## User Story

As a developer, I want a generated typed GraphQL client so that I can make type-safe API calls from
any Syntek web application without writing boilerplate fetch code.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given the GraphQL schema is introspected
When graphql-codegen runs
Then fully typed TypeScript hooks and operations are generated for every query and mutation in `shared/graphql/`

Given a generated hook is called in a component
When TypeScript compiles
Then incorrect field names or argument types are caught at compile time — not at runtime

Given the user's session token is present
When the client makes a request
Then the Authorization header is set automatically without any manual configuration at the call site

Given a network error occurs
When the client receives the error
Then it is surfaced as a typed error object matching the GraphQL error structure
```
