# US094 — `@syntek/mobile-forms`: React Native Forms Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a React Native forms package so that I can build any form in a mobile app —
of any length, any field type, any conditional logic — with touch-optimised native components,
keyboard-aware scrolling, and the same `useForm` API pattern as `@syntek/forms`. Forms must support
pre-fill, multi-step, dynamic schema rendering from `syntek-forms-core`, and headless use.

## Acceptance Criteria

```gherkin
Given a form renders in a React Native app
When the user taps a text field near the bottom of the screen
Then the KeyboardAvoidingView ensures the field is scrolled above the software keyboard

Given a SelectField is tapped on iOS
When the picker opens
Then a native ActionSheet (iOS) or Modal with a flat list (Android) is shown — not an HTML select

Given a DateField is tapped
When the picker opens
Then the platform-native DateTimePicker is displayed — no third-party date picker library required

Given a FileUploadField is tapped
When the picker opens
Then the native image picker (Expo ImagePicker) and document picker (Expo DocumentPicker) options
are presented

Given a conditional field has its display condition unmet
When the controlling field changes value
Then the dependent field is hidden immediately and excluded from the submission payload — identical
behaviour to the React web version

Given a multi-step form renders
When the user swipes left on the current step
Then the form advances to the next step if step validation passes

Given useForm is used without any of the provided native components
When the hook is called
Then it returns value, error, onChange, handleSubmit, and reset — fully usable headlessly

Given a DynamicForm renders a schema fetched from syntek-forms-core
When the schema includes all supported field types
Then the correct native components are rendered for each type with all conditional logic applied
```

## Deployment Contexts

| Context         | How to use                                                            |
| --------------- | --------------------------------------------------------------------- |
| ScrollView form | Wrap `FormProvider` in `KeyboardAwareScrollView` for any-length forms |
| Modal/sheet     | Use inside a `BottomSheet` or `Modal` — no assumption about container |
| Multi-step      | Use `MultiStepForm` with swipe gesture navigation                     |
| Pre-filled      | Pass `defaultValues` (static) or `initialValues` (async Promise)      |
| Dynamic schema  | Use `DynamicForm` with backend schema — mirrors web `DynamicForm` API |
| Headless        | Use `useForm` hook alone without any provided native components       |

## Hooks

The hook API mirrors `@syntek/forms` exactly — consuming projects share hook logic between web and
native.

| Hook                  | Returns                                                            |
| --------------------- | ------------------------------------------------------------------ |
| `useForm(options)`    | `{ register, handleSubmit, reset, setValue, watch, formState }`    |
| `useFormField(name)`  | `{ value, error, onChange, onBlur }`                               |
| `useFormWatch(name?)` | Observed field value — re-renders on change                        |
| `useFormStep()`       | `{ currentStep, totalSteps, goNext, goPrev, canGoNext, progress }` |

## Components

### `FormProvider`

Same API as `@syntek/forms`. Wraps all native form components. Accepts `form` from `useForm`.

### Native Field Components

All components accept `name`, `label`, `description`, `required`, `disabled`. Errors display inline
below the field.

| Component            | Native Implementation                           | Key Props                             |
| -------------------- | ----------------------------------------------- | ------------------------------------- |
| `TextField`          | `TextInput` (single line)                       | `autoComplete`, `maxLength`           |
| `TextareaField`      | `TextInput` (multiline)                         | `numberOfLines`, `maxLength`          |
| `NumberField`        | `TextInput` with `keyboardType="numeric"`       | `min`, `max`                          |
| `EmailField`         | `TextInput` with `keyboardType="email-address"` |                                       |
| `PhoneField`         | `TextInput` with `keyboardType="phone-pad"`     | `defaultCountry`                      |
| `SelectField`        | ActionSheet (iOS) / Modal list (Android)        | `options`, `placeholder`              |
| `MultiSelectField`   | Modal with checkbox list                        | `options`, `maxSelections`            |
| `CheckboxField`      | Pressable with custom checkbox                  |                                       |
| `CheckboxGroupField` | List of `CheckboxField` items                   | `options`                             |
| `RadioGroupField`    | List of Pressable radio items                   | `options`, `layout`                   |
| `ToggleField`        | React Native `Switch`                           |                                       |
| `DateField`          | `DateTimePicker` (mode="date")                  | `minDate`, `maxDate`                  |
| `TimeField`          | `DateTimePicker` (mode="time")                  | `minuteStep`                          |
| `DateTimeField`      | `DateTimePicker` (mode="datetime")              |                                       |
| `DateRangeField`     | Two `DateField` components (from/to)            |                                       |
| `FileUploadField`    | Expo `ImagePicker` + `DocumentPicker`           | `accept` ('images' / 'files' / 'any') |
| `RatingField`        | Row of Pressable star icons                     | `max` (default 5)                     |
| `HiddenField`        | Not rendered                                    | Value submitted silently              |

### Layout Components

```tsx
// Keyboard-aware container — required for any-length forms
<KeyboardAwareFormContainer>
  <FormProvider form={form}>
    <TextField name="name" label="Full Name" />
    <EmailField name="email" label="Email" />
    <FormSubmitButton label="Submit" />
  </FormProvider>
</KeyboardAwareFormContainer>
```

```tsx
// Multi-step with swipe gesture
<MultiStepForm form={form} swipeEnabled showProgress>
  <FormStep title="Your Details" validate={["name", "email"]}>
    <TextField name="name" label="Full Name" />
  </FormStep>
  <FormStep title="Your Message">
    <TextareaField name="message" label="Message" />
  </FormStep>
</MultiStepForm>
```

`FormSubmitButton` — handles loading state and disabled state automatically; accepts `label` prop.

### `DynamicForm`

Same API as the React web `DynamicForm`. Fetches `syntek-forms-core` schema via GraphQL and renders
native field components. All conditional logic and pre-fill sourced from backend schema.

```tsx
<DynamicForm formId="contact-form" onSuccess={() => navigation.navigate("ThankYou")} />
```

## Conditional Logic

Mirrors `@syntek/forms` exactly. `showWhen` prop on field components; backend schema conditions
evaluated via `useFormWatch`. Hidden fields excluded from submission payload.
