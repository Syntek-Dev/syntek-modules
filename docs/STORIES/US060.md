# US060 — `@syntek/mobile-payments`: Multi-Provider Mobile Payments

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a multi-provider mobile payments package so that I can integrate Apple Pay,
Google Pay, PayPal, Square, and SumUp into React Native applications through the
syntek-payments-core interface.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the provider is Stripe and the device supports Apple Pay
When the payment button renders
Then the native Apple Pay button is shown

Given the provider is Stripe and the device supports Google Pay
When the payment button renders
Then the native Google Pay button is shown

Given the provider is PayPal
When the payment button renders
Then the PayPal mobile checkout flow is initiated

Given the provider is Square
When the payment form renders
Then the Square In-App Payments card entry is shown

Given a payment succeeds via any provider
When the provider SDK calls back
Then the confirmPayment backend mutation is called and success state is displayed

Given a payment fails
When the error is returned
Then the error is displayed and the payment sheet dismisses cleanly
```
