## User Story

As a developer, I want a mobile payments package so that I can integrate Apple Pay, Google Pay, and Stripe mobile SDK into React Native applications backed by `syntek-payments`.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the device supports Apple Pay
When the Apple Pay button is rendered
Then it is displayed using the native Apple Pay button component and not a custom styled button

Given a user authorises an Apple Pay payment
When the payment sheet is confirmed
Then the Stripe payment method is created and the backend `confirmPayment` mutation is called

Given the device supports Google Pay
When the Google Pay button is rendered and tapped
Then the Google Pay sheet opens and on confirmation the payment is processed via the backend

Given a payment fails
When the payment error is returned
Then the error is displayed to the user and the payment sheet is dismissed cleanly
```
