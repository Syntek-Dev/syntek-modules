# US010 — `syntek-tenancy`: Multi-Tenancy Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

---

## User Story

As a developer building a multi-tenant application, I want schema-per-tenant isolation so that no
tenant's data is ever accessible from another tenant's request.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given `SYNTEK_TENANCY` is configured with `ISOLATION: schema`
When a request arrives with Host header `acme.example.com`
Then the active PostgreSQL schema is set to `tenant_acme` for the duration of that request only

Given a cross-tenant query is attempted programmatically
When the ORM executes it
Then it is blocked at the query-builder level and raises `CrossTenantQueryError`

Given a new tenant is provisioned
When the async provisioning task completes
Then the tenant schema exists, all migrations have run, and a default admin user is created

Given a tenant is suspended
When any request arrives for that tenant
Then a 403 is returned with a suspension notice and no data is accessible

Given two concurrent requests arrive for different tenants
When both are processed simultaneously
Then each operates entirely within its own schema with no data leakage between them
```
