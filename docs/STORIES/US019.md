## User Story

As a developer, I want a notifications module so that I can dispatch notifications to any channel
through a single API with delivery tracking and user preference enforcement.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a notification is dispatched to a user who has disabled email for that notification type
When the dispatch runs
Then the email channel is skipped and the in-app notification is sent

Given an email delivery fails on the first attempt
When the retry policy triggers
Then it retries with exponential back-off up to `RETRY_MAX_ATTEMPTS` times before routing to DLQ

Given a push notification is delivered and FCM/APNs reports an invalid device token
When the response is processed
Then the invalid token is automatically removed from the database

Given a user updates their notification preferences
When the next notification is dispatched
Then the updated preferences are honoured ‚Äî no caching lag

Given a notification is dispatched as in-app
When the recipient's browser is connected via WebSocket
Then the notification is delivered in real time without a page refresh
```
