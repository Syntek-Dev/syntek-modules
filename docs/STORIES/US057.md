# US057 — `@syntek/mobile-ui`: NativeWind Mobile Design System

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Mobile Packages

---

## User Story

As a developer, I want a NativeWind mobile design system so that I can build consistent,
token-driven, accessible UIs on iOS and Android without duplicating component code across mobile
applications.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given design tokens are defined in `shared/tokens/`
When a mobile component renders
Then it uses token values for colour, spacing, and typography via NativeWind CSS custom properties

Given a component is rendered on both iOS and Android
When the UI is inspected
Then it uses platform-appropriate native primitives (e.g., no web-only HTML tags)

Given tenant design token overrides are received from the GraphQL API
When the token override is applied at the app root
Then all mobile components reflect the tenant brand without component code changes

Given a screen reader is active on a component
When the component renders
Then it provides correct accessibilityLabel, accessibilityRole, and accessibilityHint props
```
