# US011 — `syntek-permissions`: RBAC Module

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 3 — Core Backend
Modules

---

## User Story

As a developer, I want a permissions module so that I can define roles, assign permissions, and
enforce object-level access control at the ORM layer across all modules.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a user with `member` role attempts to access a resource owned by another user
When the ORM-layer permission check runs
Then access is denied and a 403 is returned

Given a permission bundle is assigned at the group level
When a group member makes a request
Then they inherit all permissions in the bundle without individual assignment

Given a JWT token contains a scope claim `orders:read`
When a GraphQL resolver requires `orders:write`
Then the resolver returns an authorisation error

Given tenant isolation is active
When any permission check runs
Then it is always scoped to the active tenant — cross-tenant grants cannot be created

Given a privilege escalation is attempted (e.g. a `member` trying a `superuser` action)
When the permission check runs
Then it is denied and the attempt is logged via syntek-audit
```
