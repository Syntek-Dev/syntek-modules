## User Story

As a developer, I want a site search UI package so that I can add a search bar with autocomplete, a
results page with faceted filtering, and keyword highlighting — all driven by a `useSearch` hook
that connects to `syntek-search-core` via GraphQL, and configurable to match any site's content
structure.

## Acceptance Criteria

```gherkin
Given a user types in the SearchInput
When the input reaches AUTOCOMPLETE_MIN_CHARS characters and 300ms have passed since the last keystroke
Then autocomplete suggestions are fetched and displayed in a dropdown below the input

Given the user submits a search query or selects a suggestion
When the search executes
Then the useSearch hook returns results, facets, total count, and loading/error state

Given search results are returned
When SearchResults renders each item
Then matched terms in the title and snippet are visually highlighted

Given facet options are returned with the results
When FacetsPanel renders
Then each facet shows option labels with result counts; active filters are visually indicated with
a clear/remove affordance

Given a facet filter is toggled
When the filter state changes
Then useSearch re-fetches with the updated filter variables and the URL query string reflects
all active filters and the current query

Given useSearch is used without SearchInput or SearchResults components
When the hook is called with a query string and optional filters
Then it returns results, loading, error, facets, and a refetch function — the hook is fully
usable headlessly without any of the provided components

Given the search index slug is set on SearchProvider
When any child component calls useSearch or useAutocomplete
Then all queries are scoped to that index — no per-component index configuration required
```

## Components

### `SearchProvider`

Context provider that holds index slug, active query, active filters, and pagination state. Wraps
the section of the app where search is used. All other components and hooks consume this context.

```tsx
<SearchProvider index="pages" defaultPageSize={10}>
  <SearchInput />
  <FacetsPanel />
  <SearchResults />
</SearchProvider>
```

### `SearchInput`

Controlled text input with debounced autocomplete dropdown. Submits on Enter or suggestion
selection. Clears results when emptied.

```tsx
<SearchInput
  placeholder="Search..."
  autoFocus
  onSubmit={(query) => router.push(`/search?q=${query}`)}
/>
```

### `SearchResults`

Renders the result list. Accepts an optional `renderItem` render prop for custom result
presentation; falls back to the default card layout (title, snippet, highlighted terms, URL).

```tsx
<SearchResults
  renderItem={(result) => (
    <a href={result.url}>
      <h3 dangerouslySetInnerHTML={{ __html: result.highlightedTitle }} />
      <p dangerouslySetInnerHTML={{ __html: result.snippet }} />
    </a>
  )}
/>
```

Note: `result.highlightedTitle` and `result.snippet` are sanitised server-side by
`syntek-search-core` before GraphQL delivery — safe to render with `dangerouslySetInnerHTML`.

### `FacetsPanel`

Renders facet groups as checkbox filter lists. Each facet option shows its label and result count.
Active facets show a remove affordance. Communicates filter changes back to `SearchProvider`
context.

```tsx
<FacetsPanel
  facets={["category", "author", "published_year"]}
  labels={{ category: "Category", author: "Author", published_year: "Year" }}
/>
```

### `SearchCommandPalette`

Optional global search overlay triggered by a configurable keyboard shortcut (default `⌘K` /
`Ctrl+K`). Renders a floating `SearchInput` with live autocomplete results. Useful for admin
interfaces and dashboards.

```tsx
<SearchCommandPalette
  shortcut="mod+k"
  placeholder="Search anything..."
  onSelect={(result) => router.push(result.url)}
/>
```

## Hooks

| Hook                          | Returns                                                           |
| ----------------------------- | ----------------------------------------------------------------- |
| `useSearch(query?, filters?)` | `{ results, facets, total, loading, error, refetch, pagination }` |
| `useAutocomplete(prefix)`     | `{ suggestions, loading }` — debounced, min-char guarded          |
| `useSearchFilters()`          | `[activeFilters, setFilter, clearFilter, clearAll]`               |
| `useSearchPagination()`       | `[page, setPage, pageSize, setPageSize]`                          |

`useSearch` is the primary API. All components consume it internally — consuming projects can use
the hook alone without mounting any of the provided components.

## URL Sync

`SearchProvider` accepts a `syncUrl` prop (default `true`) that mirrors query and active filters to
the URL query string (`?q=...&category=news&author=sam`). On mount, it reads initial state from the
URL so search results are shareable and browser-navigable.

## Configuration

`SearchProvider` accepts:

| Prop              | Type    | Default | Description                                                |
| ----------------- | ------- | ------- | ---------------------------------------------------------- |
| `index`           | string  | —       | Required. Maps to `SYNTEK_SEARCH['REGISTERED_MODELS']` key |
| `defaultPageSize` | number  | 10      | Results per page                                           |
| `syncUrl`         | boolean | true    | Sync query + filters to URL query string                   |
| `debounceMs`      | number  | 300     | Autocomplete debounce delay                                |
| `minChars`        | number  | 2       | Minimum chars before autocomplete fires                    |

## Integration Points

### Global site search

Mount `SearchProvider` at a `/search` route with `SearchInput`, `FacetsPanel`, and `SearchResults`.
Pass the index slug that matches the consuming project's configured index.

### Header search bar

Mount `SearchInput` in the site header. On submit, navigate to the `/search` route with the query in
the URL — no `SearchProvider` needed in the header itself; the results page provides context.

### Command palette

Use `SearchCommandPalette` independently anywhere in the component tree — it self-contains its own
provider scoped to the command palette instance.
