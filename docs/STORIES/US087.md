# US087 — `syntek-notifications-email`: Email Notification Channel

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want an email notification channel adapter so that I can send transactional
notification emails via SMTP, AWS SES, SendGrid, or Mailgun — with HTML/plaintext template
rendering, unsubscribe handling, and basic bounce tracking.

## MoSCoW Priority

**Must Have**

## Acceptance Criteria

```gherkin
Given a notification is dispatched with the email channel enabled
When the email adapter is installed and the user has a verified email address
Then the email is rendered from the configured Jinja2 template and sent via the configured backend

Given an email send fails with a transient (soft) error
When the core retry policy re-enqueues the task
Then the email adapter re-attempts delivery using the same backend

Given an email is rejected by the provider with a hard bounce code
When the delivery response is processed
Then the recipient's email address is flagged as hard-bounced and excluded from all future sends
until an admin re-verifies the address

Given a recipient clicks the unsubscribe link in a transactional notification email
When the unsubscribe handler processes the request
Then the notification type is disabled in their NotificationPreference and no further emails are
sent for that type — the change takes effect on the next dispatch with no caching lag

Given an email is sent with HTML content
When the template is rendered
Then a plain-text fallback is included as a multipart alternative
```

## Tasks

- [ ] **Red phase:** write failing tests — backend switching, HTML/plaintext rendering, hard bounce
      exclusion, unsubscribe handling, missing-address no-op
- [ ] Implement `SYNTEK_NOTIFICATIONS_EMAIL` settings schema (backend choice: smtp/ses/sendgrid/
      mailgun, API keys, from address, reply-to, unsubscribe footer toggle, bounce webhook secret)
- [ ] Implement backend adapters: SMTP, AWS SES, SendGrid, Mailgun — all satisfy the same
      `EmailBackend` protocol; switching is a config-only change
- [ ] Implement Jinja2 HTML template engine with base layout (header, footer, unsubscribe link) and
      per-notification-type template override support
- [ ] Implement automatic plain-text fallback via `html2text` — no separate template required for
      plain-text unless explicitly overridden
- [ ] Implement HMAC-signed unsubscribe link generation and handler view — links are valid for
      `UNSUBSCRIBE_LINK_TTL` seconds; expired or tampered links return a 400
- [ ] Implement `TransactionalEmailBounce` model — stores bounce type (hard/soft), provider code,
      and timestamp; hard bounces set a `hard_bounced` flag on the user's profile
- [ ] Implement provider bounce webhook endpoint — verifies HMAC/provider signature before
      processing; handles SendGrid, Mailgun, and AWS SES SNS bounce notifications
- [ ] Register the email adapter with the `syntek-notifications-core` channel registry via
      `AppConfig.ready()`
- [ ] Create `syntek.manifest.toml` — module ID, settings keys with types and defaults, and
      `INSTALLED_APPS` entry
- [ ] **Green phase:** all email channel tests pass
