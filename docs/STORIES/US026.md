# US026 — `syntek-invoicing`: Invoicing, PDF Generation & UK VAT Module

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

## User Story

As a developer, I want an invoicing module so that I can generate professional PDF invoices with UK
VAT calculation, send them by email, embed payment links, and maintain HMRC Making Tax Digital
compatible records — with all behaviour configurable via `SYNTEK_INVOICING` settings.

**Should Have**

## Acceptance Criteria

```gherkin
Given an invoice is created with line items at mixed VAT rates (20%, 5%, 0%)
When the invoice is finalised
Then VAT amounts are calculated correctly per rate and the totals balance to the penny

Given an invoice is finalised
When the PDF generation task runs via Celery
Then a valid, paginated PDF is produced with invoice number, line items, VAT breakdown, payment terms, and the configured branding (logo, colours, footer) from `SYNTEK_INVOICING['PDF_TEMPLATE']`

Given a credit note is created against a finalised invoice
When the credit note is applied
Then the invoice balance is reduced and both documents reference each other by number

Given a new invoice is created
When the invoice number is assigned
Then the sequence is gap-free, no two invoices share the same number (DB-level unique constraint), and the prefix and numbering format follow `SYNTEK_INVOICING['INVOICE_NUMBER_FORMAT']`

Given an invoice is sent to a customer
When its status is queried
Then it transitions correctly through: draft → sent → partial → paid / overdue / void

Given a payment link is embedded in an invoice
When the customer opens the invoice PDF or email
Then a clickable payment link (generated via syntek-payment-links) is present pointing to the correct amount and invoice reference

Given an invoice is sent via email
When the send mutation is called
Then the invoice PDF is attached and dispatched via syntek-notifications using the configured email channel
```
