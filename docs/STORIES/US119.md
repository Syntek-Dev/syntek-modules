# US119 — `@syntek/mobile-comments`: Mobile Comments & Reviews Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 8 — Mobile Packages

---

## User Story

As a developer, I want a React Native comments package so that I can render threaded discussions,
reviews, and reaction controls on mobile — wired to `syntek-comments` via GraphQL and optimised for
touch interaction.

## Acceptance Criteria

```gherkin
Given a MobileCommentThread is rendered for a target model
When the component mounts
Then comments are fetched and displayed in a FlatList; pull-to-refresh triggers a refetch;
  nested replies are progressively disclosed behind a "show N replies" control

Given a user taps the reply button
When the inline ReplySheet (BottomSheet) opens
Then the keyboard raises and the input receives focus; submitting calls addComment with the
  correct parentId; the new reply appears optimistically in the thread

Given VOTING_MODE is updown
When a user taps an up or down vote button
Then the vote is toggled with haptic feedback; the same direction cancels the vote; the net
  score updates optimistically and rolls back on error

Given the current user has the moderator permission
When a pending comment is rendered
Then an Approve option appears in the comment's action sheet; tapping it calls approveComment

Given a user long-presses a comment
When the action sheet opens
Then available actions are shown (reply, edit own, delete own, flag, approve if moderator);
  tapping Flag calls flagComment and dismisses the sheet
```

## Components

| Component             | Description                                                                             |
| --------------------- | --------------------------------------------------------------------------------------- |
| `MobileCommentThread` | Root FlatList — top-level comments with pull-to-refresh and pagination                  |
| `MobileCommentItem`   | Single comment with body, author, timestamp, reaction/vote bar, long-press action sheet |
| `MobileReplySheet`    | BottomSheet with text input and submit button for adding replies                        |
| `MobileCommentForm`   | Standalone form for top-level comment submission (can be embedded in a screen)          |
| `MobileReactionBar`   | Reaction buttons or up/down vote controls beneath a comment                             |
| `MobileStarRating`    | Tap-to-rate star widget (reviews mode) — interactive and read-only variants             |
| `MobileReviewSummary` | Aggregate rating and count header above the comment list                                |

## Hooks

| Hook                                                | Returns                                                         |
| --------------------------------------------------- | --------------------------------------------------------------- |
| `useMobileComments(contentType, objectId, options)` | `{ comments, loading, error, refetch, fetchMore, hasNextPage }` |
| `useMobileAddComment(options)`                      | `{ addComment, loading, error }` with optimistic update         |
| `useMobileReaction(commentId)`                      | `{ reactions, addReaction, removeReaction, loading }`           |
| `useMobileVote(commentId)`                          | `{ upCount, downCount, userVote, castVote, loading }`           |
