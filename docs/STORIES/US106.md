## User Story

As a developer, I want a media upload pipeline module so that large files are uploaded in chunks
with real-time progress tracking, scanned for viruses via ClamAV before being committed to
Cloudinary or MinIO, and failed or incomplete uploads are cleaned up automatically.

## Acceptance Criteria

```gherkin
Given a client initiates a chunked upload
When the first chunk is received
Then an UploadSession is created and a session token is returned to the client

Given subsequent chunks are received in any order
When each chunk arrives
Then it is stored temporarily and the session progress is updated; no partial commit to Cloudinary
occurs until all chunks are confirmed

Given all chunks have been received and assembled
When ClamAV scans the assembled file
Then if clean, the file is committed to syntek-media-core or syntek-files; if infected, the
assembled file is deleted and the session is marked REJECTED with reason "virus_detected"

Given an upload session has no activity for SESSION_TTL_SECONDS
When the cleanup task runs
Then the session and all stored chunks are deleted and no commit is attempted

Given a file exceeds MAX_UPLOAD_SIZE_MB
When the initiation request is validated
Then the session is rejected immediately with 413 before any chunk storage occurs

Given VIRUS_SCAN_ENABLED is False
When a clean upload completes assembly
Then the file is committed directly without ClamAV тАФ the scan step is skipped entirely
```

## Configuration

```python
SYNTEK_MEDIA_UPLOAD = {
    # --- Chunking ---
    'CHUNK_SIZE_MB': 5,              # Target chunk size for client guidance
    'MAX_UPLOAD_SIZE_MB': 500,       # Hard limit across all resource types
    'CHUNK_STORAGE_BACKEND': 'redis', # 'redis' | 'local' | 's3' тАФ temporary chunk store

    # --- Sessions ---
    'SESSION_TTL_SECONDS': 3600,    # Incomplete session auto-expiry
    'MAX_CONCURRENT_UPLOADS': 10,   # Per-user concurrency limit

    # --- Virus scanning ---
    'VIRUS_SCAN_ENABLED': True,
    'CLAMD_HOST': env('CLAMD_HOST', default='127.0.0.1'),
    'CLAMD_PORT': int(env('CLAMD_PORT', default='3310')),
    'CLAMD_TIMEOUT_SECONDS': 30,

    # --- Progress ---
    'PROGRESS_BACKEND': 'redis',    # 'redis' | 'database' тАФ where progress state is stored
    'PROGRESS_TTL_SECONDS': 86400,  # How long progress records are kept after completion

    # --- Commit targets ---
    'DEFAULT_COMMIT_TARGET': 'media',   # 'media' (syntek-media-core) | 'files' (syntek-files)
}
```

### Upload Flow

```text
Client тЖТ POST /upload/initiate   тЖТ UploadSession created, session token returned
Client тЖТ POST /upload/chunk/{n}  тЖТ Chunk stored in CHUNK_STORAGE_BACKEND
Client тЖТ POST /upload/complete   тЖТ All chunks assembled
                                  тЖТ ClamAV scan (if VIRUS_SCAN_ENABLED)
                                  тЖТ CLEAN: commit to media-core or files
                                  тЖТ INFECTED: delete, mark REJECTED
Client тЖТ GET  /upload/progress   тЖТ UploadSession status + bytes_received / bytes_total
Client тЖТ POST /upload/cancel     тЖТ Delete session + all chunks
```

All upload endpoints require authentication. The session token is a signed JWT valid only for the
initiating user and session ID.

## GraphQL API

| Mutation                                                      | Description                                 |
