# US129 — `@syntek/mobile-ai-chat` — React Native AI Chat UI

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Story Points**: 5

**Epic**: Epic 8 — Mobile Packages

## User Story

**As a** developer

**I want** a React Native AI chat UI package

**So that** I can embed a full-screen chat screen or a floating chat button in any Expo project,
with streaming message rendering, image attachment support, and NativeWind token-driven styling —
wired to `syntek-ai` via GraphQL and SSE.

---

- **Must Have:**
  - Full-screen chat screen (`ChatScreen`)
  - Floating action button (`ChatFAB`) that navigates to `ChatScreen`
  - Streaming message rendering
  - Message list with role-based alignment
  - Input bar with send button
  - Session creation and retrieval
  - GraphQL integration (`@syntek/api-client`)
  - SSE stream consumption (via polyfill)

- **Should Have:**
  - Image attachment support (via `expo-image-picker`)

  - Auto-scroll to latest message

  - Voice input via Expo Audio

  - Search within conversation
  - Dark mode toggle

- **Won't Have:**
  - Web-responsive design (see US128 for web)
  - Video attachment
  - Real-time multi-user collaboration
  - Voice output (TTS)

## Architecture

### Core Design

The package provides `ChatScreen` (full-screen React Navigation screen) and `ChatFAB` (floating
action button that pushes to `ChatScreen`). The same `useChat` hook from `@syntek/ui-ai-chat` is
re-exported and adapted for React Native's `EventSource` polyfill (via `react-native-sse`).

Markdown rendering uses `react-native-markdown-display`. Image attachments use `expo-image-picker`.
All styling uses NativeWind CSS variables from `@syntek/mobile-ui`.

### Components

| Component                  | Description                                                                                                  |
| -------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `ChatScreen`               | Full-screen chat — `FlatList` of messages, bottom input bar, keyboard-avoiding behaviour                     |
| `ChatFAB`                  | Floating action button (configurable position) that navigates to `ChatScreen`                                |
| `MobileChatMessage`        | Message bubble — user/assistant, role-based alignment, markdown, code block                                  |
| `MobileChatInput`          | `TextInput` with send button, image attachment button (`expo-image-picker`), optional voice note placeholder |
| `MobileStreamingIndicator` | Three-dot animation (React Native `Animated` API)                                                            |

### Hooks

**`useChat` (re-exported from `@syntek/ui-ai-chat`)**

Adapted for React Native with `react-native-sse` polyfill.

```typescript
interface UseChatOptions {
  sessionId?: string;
  systemPrompt?: string;
  model?: string;
  onError?: (error: Error) => void;
}

interface UseChatReturn {
  messages: ChatMessage[];
  isStreaming: boolean;
  sessionId: string | null;
  sendMessage: (content: string, attachments?: File[]) => void;
  startNewSession: () => void;
  clearHistory: () => void;
  isLoading: boolean;
  error?: Error;
}
```

### Component Examples

**`ChatScreen` Usage (React Navigation)**

```typescript
import { ChatScreen } from '@syntek/mobile-ai-chat'

// In your navigation configuration
export const ChatStack = createNativeStackNavigator()

export function ChatStackNavigator() {
  return (
    <ChatStack.Navigator>
      <ChatStack.Screen
        name="Chat"
        component={ChatScreen}
        options={{ title: 'AI Assistant' }}
      />
    </ChatStack.Navigator>
  )
}
```

**`ChatFAB` Usage**

```typescript
import { ChatFAB } from '@syntek/mobile-ai-chat'
import { useNavigation } from '@react-navigation/native'

export default function HomeScreen() {
  const navigation = useNavigation()

  return (
    <View className="flex-1">
      {/* Screen content */}
      <ChatFAB
        position="bottom-right"
        onPress={() => navigation.navigate('Chat')}
      />
    </View>
  )
}
```

### Styling Strategy

All components use NativeWind CSS classes for styling. No hardcoded colours or sizes; all values
derive from design tokens defined in `@syntek/mobile-ui`.

- `className` prop accepts NativeWind CSS
- `dark:` variants for dark mode
- Design tokens for spacing, typography, colours
- Platform-specific (iOS vs Android) styling where needed

## Acceptance Criteria

### Scenario 1: Session Creation on First Message

**Given** a user opens `ChatScreen` without an existing session

**When** they type a message and tap send

**Then** the system:

- Calls `startConversation` mutation
- Creates a new session server-side
- Sends the message immediately
- Opens SSE stream for the response

**And** the session ID is stored for subsequent messages

---

### Scenario 2: SSE Streaming via Polyfill

**Given** `react-native-sse` polyfill is configured

**When** the SSE stream is open

**Then** tokens:

- Arrive from `text/event-stream` response
- Are buffered and rendered incrementally
- Update the message UI without blocking

**And** the app does not freeze during streaming

---

### Scenario 3: Message List in Order

**Given** multiple messages exist in the conversation

**When** `ChatScreen` loads

**Then** messages are:

- Rendered in chronological order (oldest to newest)
- Aligned left (assistant) or right (user) based on role
- Scrollable if list exceeds screen height

---

### Scenario 4: Image Attachment Picker Launch

**Given** the user taps the attachment button in `MobileChatInput`

**When** they select an image from their device

**Then** the system:

- Opens `expo-image-picker`
- Displays the selected image thumbnail
- Sends the image with the message when sent

**And** the image appears in the message history inline

---

### Scenario 5: Keyboard-Avoiding Behaviour

**Given** `ChatScreen` is displayed and the user taps the input field

**When** the system keyboard opens

**Then** the screen:

- Shifts the content upward
- Keeps the input field visible
- Maintains scroll position (no content loss)

**And** the user can continue typing comfortably

---

### Scenario 6: Message Markdown Rendering

**Given** an assistant message contains markdown (bold, italic, code blocks)

**When** the message is displayed

**Then** the system:

- Renders markdown to native components
- Shows bold/italic text styling
- Displays code blocks with monospace font and background
- Does NOT render as plain text

---

### Scenario 7: Streaming Indicator During Response

**Given** the SSE stream is active

**When** the assistant message is being built

**Then** a `MobileStreamingIndicator` (three-dot animation) appears at the end

**When** the stream ends

**Then** the indicator disappears

## Notes

- The `useChat` hook is shared between web (US128) and mobile (US129), but the mobile variant uses
  `react-native-sse` for EventSource support
- Image attachments are uploaded server-side (via `syntek-ai` and configured `ATTACHMENT_BACKEND`)
- Markdown rendering uses `react-native-markdown-display`, which is lighter-weight than full HTML
  rendering
- Keyboard-avoiding behaviour is required for chat UX; `KeyboardAvoidingView` is the standard React
  Native solution
- The floating action button is optional; consuming projects can choose to navigate to `ChatScreen`
  via navigation drawer, tab bar, or other means
- Dark mode support is configurable via NativeWind's `dark:` variants; the consuming project
  controls the theme mode
