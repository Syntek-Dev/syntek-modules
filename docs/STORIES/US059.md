# US059 — `@syntek/mobile-notifications`: Push Notifications

**Last Updated**: 06/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Mobile Packages

---

## User Story

As a developer, I want a mobile push notifications package so that I can register devices for
FCM/APNs, receive push notifications, and route deep links from notification taps to the correct
screen.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given the user grants notification permission
When the Expo push token is retrieved
Then the token is registered with the backend via the `registerPushToken` mutation

Given a push notification is received while the app is in the foreground
When the notification arrives
Then it is displayed as an in-app banner using the notification UI component

Given a push notification is tapped from the system tray
When the app opens
Then the deep link in the notification data routes the user to the correct screen

Given the user revokes notification permission in device settings
When the app next opens
Then the push token is de-registered from the backend
```
