# US088 — `syntek-notifications-sms`: SMS Notification Channel

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want an SMS notification channel adapter so that I can send transactional SMS
messages and OTP codes via Twilio or Bandwidth — with template support and bulk sending.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given a notification is dispatched with the SMS channel enabled
When the SMS adapter is installed and the user has a verified mobile number
Then the message is rendered from the SMS template and sent via the configured provider

Given an SMS is sent to an invalid or unregistered number
When the provider returns an undeliverable status
Then the number is flagged as invalid in the user's profile and no further SMS is sent to it

Given an OTP is requested via SMS
When the OTP task is triggered
Then a time-limited one-time code is generated, sent, and stored for verification

Given a bulk SMS dispatch is triggered for a notification type
When the Celery task processes the queue
Then messages are sent in provider-safe batches respecting rate limits — no single-thread blocking

Given `syntek-notifications-sms` is uninstalled or the SMS channel is disabled
When a notification is dispatched
Then the core engine skips the SMS channel silently — no exception raised
```

## Tasks

- [ ] **Red phase:** write failing tests — provider switching, template rendering, invalid number
      flagging, OTP generation and verification, bulk batching, absent-adapter no-op
- [ ] Implement `SYNTEK_NOTIFICATIONS_SMS` settings schema (provider choice: twilio/bandwidth, API
      credentials, sender ID/number, batch size, rate limit config, OTP TTL)
- [ ] Implement provider adapters: Twilio and Bandwidth — both satisfy the same `SmsBackend`
      protocol; switching is a config-only change
- [ ] Implement SMS template engine — plaintext only, character count enforced at 160 chars per
      segment with configurable max segments
- [ ] Implement `SmsDeliveryRecord` model — stores provider message ID, status, error code, and
      timestamp; status updated via provider webhook
- [ ] Implement provider status webhook endpoint — verifies provider signature before processing;
      handles Twilio and Bandwidth delivery receipt callbacks
- [ ] Implement invalid number flagging — on permanent failure, set `mobile_invalid` flag on the
      user profile; excluded from all future SMS until cleared by admin
- [ ] Implement OTP via SMS — generates a cryptographically random 6-digit code (secrets module),
      stores HMAC of the code in cache with TTL from `OTP_TTL` setting, sends via SMS adapter
- [ ] Implement bulk SMS Celery task — fan-out with configurable batch size and inter-batch delay to
      respect provider rate limits
- [ ] Register the SMS adapter with the `syntek-notifications-core` channel registry via
      `AppConfig.ready()`
- [ ] Create `syntek.manifest.toml` — module ID, settings keys with types and defaults, and
      `INSTALLED_APPS` entry
- [ ] **Green phase:** all SMS channel tests pass
