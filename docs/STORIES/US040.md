## User Story

As a developer, I want a fully configurable comments module so that I can attach threaded
discussions, reviews, or voting to any model in my application — with moderation, spam detection,
and reaction support built in.

## Acceptance Criteria

```gherkin
Given a comment is submitted on a model instance
When the comment is saved
Then it is linked via a generic FK (content_type + object_id), associated with the author, and
  the COMMENT_MODE (threaded / flat / reviews) determines the data structure stored

Given a nested reply is submitted to a comment
When the reply is saved
Then it is linked to the parent comment, depth is incremented, and replies beyond MAX_DEPTH are
  rejected with a clear validation error

Given a comment body matches a configured spam pattern or Akismet scores it as spam
When the spam check runs
Then the comment status is set to spam and it is not displayed; if AUTO_APPROVE_SPAM is False
  a moderator review is queued

Given a moderator approves a held comment
When approveComment mutation completes
Then the comment status changes to approved, is visible to readers, and an audit entry is recorded
  via syntek-audit (US013)

Given a user adds a reaction to a comment
When the addReaction mutation completes
Then the aggregate reaction counts update immediately; the same user adding the same reaction type
  a second time removes the reaction (toggle behaviour)

Given VOTING_MODE is set to updown
When a user casts a vote
Then up_count and down_count fields are updated and the net score is recalculated; the same user
  cannot vote in both directions simultaneously

Given ALLOW_ANONYMOUS is False and an unauthenticated request is received
When addComment is called
Then a PermissionDenied error is returned with no comment created

Given an erasure request is processed for a user
When the GDPR erasure hook fires
Then all Comment.body fields attributed to that user are replaced with the configured
  ERASURE_PLACEHOLDER text and the author FK is set to null
```

## Configuration

```python
SYNTEK_COMMENTS = {
    # General
    'COMMENT_MODE': 'threaded',        # threaded | flat | reviews (star rating required)
    'MAX_DEPTH': 3,                    # max nesting depth; 1 = flat replies only
    'ALLOW_ANONYMOUS': False,          # permit unauthenticated comments
    'ANONYMOUS_LABEL': 'Anonymous',    # display name for anonymous comments

    # Content
    'MAX_BODY_LENGTH': 5000,           # characters
    'MIN_BODY_LENGTH': 1,
    'ALLOW_RICH_TEXT': False,          # True = Markdown input, stored as Markdown, rendered server-side
    'ALLOWED_MARKDOWN_ELEMENTS': ['b', 'i', 'a', 'code', 'blockquote'],

    # Moderation
    'DEFAULT_STATUS': 'pending',       # pending | approved (auto-approve)
    'MODERATION_KEYWORDS': [],         # list of strings; match → hold for review
    'SPAM_BACKEND': 'none',            # none | akismet | builtin
    'AKISMET_API_KEY': '',
    'AUTO_REJECT_SPAM': False,         # True = auto-reject; False = hold for moderator review
    'RATE_LIMIT_PER_HOUR': 10,         # comments per user per hour; 0 = unlimited

    # Reactions
    'REACTIONS_ENABLED': True,
    'REACTION_TYPES': ['like', 'love', 'laugh', 'angry'],  # arbitrary slugs
    'VOTING_MODE': 'none',             # none | updown — replaces REACTION_TYPES if set
    'MAX_REACTIONS_PER_USER': 1,       # reactions per comment per user (1 = exclusive)

    # Reviews mode (only used when COMMENT_MODE = reviews)
    'REVIEW_MIN_RATING': 1,
    'REVIEW_MAX_RATING': 5,
    'REVIEW_LABEL': 'Rating',          # label shown in UI

    # GDPR
    'ERASURE_PLACEHOLDER': '[comment removed]',

    # Tenant
    'TENANT_SCOPED': True,
}
```

## Models

| Model             | Key Fields                                                                                                                                                                                                                                       |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Comment`         | `tenant`, `content_type`, `object_id`, `author` (nullable), `anonymous_name`, `body`, `body_format` (plain/markdown), `rating` (reviews mode), `parent` (FK→self), `depth`, `status` (pending/approved/rejected/spam), `edited_at`, `is_deleted` |
| `CommentReaction` | `comment` FK, `user` FK, `reaction_type`, unique on `(comment, user, reaction_type)`                                                                                                                                                             |

| `CommentFlag`     | `comment` FK, `flagged_by` FK, `reason`, `resolved_by`, `resolved_at` — user-submitted spam reports                                                                                                                                              |

## GraphQL API

### Queries

| Query                                             | Returns                                                    |
| ------------------------------------------------- | ---------------------------------------------------------- |
| `comments(contentType, objectId, page, pageSize)` | Paginated top-level comments for a target object           |
| `commentThread(id, depth)`                        | Single comment with nested replies to configured depth     |
| `commentModerationQueue(contentType, objectId)`   | Pending/held comments for moderators                       |
| `commentAnalytics(contentType, objectId)`         | Total counts, approval rate, average rating (reviews mode) |

### Mutations

| Mutation                                  | Description                                       |
| ----------------------------------------- | ------------------------------------------------- |
| `addComment(input)`                       | Create a new top-level comment or reply           |
| `editComment(id, body)`                   | Edit own comment body; sets `edited_at`           |
| `deleteComment(id)`                       | Soft-delete own comment; sets `is_deleted = True` |
| `approveComment(id)`                      | Moderator approves a pending comment              |
| `rejectComment(id, reason)`               | Moderator rejects; reason stored in audit log     |
| `addReaction(commentId, reactionType)`    | Toggle a reaction on a comment                    |
| `removeReaction(commentId, reactionType)` | Explicitly remove a reaction                      |
| `castVote(commentId, direction)`          | Cast or switch an up/down vote                    |
| `flagComment(commentId, reason)`          | Report a comment for moderator review             |
