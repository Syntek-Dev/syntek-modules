# US068 — `@syntek/ui-donations`: Donations UI Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

## User Story

As a developer, I want a donations UI package so that I can embed configurable donation forms with
Gift Aid declarations, recurring options, campaign progress, receipt downloads, and payment forms —
all wired to `syntek-donations` and `@syntek/ui-payments` via GraphQL.

## Acceptance Criteria

```gherkin
Given the DonationForm is rendered
When a user selects a preset amount and clicks Donate
Then the amount is passed to the payment form from @syntek/ui-payments, and the form cannot be resubmitted until the first submission resolves

Given a user selects recurring donation
When the donation type toggle changes
Then the form updates to show billing frequency options and the correct recurring mutation is called on submission

Given a user is eligible for Gift Aid
When they check the Gift Aid declaration checkbox
Then the full declaration text is displayed before submission and the `giftAid: true` flag is included in the mutation payload

Given Gift Aid is disabled in backend settings
When the DonationForm renders
Then no Gift Aid section appears in the form

Given a donation is completed
When the confirmation screen renders
Then a receipt download button is shown and the receipt PDF can be downloaded from the GraphQL endpoint

Given a fundraising campaign has a target
When CampaignProgress renders
Then the raised amount, target, and percentage bar update in real time via subscription or refetch

Given a user views their giving history
When GivingHistory renders
Then all past donations are listed with amount, date, campaign, Gift Aid status, and a receipt download link — paginated with infinite scroll
```
