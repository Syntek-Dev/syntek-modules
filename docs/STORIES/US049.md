# US049 — `@syntek/ui-gdpr`: Multi-Regime Compliance UI Package

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 7 — Web Frontend
Features

---

## User Story

As a developer, I want a multi-regime data protection UI package so that I can embed a regime-aware
consent banner, privacy centre with data subject rights, data export request UI, erasure
confirmation flow, and granular consent management — with all user-facing text and behaviour
configurable for any national data protection regulation.

## Key Coverage

- `useGDPR(options?)` hook — reads regime config from GraphQL (`consentConfiguration` query),
  exposes `regime`, `purposes`, `submitSAR`, `submitErasureRequest`, `submitPortabilityRequest`,
  `optOutOfSale` (CCPA), `updateConsent`, `consentState`
- `ConsentBanner` component — regime-aware: GDPR/UK-GDPR shows category toggles
  (essential/analytics/marketing/personalisation); CCPA shows "Do Not Sell My Personal Information"
  opt-out; all text overridable via props
- `PrivacyCentre` component — full data subject rights portal: tabs for export, erasure,
  portability, consent history, CCPA do-not-sell — tabs rendered conditionally based on active
  regime
- `DataExportRequest` component — submit SAR form, progress polling, download button when ready
- `DataDeletionConfirmation` component — multi-step confirmation: lists data that will be
  anonymised, requires typed confirmation, shows timeline
- `ConsentManager` component — granular purpose-level toggles with legal basis explanations per
  purpose
- All labels/strings overridable via `labels` prop or `GDPRProvider` context — allows "Privacy
  Settings" vs "Cookie Preferences" vs "Your Privacy Rights" etc.
- `GDPRProvider` context wrapper — provides regime config, consent state, and mutation helpers to
  all child components

## Acceptance Criteria

```gherkin
Scenario 1: GDPR regime consent banner
Given regime = 'gdpr' and a user visits for the first time
When the page loads
Then ConsentBanner is shown before any non-essential scripts execute, with category toggles for analytics and marketing

Scenario 2: CCPA regime consent banner
Given regime = 'ccpa'
When ConsentBanner renders
Then a "Do Not Sell My Personal Information" opt-out button is shown instead of category toggles

Scenario 3: LGPD regime with label overrides
Given regime = 'lgpd'
When ConsentBanner renders
Then the banner uses LGPD-appropriate terminology and shows consent purposes required by LGPD

Scenario 4: Custom label overrides
Given custom labels prop is provided to ConsentBanner
When the component renders
Then all user-facing strings use the provided overrides, with no hardcoded English text visible

Scenario 5: Data export request submission
Given a user submits a data export request via DataExportRequest
When the SAR mutation resolves
Then a progress indicator is shown and the download button appears when the export is ready

Scenario 6: Account deletion flow
Given a user initiates account deletion via DataDeletionConfirmation
When they type the required confirmation string and submit
Then the erasure mutation is called and a confirmation message with the timeline is displayed

Scenario 7: Consent withdrawal
Given a user withdraws consent for purpose marketing via ConsentManager
When the withdrawal is saved
Then the backend updateConsent mutation is called, all marketing cookies are removed from the browser, and the consent state updates in useGDPR
```
