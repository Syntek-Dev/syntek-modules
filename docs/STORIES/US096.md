# US096 — `syntek-payments-square`: Square Adapter

**Last Updated**: 10/03/2026 **Version**: 0.1.0 **Status**: To Do **Epic**: Epic 4 — Backend Feature
Modules

---

## User Story

As a developer, I want a Square-specific payments adapter so that I can process payments using
Square's Web Payments SDK, manage cards on file, and handle Square webhooks through the
syntek-payments-core interface.

## MoSCoW Priority

**Should Have**

## Acceptance Criteria

```gherkin
Given Square adapter is configured
When a payment is created with a Square nonce
Then a Square payment is processed and a Transaction record is created

Given a Square webhook arrives
When the HMAC-SHA256 signature is verified
Then the event is routed to the correct handler

Given a Square webhook arrives with an invalid signature
When verification runs
Then the request is rejected with 401

Given a card on file is saved
When a subsequent payment is initiated
Then the payment is processed using the saved card without user re-entry
```
