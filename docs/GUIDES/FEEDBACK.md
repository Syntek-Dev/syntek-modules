# Feedback — syntek-modules

**Last Updated:** 08/03/2026 | **Maintained By:** Syntek Development Team

---

We welcome feedback on any aspect of syntek-modules — API design, developer experience, tooling,
documentation, or module behaviour.

---

## How to Give Feedback

### General feedback

Open a discussion on GitHub or Forgejo (when live):

- [GitHub Discussions](https://github.com/Syntek-Dev/syntek-modules/discussions)
- Forgejo Discussions at `git.syntek-studio.com` (coming soon)

Or email: `dev@syntek-studio.com`

### Module-specific feedback

When giving feedback on a specific module or package, please include:

- The module name (e.g., `syntek-auth`, `@syntek/ui`)
- The version you are using (see the module's `pyproject.toml`, `package.json`, or `Cargo.toml`)
- The consuming project context (e.g., a Django + React app, a React Native app)

### Documentation feedback

If something in the docs is unclear, incorrect, or missing — please open an issue or PR. See
`docs/GUIDES/ISSUES.md` for how to report it, or open a PR with the correction directly.

### Tooling and CLI feedback

If `syntek-dev` is missing a command or behaving unexpectedly, see `docs/GUIDES/ISSUES.md` for how
to report it, or open a feature request on GitHub.

---

## What Happens to Feedback

- Feedback that identifies a bug → converted to a bug report and triaged
- Feedback that suggests an improvement → evaluated against the roadmap and may become a user story
- Feedback on documentation → addressed directly in a patch
